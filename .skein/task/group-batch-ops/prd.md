# 分组内平台多选 + 批量操作 — PRD (主入口)

## 目标
在 Groups 页每个分组内支持平台多选,多选后对选中平台集合执行批量操作(原本需逐个进平台编辑才能完成的事)。用户价值:多平台统一管理(删一批/统一模型/统一启停/统一移组),免去逐个编辑的重复操作。

**每个批量操作是独立动作**(各自 modal,不混在一个编辑面板)。UI 尽可能用文字按钮表达(非纯图标)。

### 多选交互(brainstorm 定)
- per-group:每个分组卡顶部「多选」文字按钮,点击进入该组多选模式 → 每张平台卡左上角显 checkbox,顶部出操作工具栏;「取消」退出。
- 其他分组不受影响(各自独立多选)。
- 工具栏:[取消] [全选] 已选 N 个 | [删除] [覆盖模型] [改状态] [移组](文字按钮)。

### 批量操作清单(各自独立 modal)
1. **删除** — 物理删平台实体(DB DELETE)。确认 modal **全列可滚**展示将删的平台名 + 跨组副作用警告("平台 X 也将从分组 Y 移除")。
2. **覆盖模型配置** — 替换语义(选中平台 models 完全用新值覆盖,原模型丢)。diff 预览**全部列出**(每个平台 before→after,含"无变化")。新模型值三种来源:手输 / preset 下拉(协议默认模型)/ 从已有平台复制。
3. **改启用状态** — enabled/disabled 二态 modal(auto_disabled 系统熔断态不给批量手设)。
4. **移组** — modal 给两选项:移动(离开当前组进目标)/ 加入(保留当前组再加目标);目标组下拉。平台可属多组。

## 边界
### 范围内
- Groups 页 per-group 多选 + 4 个批量操作(删除/覆盖模型/改状态/移组)。
- 后端 4 个批量 command(各原子事务)。
- UI 文字按钮 + modal(createPortal,遵循 modal-window-center-rule)。

### 范围外(非目标)
- 不做跨分组全局多选(per-group 独立)。
- 不做批量改 endpoint / 配额 / 熔断 / peak_hours(低频仍单平台编辑)。后续可加。
- 不做批量操作历史/撤销(原子事务保证一致,但无 undo)。
- 不持久化多选态(临时操作态,刷新/离页清空,复用 navGuard 离页拦截)。
- 不做批量改 extra JSON 细项(coding_plan/breaker/time_models/disable_during_peak)— 非目标,复杂度高。

### 已知约束
- 平台-分组是多对多(`group_platform` 关联表),平台可属多组 → 删除/移组有跨组副作用(modal 须警告)。
- 批量操作**原子事务**:N 个平台任一失败 → 整批回滚,显式报失败原因 + "未改动"。禁部分成功。
- 进入多选模式时若有平台编辑未保存 → 复用现有 navGuard 未保存拦截(不额外加层)。
- 批量禁用致整组无可用候选 → 改状态 modal 警告"可能导致分组 X 无可用候选"(基于当前路由候选计算,非阻塞提示)。

## 验收标准
- [ ] 每个分组卡顶部有「多选」文字按钮;点击进入该组多选模式,卡显 checkbox + 顶部工具栏;「取消」退出,其他分组不受影响。
- [ ] 多选 ≥1 个平台后,工具栏 4 个批量操作按钮(删除/覆盖模型/改状态/移组)可用;0 个时禁用。
- [ ] 批量删除:确认 modal 全列可滚将删平台名 + 跨组副作用警告;确认后物理删,原子事务(全成或全回滚)。
- [ ] 批量覆盖模型:modal 三种来源(手输/preset/从别平台复制)+ diff 全列出每个平台 before→after;确认后替换,原子事务。
- [ ] 批量改状态:modal 启用/禁用二态 + 整组无候选警告;确认后原子事务。
- [ ] 批量移组:modal 移动/加入两选项 + 目标组下拉;确认后原子事务(移动=离当前组进目标,保留其他组关联;加入=保留所有现组再加目标)。
- [ ] 后端 4 个 batch command(各事务)+ db helper + generate_handler 注册;api.ts 封装 + TS 类型。
- [ ] i18n:8 语言 key 齐(check-i18n 通过)。
- [ ] 门禁:cargo clippy+test / yarn build+test+check:i18n 零回归。
