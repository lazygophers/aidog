# عرض السجلات

تسجّل صفحة **«السجلات»** كل طلب وكيل يمر عبر AiDog، مما يسهّل تتبّع مشكلات التوجيه وتحويل البروتوكول واستجابات الخادم الأعلى.

## قائمة السجلات

يعرض كل سجل المعلومات التالية:

| الحقل | الوصف |
|------|------|
| المجموعة | اسم المجموعة المطابِقة |
| النموذج | النموذج الأصلي الذي طلبه العميل |
| النموذج الفعلي | النموذج المُرسَل إلى الخادم الأعلى بعد تعيين النموذج |
| البروتوكول | بروتوكول الوارد ← بروتوكول الخادم الأعلى |
| المنصة | المنصة التي عالجت الطلب فعلياً |
| الحالة | رمز حالة HTTP (نجاح / فشل) |
| المدة | إجمالي مدة الطلب (بالميلي ثانية) |
| Token | عدد رموز الإدخال / الإخراج / التخزين المؤقت |
| الوقت | وقت حدوث الطلب |

تعرض القائمة 50 سجلاً لكل صفحة، مع دعم التنقّل بين الصفحات.

## التصفية

حدّد موقع السجلات بسرعة عبر أداة التصفية العلوية:

- **المنصة**: التصفية حسب منصة محددة
- **المجموعة**: التصفية حسب المجموعة
- **الحالة**: الناجحة فقط / الفاشلة فقط
- **النطاق الزمني**: الكل / آخر ساعة واحدة / 6 ساعات / 24 ساعة / 7 أيام / 30 يوماً
- **النموذج**: البحث حسب اسم النموذج الأصلي أو النموذج الفعلي

## تفاصيل الطلب

انقر على أي سجل لتوسيع التفاصيل وعرض مسار الطلب الكامل:

- **طلب المستخدم**: عنوان URL الأصلي وترويسات الطلب ونص الطلب
- **طلب الخادم الأعلى**: عنوان URL وترويسات الطلب ونص الطلب المُرسَلة إلى الخادم الأعلى بعد التحويل
- **استجابة الخادم الأعلى**: رمز الحالة وترويسات الاستجابة ونص الاستجابة التي يعيدها الخادم الأعلى
- **استجابة المستخدم**: ترويسات الاستجابة ونص الاستجابة النهائية المُعادة إلى العميل

تدعم التفاصيل النسخ بنقرة واحدة، مما يسهّل لصقها في تذكرة دعم أو أداة تصحيح أخطاء.

:::tip
يخضع محتوى «طلب المستخدم» و«طلب الخادم الأعلى» المُسجَّل في التفاصيل لإعدادات السجلات، ولا يُسجَّل نص الطلب افتراضياً. للتفاصيل راجع [إعدادات السجلات](/logs/log-settings).
:::
