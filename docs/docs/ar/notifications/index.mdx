# إشعارات النظام

يضم AiDog وحدة إشعارات تنبّهك على الأحداث الرئيسية لمساعدات البرمجة مثل Claude Code / OpenAI Codex (اكتمال المهمة، انتظار الإدخال، خطأ) عبر أربعة قنوات — **TTS / منبثقة سطح المكتب / صوت / صندوق الوارد** — لتتبع حالة البروكسي حتى بعيدًا عن الشاشة.

## آلية العمل

1. يستدعي المساعد النقطة المحلية `POST /api/notify` عبر سكربت hook عند الحدث
2. يوزّع AiDog حسب النوع (`NotifType`) والقنوات المهيّأة (`Channels`)
3. يقرأ TTS، تظهر إشعار النظام، يُشغّل صوت، ويُكتب في صندوق الوارد (للمراجعة لاحقًا)

## أنواع الإشعارات

أربعة `NotifType`:

| النوع | الحرفي | المُطلِق |
| --- | --- | --- |
| TaskComplete | `task_complete` | اكتمال المهمة |
| WaitingInput | `waiting_input` | انتظار إدخال المستخدم |
| Error | `error` | حدوث خطأ |
| Custom | `custom` | مخصص |

يُهيّأ كل نوع بشكل مستقل: قراءة صوتية أو لا، القنوات.

## القنوات والأشكال

يحدد شكل التوزيع (`NotifForm`) القنوات النشطة:

| الشكل | القنوات |
| --- | --- |
| Full | TTS + منبثقة + صوت + صندوق |
| PopupOnly | منبثقة + صندوق |
| InboxOnly | صندوق |
| SoundOnly | صوت |

> الإشعارات المنبثقة تُحفظ في الصندوق ويمكن مراجعتها لاحقًا.

## محركات TTS

ثلاثة `TtsBackend`:

| المحرك | الوصف |
| --- | --- |
| CrossPlatform | متعدد المنصات (افتراضي) |
| MacSay | `say` الأصلي في macOS |
| WebSpeech | Web Speech API للمتصفح |
