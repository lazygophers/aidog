# تكوين الإشعارات

توجد الإعدادات (`NotificationSettings`) ضمن الإعدادات → الإشعارات. تتحكم بالمبدّئ الرئيسي، TTS، والقنوات لكل نوع.

## المبدّئات العامة

| الحقل | الافتراضي | الوصف |
| --- | --- | --- |
| `enabled` | true | المبدّئ الرئيسي. إيقاف = تجاوز كل توزيع |
| `tts_enabled` | true | مبدّئ TTS الرئيسي. AND مع `tts` لكل نوع (كلاهما يعمل للقراءة) |
| `tts_backend` | CrossPlatform | محرك TTS (CrossPlatform / MacSay / WebSpeech) |

## تكوين لكل نوع (per_type)

`per_type` خريطة `NotifType → TypeSetting`؛ كل نوع يُهيّأ باستقلالية:

| الحقل | الوصف |
| --- | --- |
| `tts` | هل يُقرأ هذا النوع صوتيًا (AND مع `tts_enabled` العام) |
| `form` | شكل التوزيع (Full / PopupOnly / InboxOnly / SoundOnly) |

المفاتيح المفقودة تُعامل ككل القنوات + Full.

## متغيرات القالب

تدعم قوالب العنوان/النص المتغيرات، تُستبدل عند التوزيع (`substitute_vars`):

- `{project}` — اسم المشروع
- `{status}` — الحالة
| متغيرات سياق أخرى

مثال: قالب `{project} · {status}` → يُملأ عند التوزيع.

## إشعار اختبار

تحتوي صفحة الإعدادات زر «اختبار» (`notification_test`) يوزّع إشعار اختبار وفق التكوين الحالي للتحقق من القنوات.

→ التالي: [تكامل Hook](/ar/notifications/hooks)
