# Create & Manage Groups

Groups are the routing core of AiDog — organize platforms by purpose and control request routing with rules.

## Creating a Group

1. Go to the **"Groups"** page
2. Click **"+ Add Group"**
3. Enter a name and path
4. Select platforms to include
5. Choose a routing mode

:::tip{title="Auto Group"}
When you add a platform, AiDog automatically creates a corresponding group. You can also create more groups manually.
:::

## Routing Modes

### Failover

The default mode. Platforms are used in order. When the current platform fails, the next one is tried automatically. Ideal for primary/backup setups.

```
Request → Platform A (primary) → Success → Return
                ↓ (failed)
         Platform B (backup) → Success → Return
```

### Load Balance

Requests are distributed across platforms by weight. Ideal for multi-platform traffic splitting.

| Platform | Weight | Expected Traffic |
|----------|--------|-----------------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## Timeout Settings

Each group can independently set timeout values, overriding global settings:

- **Request Timeout** — Maximum time to wait for an upstream response
- **Connect Timeout** — Maximum time to establish a connection

Leave empty to use global timeout settings.

## Managing Platforms

Within a group, you can:
- **Drag to reorder** — Adjust priority (determines primary/backup order in failover mode)
- **Enable/Disable** — Temporarily skip a platform
- **Set weights** — Request ratio for load balance mode
- **Model mapping** — Group-level model mapping, takes precedence over platform-level

## One-Click Launch Commands

Each group card provides two one-click copy buttons:

- **Claude** — Copies the Claude Code launch command for this group, including the settings file path
- **Codex** — Copies the Codex launch command for this group, including the profile

Paste the command in your terminal to immediately use the group's routing.

## Health Status

Group cards display health status indicators for their associated platforms:

- 🟢 All successful — Green
- 🟡 Some failures — Yellow
- 🔴 All failed — Red

## Edit & Delete

- Click a group card to enter edit mode. Click "Save" after changes
- Deleting a group does not delete its platforms — it only removes the association
- Drag group cards in the list to adjust display order
