# قواعد التوجيه

تحدد قواعد التوجيه كيف يطابق AiDog الطلبات الواردة بالمجموعة والمنصة المناسبتين.

## تدفق المطابقة

```
طلب وارد → كشف الواجهة → مطابقة المجموعة → اختيار المنصة → ربط النموذج → التحويل
```

## مطابقة المجموعة

تُطابَق الطلبات بالمجموعات حسب الأولوية:

1. **مطابقة المسار** — مسار الطلب يطابق بادئة مسار المجموعة المُعدّة
2. **مطابقة اسم النموذج** — اسم النموذج في جدول ربط مجموعة
3. **المجموعة الافتراضية** — تُستخدم عند عدم وجود تطابق
4. **المجموعة الأولى** — تُستخدم عند عدم وجود مجموعة افتراضية

## اختيار المنصة

داخل المجموعة، تُختار المنصات حسب وضع التوجيه:

### وضع التبديل عند الفشل

يختار أول منصة متاحة. عند الفشل، يجرب التالي تلقائيًا. إذا فشلت كل المنصات، يُرجع خطأً.

### وضع موازنة الحمل

يختار منصة عشوائيًا حسب الوزن. التبديل عند الفشل يُطبّق أيضًا.

## التبديل عند الفشل

يُطلَق التبديل التلقائي بـ:

- تحديد المعدل (429)
- خطأ الخادم (5xx)
- انتهاء مهلة الاتصال
- خطأ الشبكة

## سلسلة ربط النموذج

ترتيب حل اسم النموذج:

1. **ربط مستوى المجموعة** — إذا عرّفت المجموعة روابط
2. **ربط مستوى المنصة** — إذا عرّفت المنصة روابط
3. **القيمة الأصلية** — استخدم اسم النموذج الأصلي من الطلب

```
العميل يطلب: gpt-4o
  → ربط المجموعة: gpt-4o → deepseek-chat
  → ربط المنصة: (لا شيء)
  → أُرسل إلى المنصة: deepseek-chat
```
