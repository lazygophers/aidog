# قواعد التوجيه

تحدد قواعد التوجيه كيف يطابق AiDog الطلبات الواردة بالمجموعة والمنصة المناسبة.

## تدفق المطابقة

```
طلب وارد ← كشف البروتوكول ← مطابقة المجموعة ← اختيار المنصة ← تعيين النموذج ← التوجيه
```

## مطابقة المجموعة

يتم مطابقة الطلبات للمجموعات حسب الأولوية:

1. **مطابقة اسم النموذج** — اسم النموذج المطلوب موجود في جدول تعيين مجموعة
2. **المجموعة الافتراضية** — تُستخدم عند عدم وجود تطابق
3. **المجموعة الأولى** — تُستخدم عند عدم تحديد مجموعة افتراضية

## اختيار المنصة

ضمن مجموعة، يتم اختيار المنصات حسب الاستراتيجية:

1. التحقق من تفعيل المنصة
2. التحقق من صلاحية مفتاح API
3. اختيار منصة واحدة حسب الاستراتيجية
4. إرسال الطلب

إذا أعادت المنصة المختارة خطأ (مثل 429 حد السرعة)، يحاول AiDog تلقائياً المنصة التالية (ال failover).

## Failover

المنصات ضمن مجموعة مرتبة. عند فشل المنصة الحالية:

1. تسجيل الفشل
2. المحاولة التلقائية للمنصة التالية
3. إذا فشلت جميع المنصات، إرجاع خطأ للعميل

## سلسلة تعيين النماذج

ترتيب حل أسماء النماذج:

1. **التعيين على مستوى المجموعة** — إذا كانت المجموعة تحدد تعيينات
2. **التعيين على مستوى المنصة** — إذا كانت المنصة تحدد تعيينات
3. **القيمة الأصلية** — استخدام اسم النموذج الأصلي من الطلب

```
العميل يطلب: gpt-4o
  ← تعيين المجموعة: gpt-4o → deepseek-chat
  ← تعيين المنصة: (لا يوجد)
  ← أُرسل إلى المنصة: deepseek-chat
```
