# القواعد الواردة

القواعد الواردة تُنفَّذ **قبل وصول الطلب إلى المنصة الصاعدة**، على جسم الطلب (messages, system, model إلخ). يطبّق المحرك 5 أنواع واردة بترتيب ثابت:

```
RequestFilter → SensitiveWord → Redaction → ContentFilter → DynamicInjection
```

## request_filter — تصفية حقل الطلب

يصفّي الطلبات حسب قيمة الحقل. استخدام نموذجي: **قائمة model المسموح/الممنوع**.

- `match_type`: طريقة المطابقة
- `pattern`: نمط قيمة الحقل (مثل `gpt-4o`)
- `action`: `warn` (تسجيل) / `block` (رفض الطلب)

مثال: حظر نموذج → `block` عند المطابقة؛ الطلب مرفوض ومُسجَّل (`proxy_log.blocked_by`).

## sensitive_word — حظر كلمة حساسة

`pattern` هو الكلمة نفسها (ليس regex). عند المطابقة يُطبَّق `action`:

- `block`: رفض الطلب
- `mask`: استبدال بعنصر نائب

الكاشفات المدمجة تغطي أنماط الأسرار الشائعة (انظر [الإعدادات المسبقة المدمجة](/ar/middleware/builtin-presets)).

## redaction — إخفاء

يستبدل المعلومات الحساسة في النص. يُستخدم مع regex المدمج (مفتاح API، بريد، هاتف) أو `pattern` مخصص:

- `config.replacement`: نص الاستبدال (افتراضي `***`)
- الجزء الحساس المطابق → يُستبدل ثم يُوجَّه

## content_filter — تصفية المحتوى

يطابق محتوى الطلب بـ regex؛ عند المطابقة يحظر أو يعيد الكتابة. مفيد لحجب مواضيع أو حقن سياسات أمان.

## dynamic_injection — حقن ديناميكي

يحقن محتوى ديناميكيًا. ثلاث نقاط حقن (`config`):

- **system**: حقن رسالة system (هوية، سياسة أمان)
- **header**: حقن/إعادة كتابة ترويسات HTTP
- **body**: حقن/إعادة كتابة حقول جسم الطلب

مثال: إلحاق prompt نظامي بكل طلب → `dynamic_injection` + `config.inject = system`.

## تدقيق الحظر

المطابقات `action = block` تُكتب في `proxy_log`:

- `blocked_by`: مُعرّف القاعدة (`rule_type#id name`)
- `blocked_reason`: سبب مقروء

الطلبات المحظورة **لا تُحتسب رسومًا** (`est_cost = 0`) لكنها تحتفظ بصف تدقيق كامل.
