# التهيئة والنطاق

## تغطية ثلاثية المستويات

تُطبَّع القواعد حسب `scope`؛ عند التنفيذ **المستوى الأخير يغطي السابق**:

```
global (افتراضي عام) → group (مستوى مجموعة) → platform (مستوى منصة)
```

| scope | scope_ref | يُطبَّق على |
| --- | --- | --- |
| `global` | فارغ | كل الطلبات |
| `group` | اسم المجموعة | طلبات تحت تلك المجموعة |
| `platform` | اسم المنصة | الطلبات الموجَّهة لتلك المنصة |

ضمن `rule_type` واحد، نطاق أكثر تحديدًا يغطي نطاقًا أعمّ. مثال: حظر نموذج عامًا لكن السماح به على منصة → قاعدة المنصة تغطي العامة.

## واجهة الواجهة الأمامية

نقاط دخول تهيئة الوسائط:

1. **الافتراضي العام**: صفحة الإعدادات → تبويب الوسائط (`AppSettings`)
2. **مستوى المجموعة**: مضمَّن في صفحة تحرير المجموعة
3. **مستوى المنصة**: مضمَّن في صفحة تحرير المنصة

كل نقطة تتيح عرض/إضافة/تحرير/تعطيل/حذف؛ النطاق يُربط تلقائيًا بالسياق.

## محرر القواعد

لكل قاعدة:

- **النوع** (`rule_type`): واحد من 8
- **المطابقة** (`match_type` + `pattern`): contains / regex إلخ
- **الإجراء** (`action`): warn / block / mask / inject إلخ
- **الإعداد الموسّع** (`config`): JSON، يعتمد على النوع
- **الأولوية** (`priority`): الأعلى أولًا
- **مُفعّل** (`enabled`)

## الأداء والأمان

- regex يُترجم عند الإقلاع ويُخزَّن؛ صفر تكلفة لكل طلب
- حد حجم DFA ≤ 1 MiB (`REGEX_DFA_SIZE_LIMIT`)، يمنع ReDoS
- حد حجم مصدر regex ≤ 1 MiB (`REGEX_SIZE_LIMIT`)
