# القواعد الصادرة

القواعد الصادرة تُنفَّذ **بعد عودة الاستجابة من المنصة الصاعدة، قبل التوجيه للعميل**. يطبّق المحرك 3 أنواع صادرة بترتيب ثابت:

```
ResponseOverride → Rectifier → ErrorRule
```

## response_override — إعادة كتابة الاستجابة

يعيد كتابة أجسام الاستجابة الناجحة. يطابق النص بـ `pattern`؛ عند المطابقة يستبدل بمحتوى `config`. مفيد لتوحيد تنسيقات الحقول أو إخفاء أجزاء حساسة.

## rectifier — مُقوّم

يصلح عيوب التنسيق في استجابات المنصة الصاعدة:

- **تدفق SSE**: يكمل بادئات `data:` المفقودة، حدود الأحداث
- **JSON**: يصلح الاقتطاع/الهروب غير القانوني
- **الترميز**: يوحّد الترميز
- **حقل مفقود**: يكمل الحقول المطلوبة

للتوافق مع المنصات غير القياسية.

## error_rule — تصنيف الخطأ

يصنّف استجابات الخطأ الصاعدة (غير 2xx):

- **إعادة محاولة**: يُحكم قابلًا لإعادة المحاولة (429 / 503) → يشغّل إعادة محاولة تجاوز الفشل بين المنصات
- **قاطع دائرة**: يُحكم عطل منصة → يُحتسب في إخفاقات القاطع (انظر [الجدولة وقاطع الدائرة](/ar/groups/scheduling))
- **إعادة كتابة الحالة**: يربط كود حالة المنصة بكود مرئي للعميل

`config` يمكنه تحديد `retry` / `circuit_break` / `status_override`. مجموعة افتراضية من `error_rules` مدمجة (انظر [الإعدادات المسبقة المدمجة](/ar/middleware/builtin-presets)).

## التدفق قطعة قطعة

لاستجابات SSE، يطبّق المحرك القواعد الصادرة **قطعة بقطعة** (`rewrite_chunk_text`) دون تخزين كامل الاستجابة:

- كل قطعة SSE تُعالَج عند الوصول
- تُحفظ دلالات التدفق؛ العميل يستقبل الرموز لحظيًا
- حجم regex محدود (DFA ≤ 1 MiB)، يمنع ReDoS
