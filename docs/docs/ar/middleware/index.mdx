# محرك قواعد الوسائط

يحتوي AiDog على محرك قواعد وسائط قابل للتهيئة للطلبات/الاستجابات، ينفذ القواعد في مرحلتين من سلسلة توجيه الوكيل — **واردة** (طلب داخل) و**صادرة** (استجابة عائدة): تسوية، إعادة كتابة، إخفاء، حقن، تصفية الكلمات الحساسة، كشف الأخطاء، وقاطع الدائرة. الكل قابل للتهيئة عبر الواجهة دون كود.

## البنية

| الميزة | الوصف |
| --- | --- |
| تخزين أحادي الجدول | كل القواعد في جدول SQLite `middleware_rule` |
| Engine مفرد | `MiddlewareEngine` يترجم القواعد إلى regex عند الإقلاع، يطابق حسب الأولوية لكل طلب |
| تركيب ثنائي الاتجاه | وارد (إعادة كتابة جسم الطلب) + صادر (إعادة كتابة جسم الاستجابة/تدفق SSE) |
| تغطية ثلاثية المستويات | `global` → `group` → `platform`؛ المستوى الأخير يغطي القواعد المماثلة الاسم |
| تدفق قطعة قطعة | استجابات SSE تُعالَج قطعة بقطعة دون تخزين مؤقت |
| إعدادات مسبقة مدمجة | إخفاء مفتاح API/بريد/هاتف + قواعد تصنيف أخطاء افتراضية |

## أنواع القواعد

يُعرّف المحرك 8 أنواع `RuleType`، مجمّعة حسب مرحلة التنفيذ:

| النوع | المرحلة | الغرض |
| --- | --- | --- |
| `request_filter` | وارد | تصفية حقل الطلب (مثل قائمة model المسموح/الممنوع) |
| `sensitive_word` | وارد | حظر كلمة حساسة (pattern هو الكلمة) |
| `redaction` | وارد | إخفاء قيمة الحقل |
| `content_filter` | وارد | تصفية المحتوى |
| `dynamic_injection` | وارد | حقن ديناميكي (system / header / body) |
| `response_override` | صادر | إعادة كتابة جسم الاستجابة |
| `rectifier` | صادر | مُقوّم (SSE / JSON / ترميز / حقل مفقود) |
| `error_rule` | صادر | تصنيف الخطأ (إعادة محاولة/قاطع/إعادة كتابة الحالة) |

## حقول القاعدة

كل قاعدة (جدول `middleware_rule`):

| الحقل | الوصف |
| --- | --- |
| `name` / `description` | الاسم والوصف |
| `rule_type` | نوع القاعدة (انظر الجدول) |
| `scope` | النطاق: `global` / `group` / `platform` |
| `scope_ref` | مرجع النطاق (اسم group/platform؛ فارغ لـ global) |
| `match_type` | طريقة المطابقة (`contains` / `regex` إلخ) |
| `pattern` | نمط المطابقة |
| `action` | الإجراء عند المطابقة (`warn` / `block` / `mask` / `inject` إلخ) |
| `config` | إعداد JSON موسّع |
| `priority` | الأولوية (الأعلى يُنفّذ أولاً) |
| `enabled` | مُفعّل |
| `is_builtin` | إعداد مدمج (لا يُحذف، يُعطّل) |
