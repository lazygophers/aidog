# إضافة منصة

المنصات هي المفهوم الأساسي في AiDog — كل منها يمثل نقطة وصول لخدمة API ذكاء اصطناعي.

![AiDog — Add Platform](/screenshots/add-platform.png)


## خطوات الإضافة

1. انتقل إلى صفحة **"المنصات"**
2. انقر على زر **"+"** في أعلى اليمين
3. املأ التكوين:

### التكوين الأساسي

| الحقل | الوصف | المثال |
|-------|-------------|---------|
| الاسم | اسم العرض | `OpenAI الخاص بي` |
| Base URL | عنوان URL الأساسي للـ API (مع بادئة الإصدار) | `https://api.openai.com/v1` |
| API Key | مفتاح API للمنصة | `sk-...` |
| البروتوكول | نوع بروتوكول الطلب | `openai_chat` |

### اختيار البروتوكول

اختر البروتوكول المطابق للمنصة المستهدفة. يتضمن AiDog أكثر من 50 إعداداً مسبقاً للمنصات يملأ البروتوكول تلقائياً.

البروتوكولات الشائعة:
- **openai_chat** — Chat Completions المتوافق مع OpenAI (الأكثر شمولية)
- **openai_completions** — Text Completions لـ OpenAI (قديم)
- **openai_responses** — Responses API لـ OpenAI (يستخدمه Codex)
- **anthropic** — بروتوكول Anthropic Claude الأصلي
- **gemini** — بروتوكول Google Gemini الأصلي

راجع [أنواع البروتوكولات](/platforms/protocols) للتفاصيل.

### خيارات متقدمة

- **تعيين النماذج** — تعيين أسماء النماذج المطلوبة إلى النماذج الفعلية للمنصة (راجع [تعيين النماذج](/platforms/model-mapping))
- **تفعيل/تعطيل** — إيقاف منصة مؤقتاً دون حذفها

## تعديل المنصة

انقر على بطاقة المنصة لعرض التفاصيل وتعديل الإعدادات. انقر على "حفظ" بعد التعديلات.

## حذف المنصة

انقر على زر الحذف على بطاقة المنصة. السجلات والإحصائيات المرتبطة يتم الاحتفاظ بها.

## الأسئلة الشائعة

### كيفية ملء Base URL

Base URL هو المسار الأساسي الكامل متضمناً بادئة الإصدار:

| المنصة | Base URL |
|------------|----------|
| OpenAI | `https://api.openai.com/v1` |
| Anthropic | `https://api.anthropic.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |

:::caution{title="تنبيه"}
لا تضف `/chat/completions` في نهاية Base URL. يتولى AiDog ذلك تلقائياً.
:::

### أمان مفتاح API

يتم تخزين مفاتيح API في قاعدة بيانات SQLite المحلية ولا يتم رفعها إلى أي خادم أبداً.
