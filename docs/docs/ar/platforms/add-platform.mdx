# إضافة منصة

المنصات هي المفهوم الأساسي في AiDog — كل منها يمثل نقطة نهاية لخدمة AI API.

![AiDog — إضافة منصة](/screenshots/add-platform.png)

## خطوات الإضافة

1. اذهب إلى صفحة **«المنصات»**
2. انقر زر **«+**» في الأعلى يمينًا
3. اختر منصة من قائمة المُعدّات (أو املأ يدويًا)
4. أدخل مفتاح API وانقر حفظ

:::tip{title="مجموعة تلقائية"}
بعد حفظ منصة، يُنشئ AiDog تلقائيًا مجموعة لها، فتستطيع استخدامها فورًا دون إعداد يدوي.
:::

## الإعداد الأساسي

| الحقل | الوصف | مثال |
|-------|-------------|---------|
| الاسم | اسم العرض | `OpenAI الخاص بي` |
| عنوان URL الأساسي | عنوان URL الأساسي للـ API (مع بادئة الإصدار) | `https://api.openai.com/v1` |
| مفتاح API | مفتاح API الخاص بالمنصة | `sk-...` |

### مُعدّات المنصة

يضم AiDog أكثر من 50 مُعدّ منصة مدمج (OpenAI، Anthropic، Gemini، DeepSeek، Qwen، والمزيد). اختيار مُعدّ يملأ كل حقول الإعداد تلقائيًا — فقط أدخل مفتاح API.

بعض المُعدّات مُعلّمة بـ«Coding Plan» — تُهيّئ استعلامات الحصة تلقائيًا.

## وضع المرور المباشر

بعض أنواع المنصات تدعم وضع «المرور المباشر»: يمرّر AiDog ترويسات طلب العميل ونصه كما هي، دون أي تحويل بروتوكول أو حقن مصادقة.

- **عنوان المنبع** — أدخل عنوان URL الجذري للمنصة (مثل `https://api.anthropic.com`)، دون بادئة الإصدار
- **مفتاح API** — اختياري في وضع المرور المباشر؛ يمكن للعميل حمل مصادقته الخاصة

مثالي للسيناريوهات التي يجب فيها الحفاظ على مصادقة العميل (مثل اشتراكات OAuth).

## نقاط نهاية متعددة

يمكن أن يكون لكل منصة نقاط نهاية متعددة، لكل منها عنوانها ونوع واجهتها. يختار AiDog تلقائيًا نقطة النهاية المطابقة بناءً على طلب العميل.

على سبيل المثال، يمكن للمنصة نفسها دعم واجهة الدردشة وواجهة مساعد البرمجة.

## إعداد النماذج

### النماذج الافتراضية

في إعدادات المنصة، حدد أي نموذج يُستخدم افتراضيًا لأنواع الطلبات المختلفة. يطابق AiDog تلقائيًا النموذج الأنسب بناءً على اسم النموذج المطلوب.

### جلب قائمة النماذج

انقر زر «جلب النماذج» للحصول تلقائيًا على النماذج المتاحة من المنصة.

### تحويل النماذج

حوّل أسماء النماذج المطلوبة إلى أسماء النماذج الفعلية المدعومة من المنصة. راجع [تحويل النماذج](/ar/platforms/model-mapping).

## تكوين استعلام الرصيد

للمنصات التي تستخدم نظام NewAPI، اضبط استعلامات رصيد منفصلة:

- **عنوان الاستعلام** — منفصل عن عنوان API
- **رمز الاستعلام** — يُؤخذ من لوحة التحكم (مختلف عن مفتاح API)
- **User ID** — اختياري

راجع [الرصيد والحصص](/ar/stats/balance).

## الميزانية اليدوية

للمنصات التي لا تدعم الاستعلام التلقائي عن الرصيد، اضبط حدود ميزانية يدوية:

- **الإجمالي** — حد الإنفاق الكلي
- **نافذة متدحرجة** — حد الإنفاق ضمن نافذة زمنية متدحرجة
- **نافذة ثابتة** — حد إنفاق يُعاد تعيينه وفق جدول ثابت
- **يومي** — حد الإنفاق اليومي

عند الاستنزاف، يوقف AiDog التمرير إلى تلك المنصة ويفشل تلقائيًا. راجع [الرصيد والحصص](/ar/stats/balance).

## اختبار النموذج

بعد حفظ منصة، اختبر اتصالها:

1. انقر زر «اختبار» على بطاقة المنصة
2. اختر وضع اختبار: سريع / دفعة / مخصص
3. اعرض النتائج: زمن الاستجابة، عدد التوكنز، نجاح أو فشل

راجع [اختبار النموذج](/ar/platforms/model-testing).

## قائمة المنصات

تعرض صفحة المنصات كل بطاقات المنصات المُضافة، كلٌ يُظهر:

- اسم المنصة وشعارها
- مؤشر حالة الصحة (أخضر/أصفر/أحمر)
- الرصيد أو استخدام الحصة
- عدد طلبات اليوم ومعدل النجاح

تدعم البحث بالبينين لتحديد موقع منصة بسرعة.

## عمليات أخرى

- **تفعيل/تعطيل** — إيقاف منصة مؤقتًا دون حذفها
- **تحرير** — انقر بطاقة منصة لعرض التفاصيل وتعديل الإعدادات
- **حذف** — انقر زر الحذف. تُحفظ السجلّات والإحصاءات المرتبطة
- **السحب لإعادة الترتيب** — اسحب البطاقات في القائمة لضبط ترتيب العرض

## الأسئلة الشائعة

### كيف أملأ عنوان URL الأساسي؟

عنوان URL الأساسي هو مسار الأساس الكامل متضمنًا بادئة الإصدار:

| المنصة | عنوان URL الأساسي |
|----------|----------|
| OpenAI | `https://api.openai.com/v1` |
| Anthropic | `https://api.anthropic.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| Qwen | `https://dashscope.aliyuncs.com/compatible-mode/v1` |

:::caution{title="ملاحظة"}
لا تُلحق `/chat/completions` بعنوان URL الأساسي. يتولى AiDog ذلك تلقائيًا.
:::

### أمان مفتاح API

تُخزَّن مفاتيح API محليًا ولا تُرفع أبدًا إلى أي خادم.
