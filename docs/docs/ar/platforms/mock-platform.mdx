# منصة Mock

منصات mock لا ترسل طلبات AI حقيقية — بل تُرجع استجابات مُعدّة مسبقًا. مثالية للتطوير والتنقيح واختبار قواعد التوجيه والتحقق من تكامل العميل.

## إنشاء منصة mock

1. اذهب إلى صفحة **«المنصات»**
2. انقر زر **«+»**
3. اختر **«Mock»** من قائمة أنواع المنصات
4. لا حاجة لـ API Key أو Base URL
5. حفظ

بعد الحفظ، ينشئ AiDog مجموعة مقابلة تلقائيًا. وجّه عميلك إلى مسار تلك المجموعة لاستخدام الـmock.

## إعداد الاستجابة

خصّص معاملات الاستجابة:

| المعامل | الوصف | الافتراضي |
|-----------|-------------|---------|
| نص الاستجابة | نص الاستجابة المحاكى | `Hello from mock` |
| رمز الحالة | رمز حالة HTTP | `200` |
| تأخير الاستجابة | تأخير محاكى بالميلي ثانية | `0` |
| توكنز الدخل | عدد توكنز الدخل المحاكى | `10` |
| توكنز الخرج | عدد توكنز الخرج المحاكى | `20` |
| توكنز الكاش | توكنز الكاش المحاكاة | `0` |
| سبب الانتهاء | سبب الانتهاء المحاكى (مثال stop) | `stop` |
| البث | الإرجاع كتدفق (اتباع الطلب أو فرض تشغيل/إيقاف) | اتباع الطلب |

## محاكاة الأخطاء

يدعم محاكاة سيناريوهات خطأ متنوعة:

| وضع الخطأ | الوصف |
|------------|-------------|
| بلا خطأ | يُرجع استجابة طبيعية |
| خطأ HTTP | يُرجع رمز حالة خطأ HTTP محدد |
| Rate Limit (429) | يحاكي تحديد المعدل |
| انتهاء المهلة | يحاكي انتهاء مهلة الطلب |

مفيد لاختبار معالجة أخطاء العميل أو سلوك التبديل عند الفشل في AiDog.

## حالات الاستخدام

- **التطوير** — اختبر تكامل العميل بدون مفاتيح API حقيقية
- **اختبارات التوجيه** — تحقق من قواعد مطابقة المجموعات والربط
- **اختبار التبديل** — حاكِ أخطاءً لمراقبة التبديل التلقائي
- **اختبار الأداء** — اضبط التأخيرات لاختبار السلوك تحت استجابات بطيئة
