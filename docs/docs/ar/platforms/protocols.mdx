# أنواع البروتوكولات

يدعم AiDog بروتوكولات متعددة لـ API الذكاء الاصطناعي للكشف عن الطلبات الواردة وتحويل الطلبات الصادرة.

## البروتوكولات المدعومة

### سلسلة OpenAI

| البروتوكول | الوصف | نقطة النهاية |
|----------|-------------|----------|
| `openai_chat` | Chat Completions API | `/v1/chat/completions` |
| `openai_completions` | Text Completions API (قديم) | `/v1/completions` |
| `openai_responses` | Responses API (Codex) | `/v1/responses` |

### Anthropic

| البروتوكول | الوصف | نقطة النهاية |
|----------|-------------|----------|
| `anthropic` | Claude Messages API | `/v1/messages` |

### Google

| البروتوكول | الوصف | نقطة النهاية |
|----------|-------------|----------|
| `gemini` | Gemini Generate Content API | `/v1beta/models/{model}:generateContent` |

### أخرى

| البروتوكول | الوصف |
|----------|-------------|
| `minimax` | بروتوكول MiniMax الأصلي |
| `glm` | بروتوكول Zhipu GLM الأصلي |

## الكشف التلقائي

يكشف AiDog تلقائياً عن البروتوكول الوارد بناءً على مسار الطلب:

- `/v1/chat/completions` ← `openai_chat`
- `/v1/completions` ← `openai_completions`
- `/v1/responses` ← `openai_responses`

يمكنك خدمة أنواع مختلفة من العملاء عبر نفس عنوان الوكيل.

## تحويل البروتوكول

يمكن أن تختلف البروتوكولات الواردة والصادرة. يتولى AiDog التحويل تلقائياً:

1. تحليل الطلب الوارد (البروتوكول المكتشف)
2. التحويل إلى التنسيق الداخلي
3. التحويل إلى بروتوكول المنصة المستهدفة
4. إرسال الطلب

مثال: العميل يرسل `openai_chat` ← يحول AiDog إلى بروتوكول `anthropic` ← يرسل إلى Claude.

## الإعدادات المسبقة للمنصات

يتضمن AiDog أكثر من 50 إعداداً مسبقاً مدمجاً، يحتوي كل منها على:
- Base URL الافتراضي
- البروتوكول الافتراضي
- قائمة النماذج المدعومة

اختيار إعداد مسبق يملأ هذه القيم تلقائياً.
