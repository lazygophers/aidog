# اختبار النماذج

يوفر AiDog ميزة اختبار نماذج مدمجة للتحقق من اتصال المنصة وتوفر النماذج دون أدوات خارجية.

## الاستخدام

في صفحة **«المنصات»**، انقر زر «اختبار» على بطاقة منصة لفتح لوحة الاختبار.

## أوضاع الاختبار

| الوضع | الوصف |
|------|-------------|
| **اختبار سريع** | يرسل طلبًا قصيرًا بالنموذج الافتراضي للمنصة لفحص الاتصال السريع |
| **نموذج واحد** | يختبر نموذجًا محددًا |
| **اختبار دفعي** | يختبر عدة نماذج بالتسلسل للتحقق من التوفر |
| **اختبار عشوائي** | يختبر كل النماذج المتاحة في المنصة واحدًا تلو الآخر |
| **اختبار مخصص** | اختر نموذجًا وأدخل prompt مخصص وحد أقصى للتوكنز |

## نتائج الاختبار

تُظهر كل نتيجة:

- **اسم النموذج** — النموذج المستخدم في الاختبار
- **الحالة** — نجاح أو فشل
- **زمن الاستجابة** — الزمن الكلي من الإرسال حتى الاستلام الكامل
- **استخدام التوكنز** — أعداد توكنز الدخل/الخرج
- **محتوى الاستجابة** — النص المُرجع (قابل للطي)

## حالات الاستخدام

- **بعد إضافة منصة** — تحقق من صحة API Key و Base URL
- **بعد تغيير endpoints** — تأكد أن الـendpoint الجديد يعمل
- **بعد ربط النموذج** — تأكد أن النموذج الهدف متوفر
- **استكشاف الأعطال** — عند فشل الطلبات المُوكّلة، اختبر المنصة المعنية أولاً

## ملاحظات

- الاختبار يستخدم API Key و endpoint المنصة المُعدّين، ويستهلك توكنز حقيقية
- اختبارات منصة mock لا ترسل طلبات حقيقية
- أوضاع الدفعي والعشوائي تختبر عدة نماذج بالتسلسل وتستغرق وقتًا أطول
