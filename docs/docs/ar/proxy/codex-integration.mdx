# تكامل Codex

يدعم AiDog أصليًا OpenAI Codex CLI و Claude Code وغيرها من مساعدات AI مع محررات إعداد مدمجة.

## OpenAI Codex

### الإعداد

اضبط التالي في ملف إعداد Codex ‎`~/.codex/config.toml`:

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

### صفحة إعدادات Codex

يوفر AiDog محرر إعداد Codex مدمج في **«الإعدادات → Codex»**:

- حرر إعدادات ‎`~/.codex/config.toml` مباشرة
- لا حاجة لتحديد وتحرير الملفات يدويًا
- تسري التغييرات فورًا بعد الحفظ

### ملاحظات

- يستخدم Codex **Responses API** (‎`/v1/responses`). يكتشف AiDog ويوجّه تلقائيًا.
- تأكد أن المنصة المستهدفة لها endpoint ببروتوكول `openai_responses` مضبوط.
- أضف روابط النماذج على مستوى المنصة أو المجموعة حسب الحاجة.

## Claude Code

### تصدير الإعداد بنقرة واحدة

في تبويب **«الإعدادات → Claude Code»** في AiDog، انقر «تصدير الإعداد». يكتب AiDog عنوان البروكسي تلقائيًا في ملف إعداد Claude Code (‎`~/.claude.json`).

### الإعداد اليدوي

يمكنك أيضًا ضبط متغيرات البيئة:

```bash
export ANTHROPIC_BASE_URL="http://127.0.0.1:9876"
export ANTHROPIC_API_KEY="any-value"
```

### صفحة إعدادات Claude Code

يوفر AiDog محرر إعدادات مخصص في **«الإعدادات → Claude Code»**:

- **تحرير الإعداد** — حرر إعدادات Claude Code (النموذج، الصلاحيات، الصندوق الرملي، الإضافات، hooks، StatusLine، إلخ)
- **استيراد من Claude Code** — استيراد بنقرة واحدة لإعداد Claude Code الموجود
- **تصدير إلى Claude Code** — كتابة عنوان بروكسي AiDog في إعداد Claude Code
- **مزامنة إعدادات المجموعة** — ينشئ تلقائيًا ملفات إعداد مستقلة لكل مجموعة لتسهيل التبديل

### ملاحظات

- يستخدم Claude Code **البروتوكول الأصلي Anthropic**
- تأكد أن المنصة المستهدفة تدعم بروتوكول Anthropic أو اضبط التحويل
- يوفر AiDog مسارًا مخصصًا ‎`/claude/v1/messages` لـ Claude Code

## مساعدات أخرى

أي أداة تدعم عناوين API مخصصة يمكنها استخدام AiDog كبروكسي:

1. اضبط عنوان API الأساسي إلى ‎`http://127.0.0.1:9876`
2. اضبط مفتاح API لأي قيمة
3. تأكد أن AiDog لديه منصة وبروتوكول مطابقان مضبوطان
