# التوجيه المتقدم

أتقن قدرات التوجيه المتقدمة في AiDog للتحكم الدقيق في حركة البيانات.

## موازنة الحمل متعدد المنصات

أضف منصات متعددة إلى مجموعة مع استراتيجيات توجيه مختلفة:

### متسلسل

المنصات تتلقى الطلبات بالترتيب. مثالي لسيناريوهات الأساسي/الاحتياطي.

### عشوائي

يتم توزيع الطلبات عشوائياً على المنصات. جيد للتوزيع المتساوي.

### مرجح

توزيع الطلبات حسب نسبة الأوزان. مثال:

| المنصة | الوزن | الحركة المتوقعة |
|----------|--------|------------------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## Failover

عند فشل طلب المنصة، يحاول AiDog تلقائياً المنصة التالية في المجموعة:

```
طلب ← المنصة أ (429 حد السرعة)
     ← المنصة ب (نجاح)
     ← إرجاع الاستجابة
```

يتم تشغيل Failover عند:
- HTTP 429 (حد السرعة)
- HTTP 5xx (خطأ الخادم)
- مهلة الاتصال
- أخطاء الشبكة

## أسماء مستعارة للنماذج

إنشاء أسماء مستعارة عبر تعيين النماذج:

| الاسم المستعار | التعيين إلى | الغرض |
|-------|---------|---------|
| `fast` | `gpt-4o-mini` | استجابات سريعة |
| `smart` | `claude-3-opus` | جودة عالية |
| `code` | `deepseek-coder` | توليد الكود |

العملاء يستخدمون الأسماء المستعارة؛ AiDog يوجه إلى المنصات الفعلية بناءً على تكوين المجموعة.

## التوجيه المتسلسل

الجمع بين تعيينات المجموعة والمنصة:

1. العميل يطلب النموذج `smart`
2. المجموعة أ تعين `smart` ← `claude-3-opus`
3. المنصة X تعين `claude-3-opus` ← `claude-3-opus-20240229`
4. أُرسل إلى Anthropic
