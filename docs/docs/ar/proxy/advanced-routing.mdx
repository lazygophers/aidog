# التوجيه المتقدم

أتقن قدرات التوجيه المتقدم في AiDog للتحكم الدقيق بالحركة.

## تجميع متعدد المنصات

أضف منصات متعددة إلى مجموعة بأوضاع توجيه مختلفة. راجع [الإنشاء والإدارة](/ar/groups/manage-groups).

## التبديل عند الفشل

عند فشل منصة، يجرب AiDog التالية تلقائيًا:

```
طلب → المنصة A (429 Rate Limit)
        → المنصة B (نجاح)
        → إرجاع الاستجابة
```

يُطلَق التبديل عند:

- HTTP 429 (Rate Limit)
- HTTP 5xx (خطأ الخادم)
- انتهاء مهلة الاتصال
- أخطاء الشبكة

## موازنة الحمل

تُوزَّع الطلبات بين المنصات حسب الوزن. مثال:

| المنصة | الوزن | الحركة المتوقعة |
|----------|--------|------------------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## أسماء النماذج المستعارة

أنشئ أسماء مستعارة عبر ربط النماذج:

| الاسم المستعار | يُربط بـ | الغرض |
|-------|---------|---------|
| `fast` | `gpt-4o-mini` | استجابات سريعة |
| `smart` | `claude-3-opus` | جودة عالية |
| `code` | `deepseek-coder` | توليد الكود |

يستخدم العملاء الأسماء المستعارة؛ يوجّه AiDog إلى المنصات الفعلية حسب إعداد المجموعة.

## التوجيه المتسلسل

ادمج روابط المجموعة والمنصة لتحويل متعدد الطبقات:

1. العميل يطلب النموذج `smart`
2. المجموعة A تربط `smart` → `claude-3-opus`
3. المنصة X تربط `claude-3-opus` → `claude-3-opus-20240229`
4. أُرسل إلى Anthropic

## المهلة على مستوى المجموعة

كل مجموعة يمكنها ضبط مهلة الطلب ومهلة الاتصال بشكل مستقل، متجاوزة الإعدادات العامة. مفيد عندما تحتاج نماذج أبطأ أوقات انتظار أطول.

راجع [الإنشاء والإدارة](/ar/groups/manage-groups).
