# Codex

اضبط تكامل AiDog مع OpenAI Codex CLI في تبويب **«الإعدادات» → «Codex»**.

## تحرير الإعداد

حرّر إعدادات Codex في `~/.codex/config.toml` مباشرة في AiDog:

- **Provider** — مزود API وعنوان URL الأساسي
- **النموذج** — اسم النموذج المراد استخدامه
- **خيارات أخرى** — كل عناصر الإعداد التي يدعمها Codex

تُكتب التغييرات فورًا في ملف الإعداد — لا حاجة للبحث عن الملفات وتحريرها يدويًا.

## الإعداد السريع

الإعداد الأدنى يتطلب عنوان البروكسي وأي مفتاح API:

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

## إعداد المجموعة

يدعم Codex التبديل بين المجموعات عبر الملفات الشخصية:

1. انقر زر «Codex» على بطاقة مجموعة لنسخ أمر التشغيل
2. الصق في طرفيتك:

```bash
AIDOG_KEY='<group-name>' codex -p '<group-name>' \
  --dangerously-bypass-approvals-and-sandbox -a never
```

يوجّه AiDog الطلبات إلى المجموعة المقابلة بناءً على متغير البيئة `AIDOG_KEY`.

## ملاحظات

- يستخدم Codex واجهة Responses API (`/v1/responses`)، التي يكتشفها AiDog ويوجّهها تلقائيًا
- تأكد أن المنصة المقابلة قد أُضيفت إلى AiDog
- أضف تحويل النماذج في إعدادات المنصة أو المجموعة عند الحاجة
- بادئة المسار `/proxy/v1` في عنوان البروكسي لا يجوز حذفها
