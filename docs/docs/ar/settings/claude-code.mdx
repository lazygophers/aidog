# Claude Code

اضبط تكامل AiDog مع Claude Code في تبويب **«الإعدادات» → «Claude Code»**.

## تحرير الإعداد

حرّر كل إعدادات Claude Code مباشرة في AiDog:

- **النموذج** — يحدد النموذج الذي يستخدمه Claude Code
- **الصلاحيات** — تحكمات قراءة/كتابة الملفات، وصول الشبكة
- **Sandbox** — تكوين sandbox تنفيذ الكود
- **الإضافات** — إدارة إضافات Claude Code
- **Hooks** — خطافات دورة الحياة (PreToolUse / PostToolUse، إلخ)
- **StatusLine** — تكوين عرض شريط الحالة

تُحفظ كل التغييرات فورًا في `~/.claude.json`.

## الاستيراد من Claude Code

انقر «الاستيراد من Claude Code» لقراءة ملف إعداد Claude Code القائم وتحميله في المحرر. مثالي لترحيل الإعدادات القائمة أثناء الإعداد الأولي.

## التصدير إلى Claude Code

انقر «التصدير إلى Claude Code» لكتابة عنوان البروكسي في ملف إعداد Claude Code:

```json
{
  "apiBaseUrl": "http://127.0.0.1:9876"
}
```

بعد التصدير، ستُمرّر كل طلبات Claude Code عبر AiDog.

## مزامنة إعدادات المجموعة

انقر «مزامنة إعدادات المجموعة» لتوليد ملفات إعداد Claude Code مستقلة لكل مجموعة:

- تُحفظ في `~/.aidog/settings.<group-name>.json`
- كل ملف يستخدم مسار بروكسي المجموعة المقابلة
- استخدم مع زر نسخ «Claude» على بطاقات المجموعات

### الاستخدام

1. انقر زر «Claude» على بطاقة مجموعة لنسخ أمر التشغيل
2. الصق في طرفيتك:

```bash
claude --brief --dangerously-skip-permissions \
  --settings ~/.aidog/settings.<group-name>.json
```

## الإعداد اليدوي

بدلاً من ذلك، اضبط عبر متغيرات البيئة:

```bash
export ANTHROPIC_BASE_URL="http://127.0.0.1:9876"
export ANTHROPIC_API_KEY="any-value"
```

## ملاحظات

- يستخدم Claude Code بروتوكول Anthropic الأصلي
- تأكد أن المنصة المستهدفة تدعم بروتوكول Anthropic أو لديها تحويل بروتوكول مُهيّأ
- يوفر AiDog مسارًا مخصصًا لـ Claude Code
- أضف تحويل النماذج في إعدادات المنصة أو المجموعة عند الحاجة
