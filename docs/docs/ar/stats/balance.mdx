# الرصيد والحصص

يتتبع AiDog أرصدة الحسابات وحصص coding plan عبر المنصات، مانحًا إياك رؤية كاملة لإنفاقك.

## الاستعلام التلقائي عن الرصيد

يستعلم AiDog تلقائيًا عن الأرصدة لهذه المنصات:

| المنصة | المحتوى المُستعلم |
|----------|---------------|
| OpenAI | رصيد الحساب |
| DeepSeek | رصيد الحساب |
| StepFun | رصيد الحساب |
| SiliconFlow | رصيد الحساب |
| OpenRouter | الرصيد / الاستخدام |
| Novita | رصيد الحساب |
| Kimi (Moonshot) | رصيد الحساب |
| GLM (Zhipu) | رصيد الحساب |
| MiniMax | رصيد الحساب |

تُحدَّث بيانات الرصيد تلقائيًا وفق جدول. كما يمكنك النقر يدويًا على زر التحديث.

## حصص Coding Plan

بعض المنصات تقدم خطط coding، مثل OpenAI Plus/Team/Enterprise، Zhipu Coding Plan، Kimi Code Plan، إلخ.

يستعلم AiDog ويعرض معلومات الحصة:

- أسماء المستويات
- معدل الاستخدام الحالي
- النسبة المتبقية

المُعدّات المُعلّمة بـ«Coding Plan» تُهيّئ استعلامات الحصة تلقائيًا.

## استعلام رصيد NewAPI

للمنصات التي تستخدم نظام NewAPI (مثل مجمعات API المستضافة ذاتيًا)، يدعم AiDog تكوين استعلام رصيد منفصل:

- **رابط الاستعلام** — منفصل عن رابط API
- **رمز الاستعلام** — يُؤخذ من لوحة التحكم (مختلف عن مفتاح API)
- **User ID** — اختياري

املأ هذه التفاصيل في قسم «تكوين استعلام الرصيد» الخاص بالمنصة.

## الميزانية اليدوية

للمنصات التي لا تدعم الاستعلام التلقائي عن الرصيد، يوفر AiDog حدود ميزانية يدوية.

### الإعداد

في قسم «الميزانية اليدوية» الخاص بالمنصة، انقر «إضافة حد»:

| النوع | الوصف |
|------|-------------|
| الإجمالي | حد الإنفاق الكلي |
| نافذة متدحرجة | حد الإنفاق ضمن نافذة زمنية متدحرجة |
| نافذة ثابتة | حد إنفاق يُعاد تعيينه وفق جدول ثابت |
| يومي | حد الإنفاق اليومي |

### وحدات الميزانية

- **USD** — يُقاس بالتكلفة
- **Tokens** — يُقاس بعدد التوكنز

### مدة النافذة

النافذة المتدحرجة والثابتة تدعم عدة مستويات زمنية: دقيقة، ساعة، يوم، أسبوع، شهر.

### عند استنزاف الميزانية

عند بلوغ أي حد، يوقف AiDog تمرير الطلبات لتلك المنصة (يُعيد خطأ 402) ويُطلق تلقائيًا الفشل للمنصات الأخرى في المجموعة. يستأنف التمرير عند إعادة تعيين النافذة.

## أين تُرى الأرصدة

تُعرض معلومات الرصيد في عدة مواضع:

- **بطاقات المنصات** — رصيد لحظي واستخدام حصة لكل منصة
- **بطاقات المجموعات** — رصيد مُجمّع لمنصات المجموعة
- **شريط النظام** — عرض قابل للتكوين لحالة الرصيد

## صفحة الإحصاءات

في صفحة **«الإحصاءات»**، اعرض البيانات حسب المنصة أو المجموعة أو النموذج:

- عدد طلبات اليوم ومعدل النجاح
- استخدام التوكنز (إدخال / إخراج / كاش)
- التكلفة التقديرية
- مخططات اتجاه الاستخدام (يومي / أسبوعي / شهري)
