# Statistiques d'utilisation

La page **« Statistiques »** fournit un aperçu d'utilisation, une analyse de tendance et des classements par dimension pour toutes les requêtes proxy.

## Plage de temps

Trois préréglages temporels sont disponibles en haut :

| Préréglage | Plage |
|--------|-------|
| Aujourd'hui | Depuis minuit aujourd'hui |
| 7 derniers jours | 7 derniers jours |
| 30 derniers jours | 30 derniers jours |

Toutes les métriques et graphiques se rafraîchissent automatiquement à la sélection d'un préréglage.

## Cartes d'aperçu

Huit métriques clés sont affichées en haut, chacune avec comparaison de période :

| Métrique | Description |
|--------|-------------|
| Requêtes totales | Nombre total de requêtes sur la période |
| Taux de succès | Pourcentage de requêtes réussies (codé couleur) |
| Tokens d'entrée | Total tokens entrée |
| Tokens de sortie | Total tokens sortie |
| Tokens cache | Total tokens de cache hit |
| Taux de cache | Tokens cache en pourcentage des tokens entrée |
| Latence moyenne | Temps de réponse moyen en millisecondes |
| Coût estimé | Coût estimé selon la table tarifaire |

Flèches de comparaison : vert vers le haut = changement positif (ex. taux de succès plus élevé), rouge vers le haut = changement négatif (ex. coût plus élevé).

## Filtres

À côté des préréglages temporels, affinez avec ces filtres :

- **Granularité** : Journalière / Horaire
- **Grouper par** : Plateforme / Modèle / Groupe
- **Groupe** : Filtrer par nom de groupe (recherchable)
- **Modèle** : Filtrer par nom de modèle (recherchable)
- **Type de plateforme** : Filtrer par type de plateforme (recherchable)

Les données se rechargent automatiquement au changement des filtres.

## Graphique de tendance

Un graphique à barres sous les cartes d'aperçu montre les tendances du volume de requêtes :

- L'axe X regroupe par granularité sélectionnée (jour/heure)
- La hauteur des barres représente le nombre de requêtes
- Le dégradé de couleur reflète le ratio d'erreurs (plus rouge = plus d'échecs)
- Survol pour infobulle : requêtes, succès, erreurs, latence moyenne, coût estimé

## Tableau de classement par dimension

Sous le graphique, un tableau classe par la dimension courante (plateforme/modèle/groupe) :

| Colonne | Description |
|--------|-------------|
| Nom | Nom de plateforme / modèle / groupe |
| Requêtes | Total des requêtes pour cette dimension |
| Succès | Nombre de requêtes réussies (codé couleur pour santé) |
| Entrée | Nombre de tokens entrée |
| Sortie | Nombre de tokens sortie |
| Cache | Nombre de tokens cache |
| Latence moyenne | Temps de réponse moyen |
| Coût estimé | Coût estimé (codé couleur pour niveau de dépense) |

### Tri & Pagination

- Cliquez un en-tête de colonne pour trier (recliquez pour basculer ascendant/descendant)
- 50 lignes par page avec contrôles de pagination en bas

## Rafraîchissement en temps réel

Les statistiques se rafraîchissent automatiquement à l'arrivée de nouvelles requêtes — aucune action manuelle nécessaire.

:::tip{title="Note"}
Les coûts estimés sont calculés depuis la table tarifaire et fournis à titre indicatif. La facturation réelle se base sur la facturation de la plateforme.
:::
