# Règles de routage

Les règles de routage déterminent comment AiDog fait correspondre les requêtes entrantes au groupe et à la plateforme appropriés.

## Flux de correspondance

```
Requête entrante → Détection d'interface → Groupe correspondant → Sélection plateforme → Mapping modèle → Transfert
```

## Correspondance de groupe

Les requêtes sont associées aux groupes par priorité :

1. **Correspondance de chemin** — Le chemin correspond au préfixe de chemin configuré du groupe
2. **Correspondance de nom de modèle** — Le nom de modèle est dans la table de mapping d'un groupe
3. **Groupe par défaut** — Utilisé si aucune correspondance
4. **Premier groupe** — Utilisé si aucun groupe par défaut

## Sélection de plateforme

Dans un groupe, les plateformes sont sélectionnées par mode de routage :

### Mode Failover

Sélectionne la première plateforme disponible. En cas d'échec, essaie automatiquement la suivante. Si toutes échouent, renvoie une erreur.

### Mode Load Balance

Sélectionne aléatoirement une plateforme selon le poids. Le failover s'applique aussi en cas d'échec.

## Failover

Le failover automatique est déclenché par :

- Limitation de débit (429)
- Erreur serveur (5xx)
- Timeout de connexion
- Erreur réseau

## Chaîne de mapping de modèle

Ordre de résolution du nom de modèle :

1. **Mapping au niveau groupe** — Si le groupe définit des mappings
2. **Mapping au niveau plateforme** — Si la plateforme définit des mappings
3. **Valeur originale** — Utilise le nom original de la requête

```
Client demande : gpt-4o
  → Mapping groupe : gpt-4o → deepseek-chat
  → Mapping plateforme : (aucun)
  → Envoyé à la plateforme : deepseek-chat
```
