# Test de modèles

AiDog fournit un test de modèles intégré pour vérifier la connectivité de la plateforme et la disponibilité des modèles, sans outils externes.

## Utilisation

Sur la page **« Plateformes »**, cliquez sur « Tester » d'une carte de plateforme pour ouvrir le panneau de test.

## Modes de test

| Mode | Description |
|------|-------------|
| **Test rapide** | Envoie une courte requête avec le modèle par défaut de la plateforme pour une vérification rapide de connectivité |
| **Modèle unique** | Teste un modèle spécifique |
| **Test par lot** | Teste plusieurs modèles successivement pour vérifier la disponibilité |
| **Test aléatoire** | Teste tous les modèles disponibles de la plateforme un par un |
| **Test personnalisé** | Choisit un modèle et saisit un prompt personnalisé et un nombre max de tokens |

## Résultats de test

Chaque résultat affiche :

- **Nom du modèle** — Le modèle utilisé pour le test
- **Statut** — Succès ou échec
- **Temps de réponse** — Temps total entre l'envoi et la réception complète
- **Usage de tokens** — Comptages de tokens entrée/sortie
- **Contenu de la réponse** — Le texte renvoyé (dépliable)

## Cas d'usage

- **Après ajout d'une plateforme** — Vérifier que l'API Key et la Base URL sont correctes
- **Après modification d'endpoints** — Confirmer que le nouvel endpoint fonctionne
- **Après mapping de modèle** — S'assurer que le modèle cible est disponible
- **Dépannage** — Quand les requêtes proxifiées échouent, tester d'abord la plateforme concernée

## Notes

- Le test utilise l'API Key et l'endpoint configurés, consommant de vrais tokens
- Les tests de plateforme mock n'envoient pas de vraies requêtes
- Les modes lot et aléatoire testent plusieurs modèles successivement et prennent plus de temps
