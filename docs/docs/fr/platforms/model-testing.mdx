# Model Testing

AiDog provides a built-in model testing feature to verify platform connectivity and model availability without external tools.

## How to Use

On the **"Platforms"** page, click the "Test" button on a platform card to open the model testing panel.

## Test Modes

| Mode | Description |
|------|-------------|
| **Quick Test** | Send a short request using the platform's default model for fast connectivity check |
| **Single Model** | Test a specific model |
| **Batch Test** | Test multiple models sequentially to verify availability |
| **Random Test** | Test all available models on the platform one by one |
| **Custom Test** | Choose a model and enter a custom prompt and max token count |

## Test Results

Each test result shows:

- **Model Name** — The model used for testing
- **Status** — Success or failure
- **Response Time** — Total time from sending the request to receiving the full response
- **Token Usage** — Input and output token counts
- **Response Content** — The text returned by the model (collapsible)

## Use Cases

- **After adding a platform** — Verify that the API Key and Base URL are configured correctly
- **After changing endpoints** — Confirm the new endpoint works properly
- **After model mapping** — Ensure the mapped target model is available on the platform
- **Troubleshooting** — When proxied requests fail, test the corresponding platform first

## Notes

- Testing uses the platform's configured API Key and endpoint, which consumes actual tokens
- Mock platform tests do not send real requests
- Batch and random modes test multiple models sequentially and take longer
