# Mock Platform

Mock platforms do not send real AI requests — they return pre-configured responses. Ideal for development, debugging, testing routing rules, and verifying client integrations.

## Creating a Mock Platform

1. Go to the **"Platforms"** page
2. Click the **"+"** button
3. Select **"Mock"** from the platform type list
4. No API Key or Base URL required
5. Click Save

After saving, AiDog automatically creates a corresponding group. Point your client to that group's path to use the mock.

## Response Configuration

Mock platforms allow you to customize the following response parameters:

| Parameter | Description | Default |
|-----------|-------------|---------|
| Response Text | Simulated response text | `Hello from mock` |
| Status Code | HTTP status code | `200` |
| Response Delay | Simulated delay in milliseconds | `0` |
| Input Tokens | Simulated input token count | `10` |
| Output Tokens | Simulated output token count | `20` |
| Cache Tokens | Simulated cache hit tokens | `0` |
| Finish Reason | Simulated finish reason (e.g. stop) | `stop` |
| Streaming | Whether to return as stream (follow request, or force on/off) | Follow request |

## Error Simulation

Mock platforms support simulating various error scenarios:

| Error Mode | Description |
|------------|-------------|
| No Error | Return normal response |
| HTTP Error | Return a specified HTTP error status code |
| Rate Limit (429) | Simulate platform rate limiting |
| Timeout | Simulate request timeout |

Use this to test client error handling or AiDog's failover behavior.

## Use Cases

- **Development** — Test client integration without real API keys
- **Routing Tests** — Verify group matching and model mapping rules
- **Failover Testing** — Simulate errors to observe automatic platform switching
- **Performance Testing** — Set delays to test client behavior under slow responses
