# Correspondance de modèles

La correspondance de modèles permet de traduire les noms de modèles des requêtes client vers les noms réellement supportés par la plateforme.

## Cas d'usage

- Le client demande `gpt-4o`, mais vous voulez router vers `deepseek-chat` de DeepSeek
- Standardiser le nommage des modèles entre plateformes
- Simplifier la sélection via des alias

## Configuration

Ajoutez des règles dans la section « Correspondance de modèles » de la plateforme ou du groupe :

| Modèle demandé | Correspond à |
|---------------|---------|
| `gpt-4o` | `deepseek-chat` |
| `claude-3-opus` | `claude-3-opus-20240229` |

### Règles

- **Correspondance exacte** — Ne mappe que si le nom correspond exactement
- **Préserver l'original** — Les noms non matchés passent inchangés
- **Priorité du groupe** — Les mappings au niveau du groupe priment sur ceux au niveau plateforme

## Correspondance à deux niveaux

Configurable à deux niveaux :

- **Niveau plateforme** — S'applique à toutes les requêtes de cette plateforme
- **Niveau groupe** — S'applique uniquement dans les règles de routage du groupe

Voir [Règles de routage](/fr/groups/routing-rules).

## Exemple d'alias

Créez des alias courts ; les clients n'ont qu'à utiliser l'alias :

| Alias | Cible | Usage |
|-------|--------|---------|
| `fast` | `gpt-4o-mini` | Réponses rapides |
| `smart` | `claude-3-opus` | Haute qualité |
| `code` | `deepseek-coder` | Génération de code |
