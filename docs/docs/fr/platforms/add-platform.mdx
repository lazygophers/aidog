# Add Platform

Platforms are the core concept in AiDog — each represents an AI API service endpoint.

![AiDog — Add Platform](/screenshots/add-platform.png)

## Adding Steps

1. Go to the **"Platforms"** page
2. Click the **"+"** button in the top right
3. Select a platform from the preset list (or fill in manually)
4. Enter your API Key and click Save

:::tip{title="Auto Group"}
After saving a platform, AiDog automatically creates a group for it, so you can start using it right away without manual configuration.
:::

## Basic Configuration

| Field | Description | Example |
|-------|-------------|---------|
| Name | Display name | `My OpenAI` |
| Base URL | API base URL (with version prefix) | `https://api.openai.com/v1` |
| API Key | Platform API key | `sk-...` |

### Platform Presets

AiDog includes 50+ built-in platform presets (OpenAI, Anthropic, Gemini, DeepSeek, Qwen, and more). Selecting a preset auto-fills all configuration fields — just enter your API Key.

Some presets are marked with "Coding Plan" — they automatically configure quota queries.

## Passthrough Mode

Some platform types support "passthrough" mode: AiDog forwards client request headers and body as-is, without any protocol conversion or authentication injection.

- **Upstream URL** — Enter the platform root URL (e.g. `https://api.anthropic.com`), without version prefix
- **API Key** — Optional in passthrough mode; the client can carry its own authentication

Ideal for scenarios where client-side authentication (e.g. OAuth subscriptions) must be preserved.

## Multiple Endpoints

Each platform can have multiple endpoints, each with its own address and interface type. AiDog automatically selects the matching endpoint based on the client request.

For example, the same platform can support both the chat interface and the coding assistant interface.

## Model Configuration

### Default Models

In the platform settings, specify which model to use by default for different types of requests. AiDog automatically matches the most suitable model based on the requested model name.

### Fetch Model List

Click the "Fetch Models" button to automatically retrieve available models from the platform.

### Model Mapping

Map requested model names to the actual model names supported by the platform. See [Model Mapping](/platforms/model-mapping).

## Balance Query Configuration

For platforms using the NewAPI system, configure separate balance queries:

- **Query URL** — Separate from the API URL
- **Query Token** — Obtained from the dashboard (different from API Key)
- **User ID** — Optional

See [Balance & Quotas](/stats/balance).

## Manual Budget

For platforms without automatic balance queries, set manual budget limits:

- **Total** — Overall spending limit
- **Rolling Window** — Spending limit within a rolling time window
- **Fixed Window** — Spending limit that resets on a fixed schedule
- **Daily** — Daily spending limit

When depleted, AiDog stops forwarding to that platform and automatically fails over. See [Balance & Quotas](/stats/balance).

## Model Testing

After saving a platform, test its connectivity:

1. Click the "Test" button on the platform card
2. Choose a test mode: Quick / Batch / Custom
3. View results: response time, token count, success or failure

See [Model Testing](/platforms/model-testing).

## Platform List

The Platforms page displays all added platform cards, each showing:

- Platform name and logo
- Health status indicator (green/yellow/red)
- Balance or quota utilization
- Today's request count and success rate

Supports pinyin search to quickly locate a platform.

## Other Operations

- **Enable/Disable** — Pause a platform without deleting it
- **Edit** — Click a platform card to view details and modify settings
- **Delete** — Click the delete button. Related logs and statistics are preserved
- **Drag to Reorder** — Drag cards in the list to adjust display order

## FAQ

### How to fill Base URL?

The Base URL is the complete base path including the version prefix:

| Platform | Base URL |
|----------|----------|
| OpenAI | `https://api.openai.com/v1` |
| Anthropic | `https://api.anthropic.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| Qwen | `https://dashscope.aliyuncs.com/compatible-mode/v1` |

:::caution{title="Note"}
Do not append `/chat/completions` to the Base URL. AiDog handles this automatically.
:::

### API Key Security

API keys are stored locally and never uploaded to any server.
