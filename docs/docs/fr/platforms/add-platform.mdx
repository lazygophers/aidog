# Ajouter une plateforme

Les plateformes sont le concept central dans AiDog — chacune représente un endpoint de service AI API.

![AiDog — Ajouter une plateforme](/screenshots/add-platform.png)

## Étapes d'ajout

1. Allez à la page **« Plateformes »**
2. Cliquez le bouton **« + »** en haut à droite
3. Sélectionnez une plateforme dans la liste de presets (ou remplissez manuellement)
4. Entrez votre API Key et cliquez Enregistrer

:::tip{title="Groupe auto"}
Après enregistrement d'une plateforme, AiDog crée automatiquement un groupe pour elle, vous pouvez donc l'utiliser immédiatement sans configuration manuelle.
:::

## Configuration de base

| Champ | Description | Exemple |
|-------|-------------|---------|
| Nom | Nom d'affichage | `Mon OpenAI` |
| URL de base | URL de base API (avec préfixe de version) | `https://api.openai.com/v1` |
| API Key | Clé API de la plateforme | `sk-...` |

### Presets de plateforme

AiDog inclut 50+ presets de plateforme intégrés (OpenAI, Anthropic, Gemini, DeepSeek, Qwen, et plus). Sélectionner un preset remplit automatiquement tous les champs — entrez juste votre API Key.

Certains presets sont marqués « Coding Plan » — ils configurent automatiquement les requêtes de quota.

## Mode passthrough

Certains types de plateforme supportent le mode « passthrough » : AiDog transmet les en-têtes et corps de requête client tels quels, sans conversion de protocole ni injection d'authentification.

- **URL amont** — Entrez l'URL racine de la plateforme (ex. `https://api.anthropic.com`), sans préfixe de version
- **API Key** — Optionnelle en mode passthrough ; le client peut porter sa propre authentification

Idéal pour les scénarios où l'authentification côté client (ex. abonnements OAuth) doit être préservée.

## Endpoints multiples

Chaque plateforme peut avoir plusieurs endpoints, chacun avec sa propre adresse et type d'interface. AiDog sélectionne automatiquement l'endpoint correspondant selon la requête client.

Par exemple, la même plateforme peut supporter à la fois l'interface chat et l'interface assistant de codage.

## Configuration des modèles

### Modèles par défaut

Dans les réglages de plateforme, spécifiez quel modèle utiliser par défaut pour différents types de requêtes. AiDog correspond automatiquement le modèle le plus approprié selon le nom de modèle demandé.

### Récupérer la liste des modèles

Cliquez le bouton « Récupérer modèles » pour obtenir automatiquement les modèles disponibles depuis la plateforme.

### Mapping de modèles

Mappez les noms de modèles demandés aux noms réels supportés par la plateforme. Voir [Mapping de modèles](/fr/platforms/model-mapping).

## Configuration de requête de solde

Pour les plateformes utilisant le système NewAPI, configurez des requêtes de solde séparées :

- **URL de requête** — Séparée de l'URL API
- **Token de requête** — Obtenu depuis le tableau de bord (différent de la clé API)
- **User ID** — Optionnel

Voir [Solde & Quotas](/fr/stats/balance).

## Budget manuel

Pour les plateformes sans requête automatique de solde, définissez des limites de budget manuel :

- **Total** — Limite de dépense globale
- **Fenêtre glissante** — Limite sur une fenêtre temporelle glissante
- **Fenêtre fixe** — Limite réinitialisée selon un calendrier fixe
- **Quotidienne** — Limite de dépense journalière

Quand épuisé, AiDog arrête de transférer vers cette plateforme et failover automatique. Voir [Solde & Quotas](/fr/stats/balance).

## Test de modèle

Après enregistrement d'une plateforme, testez sa connectivité :

1. Cliquez le bouton « Tester » sur la carte de plateforme
2. Choisissez un mode de test : Rapide / Lot / Personnalisé
3. Consultez les résultats : temps de réponse, nombre de tokens, succès ou échec

Voir [Test de modèle](/fr/platforms/model-testing).

## Liste des plateformes

La page Plateformes affiche toutes les cartes de plateforme ajoutées, chacune montrant :

- Nom et logo de la plateforme
- Indicateur d'état de santé (vert/jaune/rouge)
- Solde ou utilisation de quota
- Nombre de requêtes du jour et taux de succès

Supporte la recherche pinyin pour localiser rapidement une plateforme.

## Autres opérations

- **Activer/Désactiver** — Mettre en pause une plateforme sans la supprimer
- **Éditer** — Cliquez une carte de plateforme pour voir les détails et modifier les réglages
- **Supprimer** — Cliquez le bouton supprimer. Les logs et statistiques liés sont préservés
- **Glisser pour réordonner** — Glissez les cartes dans la liste pour ajuster l'ordre d'affichage

## FAQ

### Comment remplir l'URL de base ?

L'URL de base est le chemin de base complet incluant le préfixe de version :

| Plateforme | URL de base |
|----------|----------|
| OpenAI | `https://api.openai.com/v1` |
| Anthropic | `https://api.anthropic.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| Qwen | `https://dashscope.aliyuncs.com/compatible-mode/v1` |

:::caution{title="Note"}
N'ajoutez pas `/chat/completions` à l'URL de base. AiDog gère cela automatiquement.
:::

### Sécurité de l'API Key

Les clés API sont stockées localement et ne sont jamais téléversées sur un serveur.
