# Utilisation du proxy

AiDog exécute un serveur proxy HTTP local qui reçoit et transfère les requêtes API IA.

## Adresse du proxy

Adresse par défaut :

```
http://127.0.0.1:9876
```

Le port peut être modifié dans **« Réglages → Proxy »**.

## Démarrage automatique

Activez « Auto-start proxy » dans les réglages. Quand activé, le service proxy démarre automatiquement au lancement d'AiDog.

## Utilisation de base

Pointez tout client IA supportant des adresses API personnalisées vers AiDog :

| Type de client | Configuration |
|-------------|---------------|
| SDK OpenAI | Définir `base_url` à l'adresse du proxy |
| curl | Requérir directement l'adresse du proxy |
| Web apps | Définir l'endpoint API au proxy |
| Assistants de code | Définir l'URL de base de l'API |

### Python (SDK OpenAI)

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:9876/proxy/v1",
    api_key="any-value"  # AiDog utilise votre vraie clé configurée
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello!"}]
)
```

### Node.js

```javascript
import OpenAI from 'openai';

const client = new OpenAI({
  baseURL: 'http://127.0.0.1:9876/proxy/v1',
  apiKey: 'any-value',
});

const response = await client.chat.completions.create({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'Hello!' }],
});
```

### curl

```bash
curl http://127.0.0.1:9876/proxy/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer any-value" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

## Streaming

AiDog supporte pleinement le streaming SSE (Server-Sent Events). Les requêtes streaming du client sont transférées transparent vers la plateforme amont.

## Timeout

Les requêtes proxy ont une protection timeout :

- Timeout par défaut : 300 secondes
- Configurable globalement ou par groupe
- Renvoie 504 Gateway Timeout à l'expiration
