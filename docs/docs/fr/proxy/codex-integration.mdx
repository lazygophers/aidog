# Intégration Codex

AiDog supporte nativement OpenAI Codex CLI, Claude Code et d'autres assistants IA, avec des éditeurs de configuration intégrés.

## OpenAI Codex

### Configuration

Définissez ce qui suit dans le fichier de config de Codex `~/.codex/config.toml` :

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

### Page de réglages Codex

AiDog fournit un éditeur de config Codex intégré dans **« Réglages → Codex »** :

- Éditez les réglages `~/.codex/config.toml` directement
- Pas besoin de localiser et éditer les fichiers manuellement
- Les changements prennent effet immédiatement après sauvegarde

### Notes

- Codex utilise l'**API Responses** (`/v1/responses`). AiDog détecte et route automatiquement.
- Assurez-vous que la plateforme cible a un endpoint avec le protocole `openai_responses` configuré.
- Ajoutez des mappings de modèle au niveau plateforme ou groupe si besoin.

## Claude Code

### Export de config en un clic

Dans l'onglet **« Réglages → Claude Code »** d'AiDog, cliquez « Exporter config ». AiDog écrit automatiquement l'adresse du proxy dans le fichier de config de Claude Code (`~/.claude.json`).

### Configuration manuelle

Vous pouvez aussi définir des variables d'environnement :

```bash
export ANTHROPIC_BASE_URL="http://127.0.0.1:9876"
export ANTHROPIC_API_KEY="any-value"
```

### Page de réglages Claude Code

AiDog fournit un éditeur dédié dans **« Réglages → Claude Code »** :

- **Édition de config** — Éditez les réglages Claude Code (modèle, permissions, sandbox, plugins, hooks, StatusLine, etc.)
- **Importer de Claude Code** — Import en un clic de la config Claude Code existante
- **Exporter vers Claude Code** — Écrire l'adresse du proxy d'AiDog dans la config Claude Code
- **Sync des réglages de groupe** — Génère automatiquement des fichiers de config indépendants par groupe, facilitant la commutation

### Notes

- Claude Code utilise le **protocole natif Anthropic**
- Assurez-vous que la plateforme cible supporte le protocole Anthropic ou configurez la conversion
- AiDog fournit un chemin dédié `/claude/v1/messages` pour Claude Code

## Autres assistants

Tout outil supportant des adresses API personnalisées peut utiliser AiDog comme proxy :

1. Définissez l'URL de base de l'API à `http://127.0.0.1:9876`
2. Définissez la clé API à n'importe quelle valeur
3. Assurez-vous qu'AiDog a une plateforme et un protocole correspondants configurés
