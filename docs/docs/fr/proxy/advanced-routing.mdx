# Routage avancé

Maîtrisez les capacités de routage avancé d'AiDog pour un contrôle fin du trafic.

## Agrégation multi-plateforme

Ajoutez plusieurs plateformes à un groupe avec différents modes de routage. Voir [Créer & gérer](/fr/groups/manage-groups).

## Failover

En cas d'échec d'une plateforme, AiDog essaie automatiquement la suivante :

```
Requête → Plateforme A (429 Rate Limit)
        → Plateforme B (Succès)
        → Retour réponse
```

Le failover se déclenche sur :

- HTTP 429 (Rate Limit)
- HTTP 5xx (Server Error)
- Timeout de connexion
- Erreurs réseau

## Load Balancing

Répartit les requêtes entre plateformes par poids. Exemple :

| Plateforme | Poids | Trafic attendu |
|----------|--------|------------------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## Alias de modèles

Créez des alias via le mapping de modèle :

| Alias | Correspond à | Usage |
|-------|---------|---------|
| `fast` | `gpt-4o-mini` | Réponses rapides |
| `smart` | `claude-3-opus` | Haute qualité |
| `code` | `deepseek-coder` | Génération de code |

Les clients utilisent les alias ; AiDog route vers les plateformes réelles selon la config du groupe.

## Routage chaîné

Combinez les mappings de groupe et de plateforme pour une transformation multicouche :

1. Le client demande le modèle `smart`
2. Le groupe A mappe `smart` → `claude-3-opus`
3. La plateforme X mappe `claude-3-opus` → `claude-3-opus-20240229`
4. Envoyé à Anthropic

## Timeout au niveau groupe

Chaque groupe peut définir indépendamment le timeout de requête et le timeout de connexion, surchargeant les défauts globaux. Utile quand certains modèles plus lents nécessitent des temps d'attente plus longs.

Voir [Créer & gérer](/fr/groups/manage-groups).
