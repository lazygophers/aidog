# Réglages des logs

Sur la page **Réglages** vous contrôlez la granularité et la rétention des logs, en équilibrant observabilité et vie privée / espace disque.

## Niveaux d'enregistrement

Les logs ont trois niveaux de commutateurs qui s'appliquent progressivement :

| Niveau | Description | Défaut |
|------|------|------|
| Commutateur maître | Quand off, aucun log proxy n'est enregistré | On |
| Enregistrer requête utilisateur originale | Enregistre les en-têtes et corps de requête originaux envoyés par le client | Off |
| Enregistrer requête amont | Enregistre les en-têtes et corps convertis envoyés à l'amont | Off |

:::warning
Le corps de requête peut contenir du contenu sensible (prompts, clés, etc.). Assurez-vous que votre environnement local est de confiance avant d'activer « Enregistrer requête utilisateur originale » ou « Enregistrer requête amont ».
:::

## Rétention

Les logs sont nettoyés selon trois périodes de rétention indépendantes :

| Type | Défaut | Méthode de nettoyage |
|------|------|----------|
| Requête utilisateur originale | 7 jours | Ne vide que le champ corps, conserve la ligne |
| Requête amont | 7 jours | Ne vide que le champ corps, conserve la ligne |
| Enregistrements de log | 90 jours | Supprime l'entrée entière |

Autrement dit, les corps de requête de plus de 7 jours sont vidés (les statistiques sont conservées), et les logs de plus de 90 jours sont entièrement supprimés.

## Nettoyage manuel

Dans **Réglages → Gestion des données** vous pouvez déclencher manuellement « Nettoyer les logs expirés » à tout moment, ce qui applique immédiatement les règles de rétention ci-dessus.

## Stockage des données

Tous les logs sont stockés dans la base locale `~/.aidog/aidog.db` et ne sont jamais téléversés sur un serveur.
