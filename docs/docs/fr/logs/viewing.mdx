# Consulter les logs

La page **Logs** enregistre chaque requête proxy passant par AiDog, facilitant le diagnostic de routage, conversion de protocole et réponses amont.

## Liste des logs

Chaque entrée affiche :

| Champ | Description |
|------|------|
| Groupe | Nom du groupe correspondant |
| Modèle | Modèle original demandé par le client |
| Modèle réel | Modèle envoyé à l'amont après mapping |
| Protocole | Protocole entrant → protocole amont |
| Plateforme | Plateforme ayant traité la requête |
| Statut | Code HTTP statut (succès / échec) |
| Durée | Durée totale de la requête (millisecondes) |
| Tokens | Nombre de tokens entrée / sortie / cache |
| Heure | Moment de la requête |

La liste affiche 50 entrées par page avec pagination.

## Filtrage

Utilisez les filtres en haut pour localiser rapidement les logs :

- **Plateforme** : filtrer par plateforme spécifique
- **Groupe** : filtrer par groupe
- **Statut** : succès uniquement / échec uniquement
- **Plage de temps** : tout / 1 heure / 6 heures / 24 heures / 7 jours / 30 jours
- **Modèle** : recherche par nom de modèle original ou réel

## Détails de la requête

Cliquez une entrée pour déplier les détails et voir la chaîne complète :

- **Requête utilisateur** : URL, en-têtes et corps originaux
- **Requête amont** : URL convertie, en-têtes et corps envoyés à l'amont
- **Réponse amont** : code statut, en-têtes et corps renvoyés par l'amont
- **Réponse utilisateur** : en-têtes et corps finaux renvoyés au client

Les détails supportent la copie en un clic, pour coller dans un ticket ou outil de debug.

:::tip
Le contenu de « Requête utilisateur » et « Requête amont » est contrôlé par les réglages de logs. Par défaut, le corps n'est pas enregistré. Voir [Réglages des logs](/fr/logs/log-settings).
:::
