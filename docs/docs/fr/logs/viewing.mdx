# Consulter les journaux

La page **« Journaux »** enregistre chaque requête proxy passant par AiDog, ce qui facilite le diagnostic des problèmes de routage, de conversion de protocole et de réponse de l'amont.

## Liste des journaux

Chaque entrée de journal affiche les informations suivantes :

| Champ | Description |
|------|------|
| Groupes | Nom du groupe correspondant |
| Modèle | Modèle d'origine demandé par le client |
| Modèle réel | Modèle envoyé à l'amont après le mappage de modèle |
| Protocole | Protocole entrant → protocole de l'amont |
| Plateformes | Plateforme ayant réellement traité la requête |
| Statut | Code de statut HTTP (succès / échec) |
| Durée | Durée totale de la requête (en millisecondes) |
| Token | Nombre de tokens d'entrée / de sortie / en cache |
| Heure | Heure à laquelle la requête a eu lieu |

La liste affiche 50 entrées par page et prend en charge la pagination.

## Filtrage

Utilisez les filtres en haut de la page pour localiser rapidement les journaux :

- **Plateformes** : filtrer par plateforme spécifiée
- **Groupes** : filtrer par groupe
- **Statut** : succès uniquement / échecs uniquement
- **Plage de temps** : tout / dernière 1 heure / 6 heures / 24 heures / 7 jours / 30 jours
- **Modèle** : rechercher par nom du modèle d'origine ou du modèle réel

## Détails de la requête

Cliquez sur n'importe quelle entrée de journal pour développer ses détails et consulter la chaîne complète de la requête :

- **Requête utilisateur** : URL, en-têtes de la requête et corps de la requête d'origine
- **Requête amont** : URL, en-têtes de la requête et corps de la requête convertis et envoyés à l'amont
- **Réponse amont** : code de statut, en-têtes de réponse et corps de réponse renvoyés par l'amont
- **Réponse utilisateur** : en-têtes de réponse et corps de réponse finalement renvoyés au client

Les détails peuvent être copiés en un clic, pour faciliter le collage dans un ticket ou un outil de débogage.

:::tip
Le contenu « Requête utilisateur » et « Requête amont » enregistré dans les détails est contrôlé par les paramètres des journaux ; par défaut, le corps de la requête n'est pas enregistré. Pour en savoir plus, consultez [Paramètres des journaux](/logs/log-settings).
:::
