# Codex

Configurez l'intégration AiDog avec OpenAI Codex CLI dans l'onglet **« Réglages » → « Codex »**.

## Édition de la configuration

Éditez les réglages `~/.codex/config.toml` de Codex directement dans AiDog :

- **Provider** — Fournisseur API et URL de base
- **Modèle** — Nom du modèle à utiliser
- **Autres options** — Tous les éléments de configuration supportés par Codex

Les changements sont écrits immédiatement dans le fichier de configuration — aucun besoin de localiser et éditer les fichiers manuellement.

## Configuration rapide

Une configuration minimale ne requiert que l'adresse proxy et une clé API quelconque :

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

## Configuration de groupe

Codex supporte le bascule entre groupes via les profiles :

1. Cliquez le bouton « Codex » sur une carte de groupe pour copier la commande de lancement
2. Collez dans votre terminal :

```bash
AIDOG_KEY='<group-name>' codex -p '<group-name>' \
  --dangerously-bypass-approvals-and-sandbox -a never
```

AiDog route les requêtes vers le groupe correspondant selon la variable d'environnement `AIDOG_KEY`.

## Notes

- Codex utilise l'API Responses (`/v1/responses`), qu'AiDog détecte et route automatiquement
- Assurez-vous que la plateforme correspondante a été ajoutée à AiDog
- Ajoutez du model mapping dans les réglages plateforme ou groupe si nécessaire
- Le préfixe de chemin `/proxy/v1` dans l'adresse proxy ne doit pas être omis
