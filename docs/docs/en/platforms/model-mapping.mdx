# Model Mapping

Model mapping allows you to translate model names from client requests to the actual model names supported by the platform.

## Use Cases

- Client requests `gpt-4o`, but you want to route to DeepSeek's `deepseek-chat`
- Standardize model naming across platforms
- Simplify model selection with aliases

## Configuration

Add mapping rules in the platform or group's "Model Mapping" section:

| Request Model | Maps To |
|---------------|---------|
| `gpt-4o` | `deepseek-chat` |
| `claude-3-opus` | `claude-3-opus-20240229` |

### Rules

- **Exact match** — Only maps when the model name matches exactly
- **Preserve original** — Unmatched model names pass through unchanged
- **Group priority** — Group-level mappings take precedence over platform-level mappings

## Two-Level Mapping

Model mapping can be configured at two levels:

- **Platform level** — Applies to all requests on that platform
- **Group level** — Only applies within that group's routing rules

See [Routing Rules](/groups/routing-rules).

## Alias Example

Use model mapping to create short aliases. Clients only need to use the alias:

| Alias | Target | Purpose |
|-------|--------|---------|
| `fast` | `gpt-4o-mini` | Fast responses |
| `smart` | `claude-3-opus` | High quality |
| `code` | `deepseek-coder` | Code generation |
