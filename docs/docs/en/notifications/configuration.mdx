# Notification Configuration

Notification settings (`NotificationSettings`) live in Settings → Notifications. Controls master switch, TTS, per-type channels.

## Global Switches

| Field | Default | Description |
| --- | --- | --- |
| `enabled` | true | Master switch. When off, all dispatch is bypassed |
| `tts_enabled` | true | TTS master switch. ANDed with per-type `tts` (both on to speak) |
| `tts_backend` | CrossPlatform | TTS backend (CrossPlatform / MacSay / WebSpeech) |

## Per-Type Config (per_type)

`per_type` is a `NotifType → TypeSetting` map; each type configured independently:

| Field | Description |
| --- | --- |
| `tts` | Whether this type speaks (ANDed with global `tts_enabled`) |
| `form` | Dispatch form (Full / PopupOnly / InboxOnly / SoundOnly) |

Missing keys are treated as all-channels-enabled + Full.

## Template Variables

Title/body templates support variable placeholders, substituted at dispatch (`substitute_vars`):

- `{project}` — project name
- `{status}` — status
- other context variables

Example: title template `{project} · {status}` → filled with actual values at dispatch.

## Test Notification

The settings page has a "Test Notification" button (`notification_test` command) that dispatches a test notification per the current config to verify channels.
