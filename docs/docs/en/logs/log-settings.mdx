# Log Settings

On the **Settings** page you can control the granularity and retention of logging, striking a balance between observability and privacy / disk usage.

## Recording Levels

Logging has three levels of switches that take effect progressively:

| Level | Description | Default |
|------|------|------|
| Master switch | When off, no proxy logs are recorded at all | On |
| Record original user request | Records the original request headers and request body sent by the client | Off |
| Record upstream request | Records the converted request headers and request body sent to the upstream | Off |

:::warning
The request body may contain sensitive content (prompts, keys, etc.). Make sure your local environment is trusted before enabling "Record original user request" or "Record upstream request".
:::

## Retention

Logs are cleaned up according to three independent retention periods:

| Type | Default | Cleanup method |
|------|------|----------|
| Original user request | 7 days | Clears only the request body field, keeps the log row |
| Upstream request | 7 days | Clears only the request body field, keeps the log row |
| Log records | 90 days | Deletes the entire log entry |

In other words, request bodies older than 7 days are cleared (statistics are still retained), and logs older than 90 days are deleted entirely.

## Manual Cleanup

In **Settings → Data Management** you can manually trigger "Clean up expired logs" at any time, which immediately applies the retention rules above.

## Data Storage

All logs are stored in the local database `~/.aidog/aidog.db` and are never uploaded to any server.
