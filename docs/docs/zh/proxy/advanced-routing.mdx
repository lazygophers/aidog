# 高级路由

掌握 AiDog 的高级路由能力，实现更精细的流量控制。

## 多平台聚合

在分组中添加多个平台，配合不同的路由模式实现流量管理。详见 [创建与管理](/groups/manage-groups)。

## 故障转移

当平台请求失败时，AiDog 自动尝试组内下一个平台：

```
请求 → Platform A (429 Rate Limit)
     → Platform B (成功)
     → 返回响应
```

触发故障转移的条件：
- HTTP 429 (Rate Limit)
- HTTP 5xx (Server Error)
- 连接超时
- 网络错误

## 负载均衡

按权重比例在多个平台间分配请求。例如：

| 平台 | 权重 | 预期流量 |
|------|------|---------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## 模型别名

通过模型映射建立别名系统：

| 别名 | 映射目标 | 用途 |
|------|---------|------|
| `fast` | `gpt-4o-mini` | 快速响应 |
| `smart` | `claude-3-opus` | 高质量 |
| `code` | `deepseek-coder` | 代码生成 |

客户端只需使用别名，AiDog 根据分组路由到实际平台。

## 链式路由

通过分组嵌套模型映射，实现多层转换：

1. 客户端请求模型 `smart`
2. 分组 A 映射 `smart` → `claude-3-opus`
3. 平台 X 映射 `claude-3-opus` → `claude-3-opus-20240229`
4. 发送到 Anthropic

## 分组级超时

每个分组可独立设置请求超时和连接超时，覆盖全局默认值。适合某些响应较慢的模型需要更长等待时间的场景。

详见 [创建与管理](/groups/manage-groups)。
