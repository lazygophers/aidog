# 高级路由

掌握 AiDog 的高级路由能力，实现更精细的流量控制。

## 多平台负载均衡

在分组中添加多个平台，配合不同的路由策略实现负载均衡：

### 顺序轮转

平台按排列顺序依次接收请求。适合有主备需求的场景。

### 随机分配

请求随机分配到组内平台。适合多平台均分的场景。

### 加权分配

按权重比例分配请求。例如：

| 平台 | 权重 | 预期流量 |
|------|------|---------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## 故障转移

当平台请求失败时，AiDog 自动尝试组内下一个平台：

```
请求 → Platform A (429 Rate Limit)
     → Platform B (成功)
     → 返回响应
```

触发故障转移的条件：
- HTTP 429 (Rate Limit)
- HTTP 5xx (Server Error)
- 连接超时
- 网络错误

## 模型别名

通过模型映射建立别名系统：

| 别名 | 映射目标 | 用途 |
|------|---------|------|
| `fast` | `gpt-4o-mini` | 快速响应 |
| `smart` | `claude-3-opus` | 高质量 |
| `code` | `deepseek-coder` | 代码生成 |

客户端只需使用别名，AiDog 根据分组路由到实际平台。

## 链式路由

通过分组嵌套模型映射，实现：

1. 客户端请求模型 `smart`
2. 分组 A 映射 `smart` → `claude-3-opus`
3. 平台 X 映射 `claude-3-opus` → `claude-3-opus-20240229`
4. 发送到 Anthropic
