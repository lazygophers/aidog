# 系统设置

在 **「设置」→「系统设置」** 标签页中管理代理服务与应用配置。

![AiDog — System Settings](/screenshots/settings.png)

## 代理服务

顶部状态栏直观显示代理运行状态，支持一键启动/停止：

- 运行状态指示灯（绿色 = 运行中）
- 端口配置输入框（运行中不可修改）
- 一键启动/停止按钮

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 代理端口 | `9876` | HTTP 代理监听端口 |
| 自动启动 | 开启 | AiDog 启动时自动运行代理服务 |

修改端口后需要重新启动代理服务才能生效。

## 超时设置

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 请求超时 | `300s` | 等待上游响应的最长时间 |
| 连接超时 | `10s` | 建立连接的最长等待时间 |

系统默认超时，分组和模型级别可覆盖。

## 日志设置

控制日志的记录粒度与保留时长。详见 [日志设置](/logs/log-settings)。

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 总开关 | 开启 | 关闭后完全不记录日志 |
| 记录用户原始请求 | 关闭 | 记录客户端发来的原始请求头与请求体 |
| 记录上游请求 | 关闭 | 记录转换后发往上游的请求头与请求体 |
| 用户请求保留天数 | 7 天 | 超期清空请求体，保留统计行 |
| 上游请求保留天数 | 7 天 | 超期清空请求体，保留统计行 |
| 日志保留天数 | 90 天 | 超期删除整条日志 |

## 应用日志

控制台日志始终输出；以下设置仅影响日志文件。

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 文件日志 | 开启 | 将日志写入文件 |
| 日志级别 | `INFO` | TRACE / DEBUG / INFO / WARN / ERROR |
| 保留时长 | `3 小时` | 超期自动清理 |

## 数据管理

- **导出平台配置** — 导出所有平台设置
- **导出分组配置** — 导出所有分组设置
- **导出统计数据** — 导出用量统计
- **清理过期日志** — 按保留规则立即清理
- **重置统计数据** — 清空所有统计数据
