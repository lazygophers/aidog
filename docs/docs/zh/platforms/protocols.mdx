# 支持的平台与接口

AiDog 支持丰富的 AI 平台和接口类型，并自动处理不同平台间的协议转换。

## 协议转换

AiDog 采用 **N+N 中枢架构**：所有入站请求先 parse 为中立 `ChatRequest` 模型，再 serialize 为目标平台的 wire 协议。协议间不直接两两互转（N×N），全部经中立模型中转。

```
客户端请求（5 种 wire 协议任一）
  ↓ parse_incoming_request
ChatRequest（中立模型）
  ↓ convert_request
上游请求（5 种 wire 协议任一）+ api_path

上游 SSE（5 种 wire 协议任一）
  ↓ parse_sse
ChatStreamEvent（中立）
  ↓ to_client_sse
客户端 SSE（与入站协议一致）
```

### Wire 协议

endpoint 上 `protocol` 字段支持 5 种 wire 协议：

| Wire 协议 | serde key | API path | 说明 |
|---|---|---|---|
| `Anthropic` | `anthropic` | `/v1/messages` | Claude 系列原生格式 |
| `OpenAI` | `openai` | `/chat/completions` | 默认兜底；主流聚合平台 |
| `OpenAIResponses` | `openai_responses` | `/v1/responses` | Codex / 新版 OpenAI 编程接口 |
| `OpenAICompletions` | `openai_completions` | `/v1/completions` | 旧版文本补全 |
| `Gemini` | `gemini` | `/v1beta/models/{model}:streamGenerateContent` | Google Gemini 原生 |

### 转换矩阵

中立模型保证任一入站协议都能路由到任一出站协议。非流式响应的真转换目前覆盖 **OpenAI → Anthropic**（修复 tool_calls 与 content 共存的兼容 bug）；其余跨协议路径运行时回退到上游 body 原样透传。

| 入站 \ 出站 | anthropic | openai | openai_responses | openai_completions | gemini |
|---|:---:|:---:|:---:|:---:|:---:|
| anthropic | 透传 | ✓ | ✓ | ✓ | ✓ |
| openai | ✓ | 透传 | ✓ | ✓ | ✓ |
| openai_responses | ✓ | ✓ | 透传 | ✓ | ✓ |
| openai_completions | ✓ | ✓ | ✓ | 透传 | ✓ |
| gemini | ✓ | ✓ | ✓ | ✓ | 透传 |

### 特殊平台（绕过 wire 层）

下列 `platform_type` 不经 `convert_request` / `forward_attempt`，在 `proxy/handler.rs` 重试循环之前就被拦截：

| platform_type | 处理方式 |
|---|---|
| `mock` | 内置 mock 适配器，无真实上游 |
| `claude_code` | 1:1 纯透传 OAuth 订阅型 Claude Code 请求 |
| `devin` | 在 `proxy/devin.rs` 内做 chat ↔ session 协议转换（create session → poll → fetch messages → 包装回 openai/anthropic） |
| `cli-proxy` | wire/base_url/key 在候选解析阶段从关联的 `cli_proxy_provider` 行读取 |

## 接口自动识别

AiDog 根据客户端请求路径和 body 自动识别接口类型，无需手动配置：

- **对话接口**（`/v1/chat/completions`、`/v1/messages`）— 最常用的 AI 对话请求
- **编程助手接口**（`/v1/responses`、Codex / Claude Code）— 编程工具使用的接口
- **文本补全接口**（`/v1/completions`）— 旧版文本补全请求

这意味着你可以用同一个代理地址同时服务不同类型的客户端。

## Mock

AiDog 提供 Mock 平台，不发送真实请求，用于开发测试。可自定义响应内容、状态码、延迟等参数。
