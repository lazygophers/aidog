# 协议类型

AiDog 支持多种 AI API 协议，作为入站请求的识别方式和出站请求的转换目标。

## 支持的协议

### OpenAI 系列

| 协议 | 说明 | 端点 |
|------|------|------|
| `openai_chat` | Chat Completions API | `/v1/chat/completions` |
| `openai_completions` | Text Completions API（旧版） | `/v1/completions` |
| `openai_responses` | Responses API（Codex 使用） | `/v1/responses` |

### Anthropic

| 协议 | 说明 | 端点 |
|------|------|------|
| `anthropic` | Claude Messages API | `/v1/messages` |

### Google

| 协议 | 说明 | 端点 |
|------|------|------|
| `gemini` | Gemini Generate Content API | `/v1beta/models/{model}:generateContent` |

### 其他

| 协议 | 说明 |
|------|------|
| `minimax` | MiniMax 原生协议 |
| `glm` | 智谱 GLM 原生协议 |

## 协议自动检测

AiDog 会根据请求路径和内容自动检测入站协议：

- `/v1/chat/completions` → `openai_chat`
- `/v1/completions` → `openai_completions`
- `/v1/responses` → `openai_responses`

这意味着你可以用同一个代理地址同时服务不同类型的客户端。

## 协议转换

入站和出站协议可以不同。AiDog 会自动进行协议转换：

1. 解析入站请求（根据检测到的协议）
2. 统一为内部格式
3. 转换为目标平台的出站协议
4. 发送请求

例如：客户端用 `openai_chat` 发送请求 → AiDog 转换为 `anthropic` 协议发送到 Claude。

## 平台预设

AiDog 内置 50+ 平台预设，每个预设包含：
- 默认 Base URL
- 默认协议
- 支持的模型列表

选择预设后这些值会自动填充。
