# Mock

Mock 平台不发送真实 AI 请求，而是返回预设的响应。适合开发调试、测试路由规则、验证客户端集成的场景。

## 创建 Mock 平台

1. 进入 **「平台」** 页面
2. 点击 **「+」** 按钮
3. 在平台类型中选择 **「Mock」**
4. 无需填写 API Key 和 Base URL
5. 点击保存

Mock 平台保存后，AiDog 会自动创建对应的分组。将客户端指向该分组的路径即可使用。

## 响应配置

Mock 平台允许自定义以下响应参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 响应内容 | 模拟返回的文本 | `Hello from mock` |
| 状态码 | HTTP 状态码 | `200` |
| 响应延迟 | 模拟延迟时间（毫秒） | `0` |
| 输入 Token 数 | 模拟消耗的输入 Token | `10` |
| 输出 Token 数 | 模拟消耗的输出 Token | `20` |
| 缓存 Token 数 | 模拟的缓存命中 Token | `0` |
| 完成原因 | 模拟的完成原因（stop 等） | `stop` |
| 流式响应 | 是否以流式返回（跟随请求，或强制开/关） | 跟随请求 |

## 错误模拟

Mock 平台支持模拟各种错误场景：

| 错误模式 | 说明 |
|----------|------|
| 无错误 | 正常返回响应 |
| HTTP 错误 | 返回指定 HTTP 错误状态码 |
| 限速（429） | 模拟平台限速 |
| 超时 | 模拟请求超时 |

这可以用来测试客户端的错误处理逻辑或 AiDog 的故障转移行为。

## 使用场景

- **开发调试** — 在没有真实 API Key 时测试客户端集成
- **测试路由** — 验证分组匹配和模型映射是否正确
- **测试故障转移** — 模拟错误，观察是否自动切换到备用平台
- **性能测试** — 设置延迟，测试客户端在慢响应下的表现
