# 添加平台

平台是 AiDog 的核心概念 — 每个平台代表一个 AI API 服务商的接入点。

![AiDog — Add Platform](/screenshots/add-platform.png)

## 添加步骤

1. 进入 **「平台」** 页面
2. 点击右上角 **「+」** 按钮
3. 从预设列表选择平台（或自定义填写）
4. 填入 API Key，点击保存

:::tip{title="自动分组"}
保存平台后，AiDog 会自动为该平台创建一个分组，无需手动配置即可使用。
:::

## 基本配置

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 平台显示名称 | `My OpenAI` |
| Base URL | API 基础地址（含版本前缀） | `https://api.openai.com/v1` |
| API Key | 平台的 API Key | `sk-...` |

### 平台预设

AiDog 内置 50+ 平台预设（OpenAI、Anthropic、Gemini、DeepSeek、通义千问等）。选择预设后，所有配置项自动填充，只需填入 API Key 即可。

部分平台预设带有「编程计划」标识，选择后会自动配置对应的配额查询。

## 透传配置

部分平台类型支持「纯透传」模式：AiDog 将客户端的请求头和请求体原样转发到上游，不做任何协议转换或认证注入。

- **上游地址** — 填写平台根地址（如 `https://api.anthropic.com`），不带版本前缀
- **API Key** — 透传模式下可留空，由客户端自行携带认证信息

适合需要保留客户端原始认证（如 OAuth 订阅）的场景。

## 多端点

每个平台可配置多个端点，每个端点有独立的地址和接口类型。AiDog 会根据客户端请求自动选择匹配的端点。

例如，同一个平台可以同时支持对话接口和编程助手接口。

## 模型配置

### 指定常用模型

在平台配置中，可以为不同类型的请求指定默认使用的模型。AiDog 会根据请求中的模型名自动匹配最合适的模型。

### 拉取模型列表

点击「拉取模型」按钮，AiDog 会从平台自动获取可用模型列表，方便选择。

### 模型映射

可以将客户端请求的模型名映射到平台实际支持的模型名。详见 [模型映射](/platforms/model-mapping)。

## 余额查询配置

对于使用 NewAPI 系统的平台，可以配置独立的余额查询：

- **查询地址** — 独立于 API 地址的余额查询入口
- **查询 Token** — 从控制台获取（与 API Key 不同）
- **用户 ID** — 可选

详见 [余额与配额](/stats/balance)。

## 手动预算

对于不支持自动余额查询的平台，可以设置手动预算限额：

- **总额** — 总花费上限
- **滑动窗口** — 滚动时间窗口内的花费上限
- **固定窗口** — 按固定周期重置的花费上限
- **每日** — 每日花费上限

预算耗尽后，AiDog 停止向该平台转发请求，并自动故障转移到组内其他平台。详见 [余额与配额](/stats/balance)。

## 模型测试

保存平台后，可以测试连通性：

1. 在平台卡片上点击「测试」按钮
2. 选择测试模式：快速 / 批量 / 自定义
3. 查看测试结果：响应时间、Token 数量、是否成功

详见 [模型测试](/platforms/model-testing)。

## 平台列表

平台页面展示所有已添加的平台卡片，每张卡片显示：

- 平台名称和 Logo
- 健康状态指示器（绿/黄/红）
- 余额或配额使用率
- 今日请求数和成功率

支持拼音搜索快速定位平台。

## 其他操作

- **启用/禁用** — 暂停某平台而不删除，禁用后不参与路由
- **编辑** — 点击平台卡片进入详情，修改后点击「保存」
- **删除** — 点击平台卡片右上角的删除按钮。相关日志和统计数据会保留
- **拖拽排序** — 在列表中拖拽卡片调整显示顺序

## 常见问题

### Base URL 怎么填？

Base URL 是 API 服务的完整基础路径，包含版本前缀：

| 平台 | Base URL |
|------|----------|
| OpenAI | `https://api.openai.com/v1` |
| Anthropic | `https://api.anthropic.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` |

:::caution{title="注意"}
不要在 Base URL 末尾加 `/chat/completions`，AiDog 会自动拼接。
:::

### API Key 安全

API Key 存储在本地，不会上传到任何服务器。
