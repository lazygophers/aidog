# 添加平台

平台是 AiDog 的核心概念 — 每个平台代表一个 AI API 服务商的接入点。

## 添加步骤

1. 进入 **「平台」** 页面
2. 点击右上角 **「+」** 按钮
3. 选择平台预设（或自定义填写）
4. 填写配置项，点击保存

:::tip{title="自动分组"}
保存平台后，AiDog 会自动为该平台创建一个分组，无需手动配置即可使用。
:::

## 基本配置

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 平台显示名称 | `My OpenAI` |
| Base URL | API 基础地址（含版本前缀） | `https://api.openai.com/v1` |
| API Key | 平台的 API Key | `sk-...` |
| 协议 | 请求协议类型 | `openai_chat` |

### 平台预设

AiDog 内置 50+ 平台预设。选择预设后，Base URL 和协议会自动填充。

### 协议选择

选择与目标平台匹配的协议。常见协议：

- **openai_chat** — OpenAI 兼容的 Chat Completions（最通用）
- **openai_responses** — OpenAI Responses API（Codex 使用）
- **anthropic** — Anthropic Claude 原生协议
- **gemini** — Google Gemini 原生协议
- **mock** — 模拟平台，用于测试，不发送真实请求

详见 [协议类型](/platforms/protocols)。

## 端点（Endpoints）

每个平台可配置多个端点，每个端点有独立的协议和 Base URL：

| 字段 | 说明 |
|------|------|
| 协议 | 该端点使用的出站协议 |
| Base URL | 该端点的 API 地址 |
| 客户端类型 | 请求头模拟方式（影响某些平台的兼容性） |
| Coding Plan | 该端点是否支持 Coding Plan 配额查询 |

例如，一个平台可以同时拥有 `openai_chat` 和 `openai_responses` 两个端点，AiDog 根据入站协议自动匹配对应的端点。

## 模型配置

### 模型槽位

平台支持 5 个模型槽位，用于快速指定常用模型：

| 槽位 | 用途 |
|------|------|
| **default** | 默认模型，测试和未指定模型时使用 |
| **sonnet** | Sonnet 级别模型（如 `claude-3-5-sonnet`） |
| **opus** | Opus 级别模型（如 `claude-3-opus`） |
| **haiku** | Haiku 级别模型（如 `claude-3-haiku`） |
| **gpt** | GPT 系列模型（如 `gpt-4o`） |

### 拉取模型列表

在编辑平台时，点击模型输入框旁的「拉取模型」按钮，AiDog 会从平台 API 自动获取可用模型列表，方便选择。

### 模型映射

在「模型映射」部分，可将请求中的模型名映射到平台实际支持的模型名。详见 [模型映射](/platforms/model-mapping)。

## 模型测试

保存平台后，可以测试连通性：

1. 在平台卡片上点击「测试」按钮
2. 选择测试模式：快速（默认模型）/ 批量（多个模型）/ 自定义（指定模型和提示词）
3. 查看测试结果：响应时间、Token 数量、是否成功

详见 [模型测试](/platforms/model-testing)。

## 其他操作

### 启用/禁用

暂停某平台而不删除，禁用后该平台不参与路由。

### 编辑平台

点击平台卡片进入详情，修改后点击「保存」。

### 删除平台

点击平台卡片右上角的删除按钮。删除后相关日志和统计数据会保留。

### 拖拽排序

在平台列表中拖拽平台卡片可调整显示顺序。

## 常见问题

### Base URL 怎么填？

Base URL 是 API 服务的完整基础路径，包含版本前缀：

| 平台 | Base URL |
|------|----------|
| OpenAI | `https://api.openai.com/v1` |
| Anthropic | `https://api.anthropic.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` |

:::caution{title="注意"}
不要在 Base URL 末尾加 `/chat/completions`，AiDog 会自动拼接。
:::

### API Key 安全

API Key 存储在本地数据库中，不会上传到任何服务器。

### Mock 平台有什么用？

Mock 平台模拟 AI 响应，不发送真实请求。适合在无网络环境或开发调试时使用。可以自定义响应内容、状态码、延迟、Token 数量，甚至模拟限速和超时错误。
