# 模型映射

模型映射允许你将客户端请求中的模型名称转换为平台实际支持的模型名。

## 使用场景

- 客户端请求 `gpt-4o`，但你希望路由到 DeepSeek 的 `deepseek-chat`
- 统一模型命名规范，屏蔽底层平台差异
- 通过别名简化模型选择（如 `fast` → `gpt-4o-mini`）

## 配置方式

在平台或分组配置的「模型映射」部分，添加映射规则：

| 请求模型 | 映射到 |
|----------|--------|
| `gpt-4o` | `deepseek-chat` |
| `claude-3-opus` | `claude-3-opus-20240229` |

### 映射规则

1. **精确匹配** — 请求模型名完全匹配时才映射
2. **保留原值** — 没有匹配规则的请求模型名保持不变
3. **优先级** — 分组级映射优先于平台级映射

## 两级映射

模型映射可以在两个层级配置：

- **平台级** — 该平台上所有请求的模型映射
- **分组级** — 仅在该分组的路由规则中生效

分组级映射优先于平台级映射。详见 [路由规则](/groups/routing-rules)。

## 模型别名示例

通过模型映射建立别名系统，客户端只需使用简短的别名：

| 别名 | 映射目标 | 用途 |
|------|---------|------|
| `fast` | `gpt-4o-mini` | 快速响应 |
| `smart` | `claude-3-opus` | 高质量 |
| `code` | `deepseek-coder` | 代码生成 |

## 链式映射

模型映射可以跨分组和平台级联：

1. 客户端请求模型 `smart`
2. 分组映射：`smart` → `claude-3-opus`
3. 平台映射：`claude-3-opus` → `claude-3-opus-20240229`
4. 发送到 Anthropic

详见 [高级路由](/proxy/advanced-routing)。
