# 路由规则

路由规则决定 AiDog 如何将入站请求匹配到对应的分组和平台。

## 匹配流程

```
入站请求 → 识别接口类型 → 匹配分组 → 选择平台 → 模型映射 → 转发请求
```

## 分组匹配

请求按以下优先级匹配分组：

1. **路径匹配** — 请求路径与分组路径前缀匹配
2. **模型名匹配** — 请求中的模型名在分组的模型映射表中
3. **默认分组** — 无匹配时使用默认分组
4. **第一个分组** — 未设置默认分组时使用第一个

## 平台选择

在分组内，按路由模式选择平台：

### 故障转移模式

选择第一个可用的平台发送请求。如果失败，自动尝试下一个。所有平台都失败则返回错误。

### 负载均衡模式

按权重随机选择一个平台。失败时同样会自动故障转移。

## 故障转移

触发自动故障转移的条件：
- 请求被限速（429）
- 服务器错误（5xx）
- 连接超时
- 网络错误

## 模型映射链

请求模型名的解析顺序：

1. **分组级映射** — 如果分组定义了模型映射
2. **平台级映射** — 如果平台定义了模型映射
3. **原始值** — 无映射则使用请求中的原始模型名

```
客户端请求: gpt-4o
  → 分组映射: gpt-4o → deepseek-chat
  → 平台映射: (无)
  → 发送到平台: deepseek-chat
```
