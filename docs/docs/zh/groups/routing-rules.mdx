# 路由规则

路由规则决定 AiDog 如何将入站请求匹配到对应的分组和平台。

## 匹配流程

```
入站请求 → 检测协议 → 匹配分组 → 选择平台 → 模型映射 → 转发请求
```

## 分组匹配

请求按以下优先级匹配分组：

1. **模型名匹配** — 请求中的模型名在分组的模型映射表中
2. **默认分组** — 无匹配时使用默认分组
3. **第一个分组** — 未设置默认分组时使用第一个

## 平台选择

在分组内，按策略选择平台：

1. 检查平台是否启用
2. 检查 API Key 是否有效
3. 按策略选择一个平台
4. 发送请求

如果选中的平台返回错误（如 429 限速），AiDog 会自动尝试下一个平台（故障转移）。

## 故障转移

分组内的平台按顺序排列。当前平台请求失败时：

1. 记录失败日志
2. 自动尝试下一个平台
3. 所有平台都失败则返回错误给客户端

## 模型映射链

请求模型名的解析顺序：

1. **分组级映射** — 如果分组定义了模型映射
2. **平台级映射** — 如果平台定义了模型映射
3. **原始值** — 无映射则使用请求中的原始模型名

```
客户端请求: gpt-4o
  → 分组映射: gpt-4o → deepseek-chat
  → 平台映射: (无)
  → 发送到平台: deepseek-chat
```
