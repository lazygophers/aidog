# 调度与熔断

AiDog 为分组内的多平台提供**智能调度**与**平台级熔断器**，在故障转移 / 负载均衡路由之上叠加健康感知与自动保护。

## 智能调度

调度器（`SchedulerState`）为每个平台维护实时健康指标：

| 指标 | 说明 |
| --- | --- |
| 延迟 EMA | 指数移动平均延迟（α = 0.3，约近 6 个样本） |
| 在途请求 | 当前未完成的请求数（inflight） |
| 连续失败 | 熔断器失败计数 |
| 熔断状态 | 三态机当前状态 |

选平台时综合健康指标排序，优先选延迟低、在途少、健康的平台。

## 熔断器三态机

每个平台独立维护熔断状态（`BreakerState`）：

```
Closed ──失败累计达阈值──▶ Open ──超时──▶ HalfOpen ──探测成功──▶ Closed
                                              └─探测失败──▶ Open
```

| 状态 | 字段 | 行为 |
| --- | --- | --- |
| **Closed** | `fails` | 正常放行，记录连续失败次数 |
| **Open** | `until_ms` | 熔断打开，候选过滤踢出（`Admission = Reject`） |
| **HalfOpen** | `probes` | 半开探测，限量放行（`Admission = Probe`） |

### 阈值配置（`BreakerThresholds`）

| 参数 | 说明 |
| --- | --- |
| `failure_threshold` | 触发熔断的连续失败次数 |
| `open_secs` | 熔断打开持续时间（秒） |
| `half_open_max` | 半开探测请求上限 |

## 准入判定（Admission）

候选平台选择时，调度器返回三态准入结果：

- **Allow** — 正常放行
- **Reject** — 平台熔断 Open，踢出候选
- **Probe** — 半开放行（限量探测）

## 与 auto_disabled 解耦

熔断器（`BreakerState`）与平台自动禁用（`auto_disabled`，鉴权失败触发）是**并集过滤**：

- 熔断 Open 的平台 → 临时踢出（可恢复）
- auto_disabled 的平台 → 持久禁用（需手动 / 鉴权恢复）
- 两者任一命中 → 不参与路由

两套机制解耦：熔断是健康感知的临时保护，auto_disabled 是鉴权失败的持久状态。

## 粘性会话（Sticky Session）

调度器内置粘性会话表（`StickyTable`）：

- 同一会话键的请求绑定到同一平台（保上下文一致）
- 内存 LRU + TTL 淘汰
- TTL：30 分钟无访问
- 容量上限：4096 条

## 配置入口

- **Platform 级**：平台编辑页配置熔断阈值
- **Group 级**：分组编辑页选择调度策略
- **全局默认**：设置页配默认阈值
