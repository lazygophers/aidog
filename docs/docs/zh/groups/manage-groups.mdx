# 创建与管理

分组是 AiDog 的路由核心 — 将多个平台按用途组织，配合规则控制请求路由。

## 创建分组

1. 进入 **「分组」** 页面
2. 点击 **「+ 添加分组」**
3. 填写名称和路径
4. 选择要加入的平台
5. 选择路由模式

:::tip{title="自动分组"}
添加平台时，AiDog 会自动创建一个对应的分组，无需手动操作。你也可以手动创建更多分组。
:::

## 路由模式

### 故障转移

默认模式。组内平台按排列顺序依次使用。当当前平台请求失败时，自动尝试下一个平台。适合主备场景。

```
请求 → Platform A (主) → 成功 → 返回
              ↓ (失败)
       Platform B (备) → 成功 → 返回
```

### 负载均衡

组内平台按权重分配请求。适合多平台分流场景。

| 平台 | 权重 | 预期流量 |
|------|------|---------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## 超时设置

每个分组可独立设置超时时间，覆盖全局设置：

- **请求超时** — 等待上游响应的最长时间
- **连接超时** — 建立连接的最长时间

留空则使用全局超时设置。

## 管理平台

分组内的平台支持：
- **拖拽排序** — 调整优先级（故障转移模式下决定主备顺序）
- **启用/禁用** — 临时跳过某平台
- **设置权重** — 负载均衡模式下的请求比例
- **模型映射** — 分组级别的模型映射，优先于平台级映射

## 一键启动命令

每个分组卡片上提供两个一键复制按钮：

- **Claude** — 复制该分组的 Claude Code 启动命令，包含对应的配置文件路径
- **Codex** — 复制该分组的 Codex 启动命令，包含对应的配置 profile

复制后在终端粘贴即可直接使用对应分组路由。

## 健康状态

分组卡片会显示关联平台的健康状态指示：

- 🟢 全部成功 — 绿色
- 🟡 存在失败 — 黄色
- 🔴 全部失败 — 红色

## 编辑与删除

- 点击分组卡片进入编辑模式，修改后点击「保存」
- 删除分组不会删除其中的平台，只是解除关联
- 在列表中拖拽分组卡片可调整显示顺序
