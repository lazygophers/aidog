# 配置与作用域

## 三级就近覆盖

中间件规则按作用域（`scope`）分层，执行时**后者覆盖前者**：

```
global（全局默认） → group（分组级） → platform（平台级）
```

| scope | scope_ref | 生效范围 |
| --- | --- | --- |
| `global` | 空 | 所有请求 |
| `group` | 分组名 | 该分组下的请求 |
| `platform` | 平台名 | 转发到该平台的请求 |

同一 `rule_type` 下，更具体的作用域规则覆盖更通用的。例如：全局禁用某模型，但某平台单独放行 → 平台规则覆盖全局。

## 前端 UI

中间件配置入口：

1. **全局默认**：设置页 → 中间件 tab（`AppSettings`）
2. **分组级**：分组编辑页嵌入中间件配置
3. **平台级**：平台编辑页嵌入中间件配置

每处可查看 / 新增 / 编辑 / 禁用 / 删除规则，作用域自动绑定到当前上下文。

## 规则编辑器

每条规则可配置：

- **类型**（`rule_type`）：8 种之一
- **匹配**（`match_type` + `pattern`）：包含 / 正则等
- **动作**（`action`）：warn / block / mask / inject 等
- **扩展配置**（`config`）：JSON，类型相关（如 `replacement`、注入点、错误分类策略）
- **优先级**（`priority`）：数值大者先执行
- **启用**（`enabled`）

## 性能与安全

- 规则正则启动时编译并缓存，请求时零编译开销
- DFA 大小限制 ≤ 1 MiB（`REGEX_DFA_SIZE_LIMIT`），防 ReDoS
- 正则源码大小限制 ≤ 1 MiB（`REGEX_SIZE_LIMIT`）
