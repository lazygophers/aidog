# 查看日志

**「日志」** 页面记录每一次经过 AiDog 的代理请求，便于排查路由、协议转换和上游响应问题。

## 日志列表

每条日志展示以下信息：

| 字段 | 说明 |
|------|------|
| 分组 | 命中的分组名称 |
| 模型 | 客户端请求的原始模型 |
| 实际模型 | 经模型映射后发往上游的模型 |
| 协议 | 入站协议 → 上游协议 |
| 平台 | 实际处理请求的平台 |
| 状态 | HTTP 状态码（成功 / 失败） |
| 耗时 | 请求总耗时（毫秒） |
| Token | 输入 / 输出 / 缓存 token 数 |
| 时间 | 请求发生时间 |

列表每页 50 条，支持翻页。

## 筛选

通过顶部筛选器快速定位日志：

- **平台**：按指定平台过滤
- **分组**：按分组过滤
- **状态**：仅成功 / 仅失败
- **时间范围**：全部 / 最近 1 小时 / 6 小时 / 24 小时 / 7 天 / 30 天
- **模型**：按原始模型或实际模型名称搜索

## 请求详情

点击任意一条日志展开详情，可查看完整的请求链路：

- **用户请求**：原始请求 URL、请求头、请求体
- **上游请求**：转换后发往上游的 URL、请求头、请求体
- **上游响应**：上游返回的状态码、响应头、响应体
- **用户响应**：最终返回给客户端的响应头、响应体

详情支持一键复制，便于粘贴到工单或调试工具。

:::tip
详情中记录的「用户请求」和「上游请求」内容受日志设置控制，默认不记录请求体。详见 [日志设置](/logs/log-settings)。
:::
