# 対応プラットフォームとインターフェース

AiDog は幅広い AI プラットフォームとインターフェースタイプをサポートし、異なるプラットフォーム間のプロトコル変換を自動処理します。

## プロトコル変換

AiDog は **N+N 中枢アーキテクチャ** を採用：全入力リクエストを一度中立 `ChatRequest` モデルに parse し、その後対象プラットフォームの wire プロトコルへ再 serialize します。プロトコル間の直接相互変換（N×N）ではなく、全て中立モデル経由で中継します。

```
クライアントリクエスト（5 種の wire プロトコルいずれか）
  ↓ parse_incoming_request
ChatRequest（中立モデル）
  ↓ convert_request
上流リクエスト（5 種の wire プロトコルいずれか）+ api_path

上流 SSE（5 種の wire プロトコルいずれか）
  ↓ parse_sse
ChatStreamEvent（中立）
  ↓ to_client_sse
クライアント SSE（入力プロトコルと一致）
```

### Wire プロトコル

endpoint の `protocol` フィールドで 5 種の wire プロトコルをサポート：

| Wire プロトコル | serde key | API path | 説明 |
|---|---|---|---|
| `Anthropic` | `anthropic` | `/v1/messages` | Claude 系ネイティブ形式 |
| `OpenAI` | `openai` | `/chat/completions` | デフォルトフォールバック；主要アグリゲータ |
| `OpenAIResponses` | `openai_responses` | `/v1/responses` | Codex / 新版 OpenAI コーディング接口 |
| `OpenAICompletions` | `openai_completions` | `/v1/completions` | レガシーテキスト補完 |
| `Gemini` | `gemini` | `/v1beta/models/{model}:streamGenerateContent` | Google Gemini ネイティブ |

### 変換マトリクス

中立モデルにより、入力プロトコルいずれからも出力プロトコルいずれへルーティング可能。非ストリーミングレスポンスの真変換は現在 **OpenAI → Anthropic** を完全実装（tool_calls と content 共存の互換バグ修正）；他のクロスプロトコルパスは実行時に上流 body のそのまま透過へフォールバックします。

| 入力 \ 出力 | anthropic | openai | openai_responses | openai_completions | gemini |
|---|:---:|:---:|:---:|:---:|:---:|
| anthropic | 透過 | ✓ | ✓ | ✓ | ✓ |
| openai | ✓ | 透過 | ✓ | ✓ | ✓ |
| openai_responses | ✓ | ✓ | 透過 | ✓ | ✓ |
| openai_completions | ✓ | ✓ | ✓ | 透過 | ✓ |
| gemini | ✓ | ✓ | ✓ | ✓ | 透過 |

### 特殊プラットフォーム（wire 層をバイパス）

以下の `platform_type` は `convert_request` / `forward_attempt` を経由せず、`proxy/handler.rs` のリトループ前に拦截されます：

| platform_type | 処理方式 |
|---|---|
| `mock` | 内蔵モックアダプタ、実上流なし |
| `claude_code` | OAuth 購読型 Claude Code リクエストの 1:1 純透過 |
| `devin` | `proxy/devin.rs` 内で chat ↔ session プロトコル変換（create session → poll → fetch messages → openai/anthropic へ包装） |
| `cli-proxy` | wire/base_url/key は候補解決時に関連 `cli_proxy_provider` 行から読取 |

## インターフェース自動検出

AiDog はクライアントリクエストのパスと body からインターフェースタイプを自動検出 — 手動設定不要：

- **チャットインターフェース**（`/v1/chat/completions`、`/v1/messages`）— 最も一般的な AI チャットリクエスト
- **コーディングアシスタントインターフェース**（`/v1/responses`、Codex / Claude Code）— コーディングツールが使用
- **テキスト補完インターフェース**（`/v1/completions`）— レガシーテキスト補完リクエスト

同じプロキシアドレスで異なるタイプのクライアントにサービス提供可能です。

## モックプラットフォーム

AiDog は実際のリクエストを送信せずに AI レスポンスをシミュレートするモックプラットフォームを提供。レスポンス内容、ステータスコード、遅延などをカスタマイズ可能。
