# モデルテスト

AiDog は、外部ツールなしでプラットフォームの接続性とモデル可用性を検証するビルトインモデルテスト機能を提供します。

## 使い方

**「プラットフォーム」** ページで、プラットフォームカードの「テスト」ボタンをクリックしてテストパネルを開きます。

## テストモード

| モード | 説明 |
|------|-------------|
| **クイックテスト** | プラットフォームのデフォルトモデルで短いリクエストを送り、接続性を素早く確認 |
| **単一モデル** | 特定のモデルをテスト |
| **バッチテスト** | 複数モデルを順次テストして可用性を検証 |
| **ランダムテスト** | プラットフォームの全利用可能モデルを一つずつテスト |
| **カスタムテスト** | モデル選択 + カスタムプロンプトと最大トークン数入力 |

## テスト結果

各結果は以下を表示:

- **モデル名** — テストに使用したモデル
- **ステータス** — 成功または失敗
- **応答時間** — 送信から完全受信までの総時間
- **トークン使用量** — 入力/出力トークン数
- **応答内容** — モデルが返したテキスト（折りたたみ可能）

## ユースケース

- **プラットフォーム追加後** — API Key と Base URL が正しいか確認
- **エンドポイント変更後** — 新エンドポイントが動作するか確認
- **モデルマッピング後** — マップ先モデルが利用可能か確認
- **トラブルシューティング** — プロキシ経由リクエスト失敗時、該当プラットフォームを先にテスト

## 注意

- テストはプラットフォームの設定済み API Key とエンドポイントを使用し、実際のトークンを消費
- モックプラットフォームのテストは実際のリクエストを送信しない
- バッチ・ランダムモードは複数モデルを順次テストし時間がかかる
