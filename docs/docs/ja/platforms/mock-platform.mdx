# モックプラットフォーム

モックプラットフォームは実際の AI リクエストを送信せず、事前設定されたレスポンスを返します。開発、デバッグ、ルーティングルールのテスト、クライアント統合の検証に最適。

## モックプラットフォーム作成

1. **「プラットフォーム」** ページへ
2. **「+」** ボタンをクリック
3. プラットフォームタイプリストから **「Mock」** を選択
4. API Key も Base URL も不要
5. 保存

保存後、AiDog が対応するグループを自動作成。クライアントをそのグループのパスに向けてモックを使用。

## レスポンス設定

レスポンスパラメータをカスタマイズ:

| パラメータ | 説明 | デフォルト |
|-----------|-------------|---------|
| レスポンステキスト | シミュレートされた応答テキスト | `Hello from mock` |
| ステータスコード | HTTP ステータスコード | `200` |
| レスポンス遅延 | シミュレートされた遅延 (ミリ秒) | `0` |
| 入力トークン | シミュレート入力トークン数 | `10` |
| 出力トークン | シミュレート出力トークン数 | `20` |
| キャッシュトークン | シミュレートキャッシュトークン | `0` |
| 終了理由 | シミュレート終了理由 (例 stop) | `stop` |
| ストリーミング | ストリームとして返すか (リクエストに従う / 強制 on/off) | リクエストに従う |

## エラーシミュレーション

各種エラーシナリオのシミュレーションをサポート:

| エラーモード | 説明 |
|------------|-------------|
| エラーなし | 正常応答を返す |
| HTTPエラー | 指定 HTTP エラーステータスを返す |
| Rate Limit (429) | レート制限をシミュレート |
| タイムアウト | リクエストタイムアウトをシミュレート |

クライアントのエラー処理や AiDog のフェイルオーバー動作テストに有用。

## ユースケース

- **開発** — 実 API Key なしでクライアント統合をテスト
- **ルーティングテスト** — グループマッチングとマッピングルールを検証
- **フェイルオーバーテスト** — エラーをシミュレートし自動切り替えを観察
- **パフォーマンステスト** — 遅延を設定し遅い応答下の挙動をテスト
