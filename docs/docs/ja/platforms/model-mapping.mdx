# モデルマッピング

モデルマッピングは、クライアントリクエストのモデル名を、プラットフォームが実際にサポートするモデル名に変換します。

## ユースケース

- クライアントが `gpt-4o` を要求するが、DeepSeek の `deepseek-chat` にルーティングしたい
- プラットフォーム間でモデル名を標準化
- エイリアスで選択を簡略化

## 設定

プラットフォームまたはグループの「モデルマッピング」セクションにルールを追加:

| 要求モデル | マップ先 |
|---------------|---------|
| `gpt-4o` | `deepseek-chat` |
| `claude-3-opus` | `claude-3-opus-20240229` |

### ルール

- **完全一致** — モデル名が完全一致した場合のみマップ
- **オリジナル保持** — 一致しない名前はそのまま通過
- **グループ優先** — グループレベルのマッピングがプラットフォームレベルより優先

## 2レベルマッピング

2つのレベルで設定可能:

- **プラットフォームレベル** — そのプラットフォームの全リクエストに適用
- **グループレベル** — そのグループのルーティングルール内のみ適用

[ルーティングルール](/ja/groups/routing-rules) 参照。

## エイリアス例

短いエイリアスを作成；クライアントはエイリアスを使うだけ:

| エイリアス | ターゲット | 用途 |
|-------|--------|---------|
| `fast` | `gpt-4o-mini` | 高速応答 |
| `smart` | `claude-3-opus` | 高品質 |
| `code` | `deepseek-coder` | コード生成 |
