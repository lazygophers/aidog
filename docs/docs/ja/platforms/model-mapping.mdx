# モデルマッピング

モデルマッピングにより、クライアントリクエストのモデル名をプラットフォームが実際にサポートするモデル名に変換できます。

## ユースケース

- クライアントが `gpt-4o` をリクエストするが、DeepSeek の `deepseek-chat` にルーティングしたい
- プラットフォーム間でモデル命名を統一
- 異なるプラットフォームの同等モデルを A/B テスト

## 設定方法

プラットフォーム設定の「モデルマッピング」セクションでマッピングルールを追加：

| リクエストモデル | マッピング先 |
|---------------|---------|
| `gpt-4o` | `deepseek-chat` |
| `claude-3-opus` | `claude-3-opus-20240229` |

### マッピングルール

1. **完全一致** — モデル名が完全に一致した場合のみマッピング
2. **元の値を保持** — マッピングルールに一致しないモデル名はそのまま
3. **優先順位** — プラットフォームレベルのマッピングが先、次にグループレベル

## グループでの使用

モデルマッピングは 2 つのレベルで設定可能：

- **プラットフォームレベル** — そのプラットフォームの全リクエストに適用
- **グループレベル** — そのグループのルーティングルール内でのみ適用

グループレベルのマッピングがプラットフォームレベルより優先されます。詳しくは [ルーティングルール](/groups/routing-rules) を参照。
