# 残高とクォータ

AiDog はプラットフォーム全体のアカウント残高と coding plan クォータを追跡し、支出を完全に可視化します。

## 自動残高照会

AiDog は以下プラットフォームの残高を自動照会します:

| プラットフォーム | 照会内容 |
|----------|---------------|
| OpenAI | アカウント残高 |
| DeepSeek | アカウント残高 |
| StepFun | アカウント残高 |
| SiliconFlow | アカウント残高 |
| OpenRouter | 残高 / 使用量 |
| Novita | アカウント残高 |
| Kimi (Moonshot) | アカウント残高 |
| GLM (Zhipu) | アカウント残高 |
| MiniMax | アカウント残高 |

残高データはスケジュールで自動更新されます。手動で更新ボタンをクリックすることも可能。

## Coding Plan クォータ

一部プラットフォームは coding plan を提供します (OpenAI Plus/Team/Enterprise、Zhipu Coding Plan、Kimi Code Plan など)。

AiDog はクォータ情報を照会・表示します:

- ティア名
- 現在の使用率
- 残りパーセンテージ

「Coding Plan」マーク付きのプラットフォームプリセットはクォータ照会を自動設定します。

## NewAPI 残高照会

NewAPI システムを使用するプラットフォーム (自己ホスト型 API アグリゲータなど) では、AiDog は別個の残高照会設定をサポートします:

- **照会 URL** — API URL とは別
- **照会トークン** — ダッシュボードから取得 (API Key とは別)
- **User ID** — 任意

プラットフォームの「残高照会設定」セクションに入力します。

## 手動予算

自動残高照会をサポートしないプラットフォーム向けに、AiDog は手動予算リミットを提供します。

### 設定

プラットフォームの「手動予算」セクションで「リミット追加」をクリック:

| タイプ | 説明 |
|------|-------------|
| 合計 | 全体的な支出上限 |
| ローリングウィンドウ | ローリング時間ウィンドウ内の支出上限 |
| 固定ウィンドウ | 固定スケジュールでリセットされる支出上限 |
| 日次 | 日次支出上限 |

### 予算単位

- **USD** — コストで測定
- **Tokens** — トークン数で測定

### ウィンドウ期間

ローリングウィンドウと固定ウィンドウは複数の時間粒度をサポート: 分、時間、日、週、月。

### 予算枯渇時

いずれかのリミットに達すると、AiDog は該当プラットフォームへのリクエスト転送を停止し (402 エラーを返す)、グループ内の他プラットフォームへ自動フェイルオーバーします。ウィンドウリセット時に転送が再開されます。

## 残高確認場所

残高情報は複数の場所に表示されます:

- **プラットフォームカード** — プラットフォームごとのリアルタイム残高とクォータ使用率
- **グループカード** — グループプラットフォームの集計残高
- **システムトレイ** — 設定可能な残高ステータス表示

## 統計ページ

**「統計」** ページで、プラットフォーム/グループ/モデルごとにデータを確認:

- 本日のリクエスト数と成功率
- トークン使用量 (入力 / 出力 / キャッシュ)
- 推定コスト
- 使用トレンドグラフ (日次 / 週次 / 月次)
