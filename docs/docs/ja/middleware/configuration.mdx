# 設定とスコープ

## 三階層カバー

ミドルウェアルールはスコープ（`scope`）で階層化、実行時に**後者が前者を上書き**：

```
global（グローバルデフォルト）→ group（グループ級）→ platform（プラットフォーム級）
```

| scope | scope_ref | 適用範囲 |
| --- | --- | --- |
| `global` | 空 | 全リクエスト |
| `group` | グループ名 | そのグループ配下のリクエスト |
| `platform` | プラットフォーム名 | そのプラットフォームへ転送されるリクエスト |

同一 `rule_type` 内で、より具体的なスコープがより一般的なものを上書き。例：グローバルで特定モデルを禁止しつつ、あるプラットフォームでのみ許可 → プラットフォームルールがグローバルを上書き。

## フロントエンド UI

ミドルウェア設定のエントリ：

1. **グローバルデフォルト**：設定ページ → ミドルウェアタブ（`AppSettings`）
2. **グループ級**：グループ編集ページに組み込み
3. **プラットフォーム級**：プラットフォーム編集ページに組み込み

各所でルールの表示/追加/編集/無効化/削除が可能、スコープは現在のコンテキストへ自動バインド。

## ルールエディタ

各ルールで設定可能：

- **タイプ**（`rule_type`）：8 種のいずれか
- **マッチ**（`match_type` + `pattern`）：contains / regex 等
- **アクション**（`action`）：warn / block / mask / inject 等
- **拡張設定**（`config`）：JSON、タイプ依存（`replacement`、注入ポイント、エラー分類戦略等）
- **優先度**（`priority`）：大きいほど先に実行
- **有効**（`enabled`）

## パフォーマンスとセキュリティ

- ルールの正規表現は起動時にコンパイル・キャッシュ、リクエスト時にコンパイルコストゼロ
- DFA サイズ制限 ≤ 1 MiB（`REGEX_DFA_SIZE_LIMIT`）、ReDoS を防止
- 正規表現ソースサイズ制限 ≤ 1 MiB（`REGEX_SIZE_LIMIT`）
