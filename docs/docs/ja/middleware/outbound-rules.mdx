# アウトバウンドルール

アウトバウンドルールは**レスポンスが上游から戻った後、クライアントへ転送する前**に実行されます。Engine は 3 種のアウトバウンドルールを固定順で適用：

```
ResponseOverride → Rectifier → ErrorRule
```

## response_override — レスポンス上書き

成功レスポンスボディを書き換え。`pattern` でレスポンステキストをマッチ、ヒット時 `config` の内容へ置換。フィールド形式の統一やレスポンス内の機密断片のマスキングに有用。

## rectifier — 補正器

上游レスポンスの形式欠陥を修復：

- **SSE ストリーム修復**：欠落した `data:` プレフィックス、イベント境界を補完
- **JSON 修復**：トランケーション/不正エスケープを修復
- **エンコーディング修復**：文字エンコーディングを統一
- **フィールド欠落修復**：必須フィールドを補完

非標準レスポンスを返す上游プラットフォームとの互換用。

## error_rule — エラー分類

上游エラーレスポンス（非 2xx）を分類：

- **リトライ**：リトライ可能と判定（429 / 503 等）→ プラットフォーム間フェイルオーバーリトライをトリガ
- **サーキットブレーク**：プラットフォーム障害と判定 → サーキットブレーカーの失敗カウントに加算（[スケジューリングとサーキットブレーク](/ja/groups/scheduling) 参照）
- **ステータスコード上書き**：上游ステータスコードをクライアント可視コードへマッピング

`config` で `retry` / `circuit_break` / `status_override` ルールを指定可。デフォルトの `error_rules` セットが組み込み（[組み込みプリセット](/ja/middleware/builtin-presets) 参照）。

## チャンク単位ストリーミング

SSE ストリーミングレスポンスに対し、Engine は**チャンクごと**にアウトバウンドルールを適用（`rewrite_chunk_text`）、レスポンス全体をバッファリングしません：

- 各 SSE チャンク到達時にルールを通す
- ストリーミングセマンティクスを維持、クライアントはトークンをリアルタイム受信
- 正規表現サイズ制限（DFA ≤ 1 MiB）、ReDoS を防止
