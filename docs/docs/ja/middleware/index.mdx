# ミドルウェアルールエンジン

AiDog は設定可能なリクエスト/レスポンスミドルウェアルールエンジンを内蔵し、プロキシ転送チェーンの**インバウンド**（リクエスト流入）と**アウトバウンド**（レスポンス返却）の 2 段階でルールを実行します：正規化、上書き、マスキング、注入、センシティブワードフィルタ、エラー検出、サーキットブレーク。すべて UI で設定可能、コード不要。

## アーキテクチャ

| 特徴 | 説明 |
| --- | --- |
| 単一テーブル格納 | 全ルールは SQLite `middleware_rule` テーブルに保存 |
| Engine シングルトン | `MiddlewareEngine` が起動時にルールを正規表現へコンパイル、リクエスト時に優先度順でマッチ |
| 双方向マウント | インバウンド（リクエストボディ書き換え）+ アウトバウンド（レスポンスボディ/SSE ストリーム書き換え） |
| 三階層カバー | `global` → `group` → `platform`、後者が前者の同名ルールを上書き |
| チャンク単位ストリーミング | SSE レスポンスをチャンクごとに処理、バッファリング不要 |
| 組み込みプリセット | API キー/メール/電話マスキング + デフォルトエラー分類ルール |

## ルールタイプ

8 種の `RuleType` を実行段階別に定義：

| タイプ | 段階 | 目的 |
| --- | --- | --- |
| `request_filter` | インバウンド | リクエストフィールドフィルタ（model 許可/拒否リスト等） |
| `sensitive_word` | インバウンド | センシティブワードブロック（pattern が単語そのもの） |
| `redaction` | インバウンド | フィールド値マスキング |
| `content_filter` | インバウンド | コンテンツフィルタ |
| `dynamic_injection` | インバウンド | 動的注入（system / header / body） |
| `response_override` | アウトバウンド | レスポンスボディ上書き |
| `rectifier` | アウトバウンド | 補正器（SSE / JSON / エンコーディング / フィールド欠落修復） |
| `error_rule` | アウトバウンド | エラー分類（リトライ/サーキットブレーク/ステータスコード上書き） |

## ルールフィールド

各ルール（`middleware_rule` テーブル）のフィールド：

| フィールド | 説明 |
| --- | --- |
| `name` / `description` | 名称と説明 |
| `rule_type` | ルールタイプ（上表参照） |
| `scope` | スコープ：`global` / `group` / `platform` |
| `scope_ref` | スコープ参照（group/platform 名、global は空） |
| `match_type` | マッチ方式（`contains` / `regex` 等） |
| `pattern` | マッチパターン |
| `action` | マッチ時アクション（`warn` / `block` / `mask` / `inject` 等） |
| `config` | JSON 拡張設定 |
| `priority` | 優先度（大きいほど先に実行） |
| `enabled` | 有効スイッチ |
| `is_builtin` | 組み込みプリセットか（削除不可、無効化可） |
