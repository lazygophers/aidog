# グループの作成と管理

グループは AiDog のルーティングコア — プラットフォームを用途で整理し、ルールでルーティングを制御します。

## グループ作成

1. **「グループ」** ページへ
2. **「+ グループ追加」** をクリック
3. 名前とパスを入力
4. 含めるプラットフォームを選択
5. ルーティングモードを選択

:::tip{title="自動グループ"}
プラットフォーム追加時、AiDog が対応グループを自動作成。手動でさらにグループ作成も可能。
:::

## ルーティングモード

### フェイルオーバー

デフォルトモード。プラットフォームを順序通り使用。現行が失敗すると次を自動試行。プライマリ/バックアップ構成に最適。

```
リクエスト → プラットフォーム A (プライマリ) → 成功 → 返却
                ↓ (失敗)
         プラットフォーム B (バックアップ) → 成功 → 返却
```

### ロードバランス

リクエストを重み付きで分散。マルチプラットフォームのトラヒック分割に最適。

| プラットフォーム | 重み | 期待トラヒック |
|----------|--------|-----------------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## タイムアウト設定

各グループはタイムアウトを個別に設定でき、グローバル設定を上書き:

- **リクエストタイムアウト** — 上流応答の最大待機時間
- **接続タイムアウト** — 接続確立の最大時間

空欄でグローバルタイムアウト使用。

## プラットフォーム管理

グループ内で:

- **ドラッグで並び替え** — 優先度調整 (フェイルオーバーでプライマリ/バックアップ順を決定)
- **有効化/無効化** — プラットフォームを一時スキップ
- **重み設定** — ロードバランスのリクエスト比
- **モデルマッピング** — グループレベル、プラットフォームより優先

## ワンクリック起動コマンド

各グループカードに2つのワンクリックコピーボタン:

- **Claude** — このグループの Claude Code 起動コマンド (設定ファイルパス含む) をコピー
- **Codex** — このグループの Codex 起動コマンド (プロファイル含む) をコピー

ターミナルに貼り付け、即座にグループのルーティングを使用。

## ヘルスステータス

グループカードは関連プラットフォームの状態を表示:

- 🟢 全成功 — 緑
- 🟡 一部失敗 — 黄
- 🔴 全失敗 — 赤

## 編集と削除

- グループカードをクリックで編集モード。「保存」で反映
- グループ削除はプラットフォームを削除しない — 関連付けのみ削除
- リスト内のカードをドラッグで並び替え
