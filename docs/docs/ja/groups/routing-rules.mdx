# ルーティングルール

ルーティングルールは、AiDog が受信リクエストを適切なグループとプラットフォームにマッチさせる方法を決定します。

## マッチングフロー

```
受信リクエスト → インターフェース検出 → グループマッチ → プラットフォーム選択 → モデルマッピング → 転送
```

## グループマッチング

リクエストは優先度順にグループへマッチ:

1. **パスマッチ** — リクエストパスがグループの設定パスプレフィックスに一致
2. **モデル名マッチ** — リクエストモデル名がいずれかのグループのマッピングテーブルにある
3. **デフォルトグループ** — マッチしない場合に使用
4. **最初のグループ** — デフォルトグループ未設定時に使用

## プラットフォーム選択

グループ内ではルーティングモードで選択:

### フェイルオーバーモード

最初の利用可能プラットフォームを選択。失敗時、次を自動試行。全プラットフォーム失敗時はエラーを返す。

### ロードバランシングモード

重み付きランダムでプラットフォーム選択。失敗時もフェイルオーバー適用。

## フェイルオーバー

以下により自動フェイルオーバー発動:

- レート制限 (429)
- サーバーエラー (5xx)
- 接続タイムアウト
- ネットワークエラー

## モデルマッピングチェーン

モデル名の解決順:

1. **グループレベルマッピング** — グループがマッピングを定義している場合
2. **プラットフォームレベルマッピング** — プラットフォームがマッピングを定義している場合
3. **オリジナル値** — リクエストの元モデル名を使用

```
クライアント要求: gpt-4o
  → グループマッピング: gpt-4o → deepseek-chat
  → プラットフォームマッピング: (なし)
  → プラットフォームへ送信: deepseek-chat
```
