# ルーティングルール

ルーティングルールは、インバウンドリクエストを適切なグループとプラットフォームにマッチングする方法を決定します。

## マッチングフロー

```
インバウンドリクエスト → プロトコル検出 → グループマッチング → プラットフォーム選択 → モデルマッピング → 転送
```

## グループマッチング

リクエストは以下の優先順位でグループにマッチングされます：

1. **モデル名マッチ** — リクエストのモデル名がグループのモデルマッピングテーブルにある
2. **デフォルトグループ** — マッチがない場合に使用
3. **最初のグループ** — デフォルトグループが未設定の場合に使用

## プラットフォーム選択

グループ内で、戦略に従ってプラットフォームを選択：

1. プラットフォームが有効かチェック
2. API キーが有効かチェック
3. 戦略に従ってプラットフォームを 1 つ選択
4. リクエストを送信

選択されたプラットフォームがエラー（例：429 レートリミット）を返した場合、AiDog は自動的に次のプラットフォームを試行します（フェイルオーバー）。

## フェイルオーバー

グループ内のプラットフォームは順序付きです。現在のプラットフォームが失敗した場合：

1. 失敗をログに記録
2. 自動的に次のプラットフォームを試行
3. 全プラットフォームが失敗した場合、クライアントにエラーを返す

## モデルマッピングチェーン

モデル名の解決順序：

1. **グループレベルのマッピング** — グループがモデルマッピングを定義している場合
2. **プラットフォームレベルのマッピング** — プラットフォームがモデルマッピングを定義している場合
3. **元の値** — マッピングがない場合、リクエストの元のモデル名を使用

```
クライアントリクエスト: gpt-4o
  → グループマッピング: gpt-4o → deepseek-chat
  → プラットフォームマッピング: (なし)
  → プラットフォームに送信: deepseek-chat
```
