# スケジューリングとサーキットブレーク

AiDog はグループ内の複数プラットフォームに対し**スマートスケジューリング**と**プラットフォーム級サーキットブレーカー**を提供し、フェイルオーバー/ロードバランスルーティングの上にヘルス認識と自動保護を重ねます。

## スマートスケジューリング

スケジューラ（`SchedulerState`）はプラットフォームごとにリアルタイムヘルス指標を維持：

| 指標 | 説明 |
| --- | --- |
| レイテンシ EMA | 指数移動平均レイテンシ（α = 0.3、直近約 6 サンプル） |
| 実行中リクエスト | 現在未完了のリクエスト数（inflight） |
| 連続失敗 | サーキットブレーカーの失敗カウント |
| ブレーク状態 | 三状態マシンの現在状態 |

プラットフォーム選択時、ヘルス指標を統合してソート — レイテンシ低・実行中少・健全なプラットフォームを優先。

## サーキットブレーカー三状態マシン

各プラットフォームは独立してブレーク状態（`BreakerState`）を維持：

```
Closed ──失敗が閾値到達──▶ Open ──タイムアウト──▶ HalfOpen ──プローブ成功──▶ Closed
                                                    └─プローブ失敗──▶ Open
```

| 状態 | フィールド | 挙動 |
| --- | --- | --- |
| **Closed** | `fails` | 正常通行、連続失敗を記録 |
| **Open** | `until_ms` | ブレークオープン、候補から除外（`Admission = Reject`） |
| **HalfOpen** | `probes` | ハーフオープンプローブ、限量通行（`Admission = Probe`） |

### 閾値設定（`BreakerThresholds`）

| パラメータ | 説明 |
| --- | --- |
| `failure_threshold` | ブレーカをトリガする連続失敗数 |
| `open_secs` | ブレークオープン持続時間（秒） |
| `half_open_max` | ハーフオープン中のプローブリクエスト上限 |

## アドミッション（Admission）

候補プラットフォーム選択時、スケジューラは三状態のアドミッション結果を返します：

- **Allow** — 正常通行
- **Reject** — プラットフォームブレーク Open、候補から除外
- **Probe** — ハーフオープン通行（限量プローブ）

## auto_disabled との疎結合

ブレーカ（`BreakerState`）とプラットフォーム自動無効化（`auto_disabled`、認証失敗でトリガ）は**和集合**でフィルタ：

- ブレーク Open のプラットフォーム → 一時除外（回復可能）
- auto_disabled のプラットフォーム → 永続無効化（手動/認証回復が必要）
- いずれかヒット → ルーティングされない

2 つの機構は疎結合：サーキットブレークはヘルス認識の一時保護、auto_disabled は認証失敗の永続状態。

## スティッキーセッション

スケジューラは組み込みのスティッキーセッションテーブル（`StickyTable`）を持ちます：

- 同一セッションキーのリクエストは同一プラットフォームへバインド（コンテキスト整合性を保持）
- メモリ LRU + TTL 破棄
- TTL：30 分間アクセスなし
| 容量上限：4096 件

## 設定エントリ

- **プラットフォーム級**：プラットフォーム編集ページでブレーク閾値を設定
- **グループ級**：グループ編集ページでスケジューリング戦略を選択
- **グローバルデフォルト**：設定ページでデフォルト閾値を設定
