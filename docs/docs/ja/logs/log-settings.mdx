# ログ設定

**設定** ページでログの粒度と保持を制御し、可観測性とプライバシー/ディスク使用量のバランスを取れます。

## 記録レベル

ログには段階的に有効になる3レベルのスイッチがあります:

| レベル | 説明 | デフォルト |
|------|------|------|
| マスタースイッチ | オフ時、プロキシログは一切記録されない | オン |
| 元のユーザーリクエストを記録 | クライアントが送信した元のリクエストヘッダーとボディを記録 | オフ |
| 上流リクエストを記録 | 上流へ送信された変換後のリクエストヘッダーとボディを記録 | オフ |

:::warning
リクエストボディには機密内容 (プロンプト、キーなど) が含まれる可能性があります。「元のユーザーリクエストを記録」または「上流リクエストを記録」を有効にする前に、ローカル環境が信頼できることを確認してください。
:::

## 保持

ログは3つの独立した保持期間に従ってクリーンアップされます:

| タイプ | デフォルト | クリーンアップ方法 |
|------|------|----------|
| 元のユーザーリクエスト | 7日 | ボディフィールドのみクリア、ログ行は保持 |
| 上流リクエスト | 7日 | ボディフィールドのみクリア、ログ行は保持 |
| ログレコード | 90日 | ログエントリ全体を削除 |

つまり、7日より古いリクエストボディはクリアされ (統計は保持)、90日より古いログは完全に削除されます。

## 手動クリーンアップ

**設定 → データ管理** で「期限切れログをクリーンアップ」をいつでも手動実行でき、上記の保持ルールを即座に適用します。

## データ保存

すべてのログはローカルデータベース `~/.aidog/aidog.db` に保存され、サーバーへアップロードされません。
