# ログ設定

**「設定」** ページでログの記録粒度と保持期間を制御し、可観測性とプライバシー / ディスク使用量のバランスを取ります。

## 記録レベル

ログの記録には 3 段階のスイッチがあり、上位から順に有効になります。

| レベル | 説明 | デフォルト |
|------|------|------|
| マスタースイッチ | オフにすると、いかなるプロキシログも一切記録しません | オン |
| ユーザー元リクエストを記録 | クライアントから送られた元のリクエストヘッダーとリクエストボディを記録します | オフ |
| アップストリームリクエストを記録 | 変換後にアップストリームへ送信したリクエストヘッダーとリクエストボディを記録します | オフ |

:::warning
リクエストボディには機微な内容（プロンプト、シークレットなど）が含まれる場合があります。「ユーザー元リクエストを記録」または「アップストリームリクエストを記録」を有効にする前に、本機の環境が信頼できることを確認してください。
:::

## 保持期間

ログは 3 種類の独立した保持期間に従ってクリーンアップされます。

| 種類 | デフォルト | クリーンアップ方法 |
|------|------|----------|
| ユーザー元リクエスト | 7 日 | リクエストボディフィールドのみをクリアし、ログ行は保持 |
| アップストリームリクエスト | 7 日 | リクエストボディフィールドのみをクリアし、ログ行は保持 |
| ログ記録 | 90 日 | ログ全体を削除 |

つまり、7 日を超えたリクエストボディはクリアされ（統計情報は引き続き保持されます）、90 日を超えたログはログ全体が削除されます。

## 手動クリーンアップ

**「設定」→「データ管理」** からいつでも手動で「期限切れログのクリーンアップ」を実行でき、上記の保持ルールに従って即座にクリーンアップを行います。

## データ保存

すべてのログはローカルデータベース `~/.aidog/aidog.db` に保存され、いかなるサーバーにもアップロードされません。
