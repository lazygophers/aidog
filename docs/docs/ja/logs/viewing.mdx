# ログの確認

**「ログ」** ページには、AiDog を経由したすべてのプロキシリクエストが記録され、ルーティング・プロトコル変換・アップストリーム応答の問題を調査しやすくなっています。

## ログ一覧

各ログには以下の情報が表示されます。

| フィールド | 説明 |
|------|------|
| グループ | 一致したグループ名 |
| モデル | クライアントがリクエストした元のモデル |
| 実際のモデル | モデルマッピングを経てアップストリームに送信されたモデル |
| プロトコル | インバウンドプロトコル → アップストリームプロトコル |
| プラットフォーム | 実際にリクエストを処理したプラットフォーム |
| ステータス | HTTP ステータスコード（成功 / 失敗） |
| 所要時間 | リクエストの総所要時間（ミリ秒） |
| Token | 入力 / 出力 / キャッシュの token 数 |
| 時刻 | リクエストが発生した時刻 |

一覧は 1 ページあたり 50 件で、ページ送りに対応しています。

## フィルタリング

上部のフィルターを使って、ログをすばやく絞り込めます。

- **プラットフォーム**：指定したプラットフォームで絞り込み
- **グループ**：グループで絞り込み
- **ステータス**：成功のみ / 失敗のみ
- **期間**：すべて / 直近 1 時間 / 6 時間 / 24 時間 / 7 日 / 30 日
- **モデル**：元のモデル名または実際のモデル名で検索

## リクエスト詳細

任意のログをクリックして詳細を展開すると、リクエストの完全な経路を確認できます。

- **ユーザーリクエスト**：元のリクエスト URL、リクエストヘッダー、リクエストボディ
- **アップストリームリクエスト**：変換後にアップストリームへ送信した URL、リクエストヘッダー、リクエストボディ
- **アップストリーム応答**：アップストリームが返したステータスコード、レスポンスヘッダー、レスポンスボディ
- **ユーザー応答**：最終的にクライアントへ返したレスポンスヘッダー、レスポンスボディ

詳細はワンクリックでコピーでき、チケットやデバッグツールへの貼り付けに便利です。

:::tip
詳細に記録される「ユーザーリクエスト」と「アップストリームリクエスト」の内容はログ設定によって制御され、デフォルトではリクエストボディは記録されません。詳しくは [ログ設定](/logs/log-settings) を参照してください。
:::
