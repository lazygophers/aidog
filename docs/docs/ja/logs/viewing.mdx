# ログの表示

**ログ** ページは AiDog を通過するすべてのプロキシリクエストを記録し、ルーティング、プロトコル変換、上流レスポンスの問題の診断を容易にします。

## ログ一覧

各ログエントリは以下を表示します:

| フィールド | 説明 |
|------|------|
| グループ | マッチしたグループ名 |
| モデル | クライアントが要求した元のモデル |
| 実モデル | モデルマッピング後に上流へ送信されたモデル |
| プロトコル | 入力プロトコル → 上流プロトコル |
| プラットフォーム | リクエストを実際に処理したプラットフォーム |
| ステータス | HTTPステータスコード (成功 / 失敗) |
| 所要時間 | 総リクエスト時間 (ミリ秒) |
| トークン | 入力 / 出力 / キャッシュトークン数 |
| 時刻 | リクエスト発生時刻 |

一覧はページあたり50件、ページネーション対応。

## フィルタリング

上部のフィルタを使ってログをすばやく特定できます:

- **プラットフォーム**: 特定プラットフォームで絞り込み
- **グループ**: グループで絞り込み
- **ステータス**: 成功のみ / 失敗のみ
- **時間範囲**: すべて / 過去1時間 / 6時間 / 24時間 / 7日 / 30日
- **モデル**: 元または実モデル名で検索

## リクエスト詳細

任意のログをクリックして詳細を展開し、完全なリクエストチェーンを確認:

- **ユーザーリクエスト**: 元のリクエストURL、ヘッダー、ボディ
- **上流リクエスト**: 変換されたURL、ヘッダー、上流へ送信されたボディ
- **上流レスポンス**: ステータスコード、ヘッダー、上流から返されたボディ
- **ユーザーレスポンス**: クライアントへ返す最終ヘッダーとボディ

詳細はワンクリックコピー対応、サポートチケットやデバッグツールに貼り付けやすい。

:::tip
詳細に記録される「ユーザーリクエスト」「上流リクエスト」の内容はログ設定で制御されます。デフォルトではリクエストボディは記録されません。[ログ設定](/ja/logs/log-settings) 参照。
:::
