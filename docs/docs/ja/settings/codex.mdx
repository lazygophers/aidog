# Codex

**「設定」→「Codex」** タブで OpenAI Codex CLI との統合を設定します。

## 設定編集

Codex の `~/.codex/config.toml` 設定を AiDog 内で直接編集:

- **Provider** — API プロバイダーとベース URL
- **モデル** — 使用するモデル名
- **その他のオプション** — Codex がサポートするすべての設定項目

変更は設定ファイルに即座に書き込まれます — ファイルを手動で探して編集する必要なし。

## クイック設定

最小設定はプロキシアドレスと任意の API キーのみ:

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

## グループ設定

Codex はプロファイル経由でグループ切り替えをサポートします:

1. グループカードの「Codex」ボタンをクリックして起動コマンドをコピー
2. ターミナルに貼り付け:

```bash
AIDOG_KEY='<group-name>' codex -p '<group-name>' \
  --dangerously-bypass-approvals-and-sandbox -a never
```

AiDog は `AIDOG_KEY` 環境変数に基づき対応グループへリクエストをルーティングします。

## 注意

- Codex は Responses API (`/v1/responses`) を使用、AiDog が自動検出してルーティング
- 対応プラットフォームが AiDog に追加済みか確認
- 必要に応じてプラットフォームまたはグループ設定でモデルマッピングを追加
- プロキシアドレスの `/proxy/v1` パスプレフィックスは省略不可
