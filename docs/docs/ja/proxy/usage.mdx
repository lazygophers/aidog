# プロキシの使用

AiDog はローカル HTTP プロキシサーバを実行し、AI API リクエストを受信・転送します。

## プロキシアドレス

デフォルトアドレス:

```
http://127.0.0.1:9876
```

ポートは **「設定 → プロキシ」** で変更可能。

## 自動起動

設定で「プロキシ自動起動」を有効化。有効時、AiDog 起動とともにプロキシサービスが自動開始。

## 基本的な使い方

カスタム API アドレスをサポートする任意の AI クライアントを AiDog に向け:

| クライアント種別 | 設定 |
|-------------|---------------|
| OpenAI SDK | `base_url` をプロキシアドレスに設定 |
| curl | プロキシアドレスへ直接リクエスト |
| Web アプリ | API エンドポイントをプロキシに設定 |
| コーディングアシスタント | API ベース URL を設定 |

### Python (OpenAI SDK)

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:9876/proxy/v1",
    api_key="any-value"  # AiDog は設定済みの実際のキーを使用
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello!"}]
)
```

### Node.js

```javascript
import OpenAI from 'openai';

const client = new OpenAI({
  baseURL: 'http://127.0.0.1:9876/proxy/v1',
  apiKey: 'any-value',
});

const response = await client.chat.completions.create({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'Hello!' }],
});
```

### curl

```bash
curl http://127.0.0.1:9876/proxy/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer any-value" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

## ストリーミング

AiDog は SSE (Server-Sent Events) ストリーミングを完全サポート。クライアントのストリーミングリクエストは上流プラットフォームへ透明転送。

## タイムアウト

プロキシリクエストにはタイムアウト保護あり:

- デフォルトタイムアウト: 300 秒
- グローバルまたはグループごとに設定可能
- 期限切れで 504 Gateway Timeout を返す
