# 高度なルーティング

AiDog の高度なルーティング機能を使いこなし、きめ細かいトラヒック制御を実現。

## マルチプラットフォーム集約

複数プラットフォームを異なるルーティングモードでグループに追加。[作成と管理](/ja/groups/manage-groups) 参照。

## フェイルオーバー

プラットフォーム失敗時、AiDog はグループ内の次を自動試行:

```
リクエスト → プラットフォーム A (429 Rate Limit)
        → プラットフォーム B (成功)
        → レスポンス返却
```

フェイルオーバー発動条件:

- HTTP 429 (Rate Limit)
- HTTP 5xx (サーバーエラー)
- 接続タイムアウト
- ネットワークエラー

## ロードバランシング

リクエストを重み付きでプラットフォーム間に分散。例:

| プラットフォーム | 重み | 期待トラヒック |
|----------|--------|------------------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## モデルエイリアス

モデルマッピングでエイリアス作成:

| エイリアス | マップ先 | 用途 |
|-------|---------|---------|
| `fast` | `gpt-4o-mini` | 高速応答 |
| `smart` | `claude-3-opus` | 高品質 |
| `code` | `deepseek-coder` | コード生成 |

クライアントはエイリアス使用; AiDog はグループ設定に基づき実際のプラットフォームへルーティング。

## チェーンルーティング

グループとプラットフォームのマッピングを組み合わせ多層変換:

1. クライアントが `smart` を要求
2. グループ A が `smart` → `claude-3-opus` にマップ
3. プラットフォーム X が `claude-3-opus` → `claude-3-opus-20240229` にマップ
4. Anthropic へ送信

## グループレベルタイムアウト

各グループはリクエストタイムアウトと接続タイムアウトを個別に設定でき、グローバルデフォルトを上書き。遅いモデルが長い待機時間を必要とする場合に有用。

[作成と管理](/ja/groups/manage-groups) 参照。
