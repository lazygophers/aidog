# 高度なルーティング

AiDog の高度なルーティング機能を活用して、きめ細かなトラフィック制御を実現しましょう。

## マルチプラットフォーム負荷分散

グループに複数のプラットフォームを追加し、異なるルーティング戦略を組み合わせて負荷分散を実現：

### 順次

プラットフォームが順番にリクエストを受信します。プライマリ/バックアップ構成に最適。

### ランダム

リクエストがランダムにプラットフォームに分配されます。均等分配に適しています。

### 重み付き

重み比率でリクエストを分配します。例：

| プラットフォーム | 重み | 期待トラフィック |
|----------|--------|------------------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## フェイルオーバー

プラットフォームのリクエストが失敗した場合、AiDog はグループ内の次のプラットフォームを自動的に試行します：

```
リクエスト → プラットフォーム A (429 レートリミット)
          → プラットフォーム B (成功)
          → レスポンス返却
```

フェイルオーバーがトリガーされる条件：
- HTTP 429 (レートリミット)
- HTTP 5xx (サーバーエラー)
- 接続タイムアウト
- ネットワークエラー

## モデルエイリアス

モデルマッピングでエイリアスを作成：

| エイリアス | マッピング先 | 用途 |
|-------|---------|---------|
| `fast` | `gpt-4o-mini` | 高速応答 |
| `smart` | `claude-3-opus` | 高品質 |
| `code` | `deepseek-coder` | コード生成 |

クライアントはエイリアスを使用するだけで、AiDog がグループ設定に基づいて実際のプラットフォームにルーティングします。

## チェーンルーティング

グループとプラットフォームのマッピングを組み合わせ：

1. クライアントがモデル `smart` をリクエスト
2. グループ A が `smart` → `claude-3-opus` にマッピング
3. プラットフォーム X が `claude-3-opus` → `claude-3-opus-20240229` にマッピング
4. Anthropic に送信
