# Codex 連携

AiDog は OpenAI Codex CLI、Claude Code、その他 AI コーディングアシスタントをネイティブサポートし、ビルトイン設定エディタを提供します。

## OpenAI Codex

### 設定

Codex の設定ファイル `~/.codex/config.toml` に以下を設定:

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

### Codex 設定ページ

AiDog は **「設定 → Codex」** にビルトイン Codex 設定エディタを提供:

- `~/.codex/config.toml` 設定を直接編集
- ファイルを手動で探して編集する必要なし
- 保存後すぐに反映

### 注意

- Codex は **Responses API** (`/v1/responses`) を使用。AiDog が自動検出・ルーティング。
- 対象プラットフォームに `openai_responses` プロトコルのエンドポイントが設定されていることを確認。
- 必要に応じてプラットフォーム/グループレベルでモデルマッピングを追加。

## Claude Code

### ワンクリック設定エクスポート

AiDog の **「設定 → Claude Code」** タブで「設定エクスポート」をクリック。AiDog がプロキシアドレスを Claude Code 設定ファイル (`~/.claude.json`) に自動書き込み。

### 手動設定

環境変数でも設定可能:

```bash
export ANTHROPIC_BASE_URL="http://127.0.0.1:9876"
export ANTHROPIC_API_KEY="any-value"
```

### Claude Code 設定ページ

AiDog は **「設定 → Claude Code」** に専用エディタを提供:

- **設定編集** — Claude Code 設定 (モデル、権限、サンドボックス、プラグイン、hooks、StatusLine など) を編集
- **Claude Code からインポート** — 既存の Claude Code 設定をワンクリックインポート
- **Claude Code へエクスポート** — AiDog のプロキシアドレスを Claude Code 設定に書き込み
- **グループ設定同期** — グループごとに独立した設定ファイルを自動生成し、切り替えを容易に

### 注意

- Claude Code は **Anthropic ネイティブプロトコル** を使用
- 対象プラットフォームが Anthropic プロトコルをサポートするか、プロトコル変換を設定
- AiDog は Claude Code 専用パス `/claude/v1/messages` を提供

## その他のアシスタント

カスタム API アドレスをサポートする任意のツールが AiDog をプロキシとして使用可能:

1. API ベース URL を `http://127.0.0.1:9876` に設定
2. API キーを任意の値に設定
3. AiDog に一致するプラットフォームとプロトコルが設定されていることを確認
