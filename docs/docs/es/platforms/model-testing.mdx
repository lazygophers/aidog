# Prueba de modelos

AiDog ofrece una función integrada de prueba de modelos para verificar la conectividad de la plataforma y la disponibilidad de modelos sin herramientas externas.

## Uso

En la página **«Plataformas»**, haz clic en «Probar» de una tarjeta de plataforma para abrir el panel de pruebas.

## Modos de prueba

| Modo | Descripción |
|------|-------------|
| **Prueba rápida** | Envía una petición corta con el modelo por defecto de la plataforma para una verificación rápida |
| **Modelo único** | Prueba un modelo específico |
| **Prueba por lotes** | Prueba varios modelos secuencialmente |
| **Prueba aleatoria** | Prueba todos los modelos disponibles en la plataforma uno a uno |
| **Prueba personalizada** | Elige modelo + prompt personalizado y máximo de tokens |

## Resultados

Cada resultado muestra:

- **Nombre del modelo** — El modelo probado
- **Estado** — Éxito o fracaso
- **Tiempo de respuesta** — Tiempo total desde el envío hasta la recepción completa
- **Uso de tokens** — Conteos de tokens de entrada/salida
- **Contenido de respuesta** — Texto devuelto (plegable)

## Casos de uso

- **Tras añadir plataforma** — Verifica API Key y Base URL correctos
- **Tras cambiar endpoints** — Confirma que el nuevo endpoint funciona
- **Tras mapeo de modelo** — Asegura que el modelo destino está disponible
- **Resolución de problemas** — Si fallan las peticiones con proxy, prueba primero la plataforma

## Notas

- La prueba usa la API Key y endpoint configurados, consumiendo tokens reales
- Las pruebas de plataforma mock no envían peticiones reales
- Los modos por lotes y aleatorio prueban varios modelos y tardan más
