# Mapeo de modelos

El mapeo de modelos permite traducir nombres de modelos de las solicitudes del cliente a los nombres reales soportados por la plataforma.

## Casos de uso

- El cliente solicita `gpt-4o`, pero quieres enrutar a `deepseek-chat` de DeepSeek
- Estandarizar la nomenclatura de modelos entre plataformas
- Tests A/B de modelos equivalentes en diferentes plataformas

## Configuración

Agrega reglas de mapeo en la sección "Mapeo de modelos" de la plataforma:

| Modelo solicitado | Mapeado a |
|---------------|---------|
| `gpt-4o` | `deepseek-chat` |
| `claude-3-opus` | `claude-3-opus-20240229` |

### Reglas de mapeo

1. **Coincidencia exacta** — Solo mapea cuando el nombre coincide exactamente
2. **Preservar original** — Los nombres sin coincidencia pasan sin cambios
3. **Prioridad** — Mapeo a nivel de plataforma primero, luego a nivel de grupo

## Uso en grupos

El mapeo de modelos se puede configurar en dos niveles:

- **Nivel de plataforma** — Se aplica a todas las solicitudes de esa plataforma
- **Nivel de grupo** — Solo se aplica dentro de las reglas de enrutamiento del grupo

El mapeo a nivel de grupo tiene prioridad sobre el de plataforma. Ver [Reglas de enrutamiento](/groups/routing-rules).
