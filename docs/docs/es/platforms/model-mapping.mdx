# Mapeo de modelos

El mapeo de modelos permite traducir nombres de modelo de las peticiones del cliente a los nombres realmente soportados por la plataforma.

## Casos de uso

- El cliente pide `gpt-4o`, pero quieres enrutar a `deepseek-chat` de DeepSeek
- Estandarizar la nomenclatura de modelos entre plataformas
- Simplificar la selección con alias

## Configuración

Añade reglas en la sección «Mapeo de modelos» de la plataforma o grupo:

| Modelo solicitado | Mappea a |
|---------------|---------|
| `gpt-4o` | `deepseek-chat` |
| `claude-3-opus` | `claude-3-opus-20240229` |

### Reglas

- **Coincidencia exacta** — Solo mappea cuando el nombre coincide exactamente
- **Preservar original** — Los nombres no coincidentes pasan sin cambios
- **Prioridad de grupo** — Los mapeos a nivel de grupo tienen prioridad sobre los de plataforma

## Mapeo de dos niveles

Configurable en dos niveles:

- **Nivel de plataforma** — Se aplica a todas las peticiones de esa plataforma
- **Nivel de grupo** — Solo se aplica dentro de las reglas de enrutamiento del grupo

Ver [Reglas de enrutamiento](/es/groups/routing-rules).

## Ejemplo de alias

Crea alias cortos; los clientes solo necesitan usar el alias:

| Alias | Objetivo | Propósito |
|-------|--------|---------|
| `fast` | `gpt-4o-mini` | Respuestas rápidas |
| `smart` | `claude-3-opus` | Alta calidad |
| `code` | `deepseek-coder` | Generación de código |
