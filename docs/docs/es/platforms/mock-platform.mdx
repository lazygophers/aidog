# Plataforma Mock

Las plataformas mock no envían peticiones IA reales — devuelven respuestas preconfiguradas. Ideales para desarrollo, depuración, prueba de reglas de enrutamiento y verificación de integraciones de cliente.

## Crear una plataforma mock

1. Ve a la página **«Plataformas»**
2. Haz clic en **«+»**
3. Selecciona **«Mock»** en la lista de tipos de plataforma
4. No se necesita API Key ni Base URL
5. Guardar

Tras guardar, AiDog crea automáticamente un grupo correspondiente. Apunta tu cliente a la ruta de ese grupo para usar el mock.

## Configuración de respuesta

Personaliza los parámetros:

| Parámetro | Descripción | Por defecto |
|-----------|-------------|---------|
| Texto de respuesta | Texto simulado | `Hello from mock` |
| Código de estado | Código HTTP | `200` |
| Retardo de respuesta | Retardo simulado en ms | `0` |
| Tokens de entrada | Tokens de entrada simulados | `10` |
| Tokens de salida | Tokens de salida simulados | `20` |
| Tokens de caché | Tokens de caché simulados | `0` |
| Razón de fin | Razón de fin simulada (p. ej. stop) | `stop` |
| Streaming | Devolver como stream (seguir petición o forzar) | Seguir petición |

## Simulación de errores

Soporta varios escenarios:

| Modo de error | Descripción |
|------------|-------------|
| Sin error | Devuelve respuesta normal |
| Error HTTP | Devuelve un código de error HTTP |
| Rate Limit (429) | Simula limitación de tasa |
| Timeout | Simula timeout de petición |

Útil para probar el manejo de errores del cliente o el comportamiento de failover de AiDog.

## Casos de uso

- **Desarrollo** — Probar integración del cliente sin API keys reales
- **Pruebas de enrutamiento** — Verificar reglas de coincidencia de grupo y mapeo
- **Prueba de failover** — Simular errores para observar conmutación automática
- **Prueba de rendimiento** — Configurar retardos para probar bajo respuestas lentas
