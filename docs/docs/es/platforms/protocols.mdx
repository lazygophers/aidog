# Tipos de protocolos

AiDog soporta múltiples protocolos de API de IA para detección de solicitudes entrantes y conversión de solicitudes salientes.

## Protocolos soportados

### Serie OpenAI

| Protocolo | Descripción | Endpoint |
|----------|-------------|----------|
| `openai_chat` | Chat Completions API | `/v1/chat/completions` |
| `openai_completions` | Text Completions API (heredado) | `/v1/completions` |
| `openai_responses` | Responses API (Codex) | `/v1/responses` |

### Anthropic

| Protocolo | Descripción | Endpoint |
|----------|-------------|----------|
| `anthropic` | Claude Messages API | `/v1/messages` |

### Google

| Protocolo | Descripción | Endpoint |
|----------|-------------|----------|
| `gemini` | Gemini Generate Content API | `/v1beta/models/{model}:generateContent` |

### Otros

| Protocolo | Descripción |
|----------|-------------|
| `minimax` | Protocolo nativo de MiniMax |
| `glm` | Protocolo nativo de Zhipu GLM |

## Detección automática

AiDog detecta automáticamente el protocolo entrante basándose en la ruta de la solicitud:

- `/v1/chat/completions` → `openai_chat`
- `/v1/completions` → `openai_completions`
- `/v1/responses` → `openai_responses`

Puedes servir diferentes tipos de clientes a través de la misma dirección de proxy.

## Conversión de protocolo

Los protocolos entrantes y salientes pueden ser diferentes. AiDog gestiona la conversión automáticamente:

1. Analiza la solicitud entrante (protocolo detectado)
2. Convierte al formato interno
3. Transforma al protocolo saliente de la plataforma destino
4. Envía la solicitud

Ejemplo: El cliente envía `openai_chat` → AiDog convierte a protocolo `anthropic` → envía a Claude.

## Preajustes de plataforma

AiDog incluye más de 50 preajustes de plataforma integrados, cada uno conteniendo:
- Base URL predeterminada
- Protocolo predeterminado
- Lista de modelos soportados

Seleccionar un preajuste completa automáticamente estos valores.
