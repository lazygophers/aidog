# Añadir Plataforma

Las plataformas son el concepto central en AiDog — cada una representa un endpoint de servicio de AI API.

![AiDog — Añadir Plataforma](/screenshots/add-platform.png)

## Pasos para añadir

1. Ve a la página **«Plataformas»**
2. Pulsa el botón **«+»** arriba a la derecha
3. Selecciona una plataforma de la lista de presets (o rellena manualmente)
4. Introduce tu API Key y pulsa Guardar

:::tip{title="Grupo automático"}
Tras guardar una plataforma, AiDog crea automáticamente un grupo para ella, por lo que puedes empezar a usarla de inmediato sin configuración manual.
:::

## Configuración básica

| Campo | Descripción | Ejemplo |
|-------|-------------|---------|
| Nombre | Nombre a mostrar | `Mi OpenAI` |
| URL Base | URL base de la API (con prefijo de versión) | `https://api.openai.com/v1` |
| API Key | Clave API de la plataforma | `sk-...` |

### Presets de plataforma

AiDog incluye 50+ presets de plataforma integrados (OpenAI, Anthropic, Gemini, DeepSeek, Qwen y más). Seleccionar un preset autorellena todos los campos de configuración — solo introduce tu API Key.

Algunos presets están marcados con «Coding Plan» — configuran automáticamente consultas de cuota.

## Modo passthrough

Algunos tipos de plataforma soportan el modo «passthrough»: AiDog reenvía las cabeceras y cuerpo de la petición del cliente tal cual, sin conversión de protocolo ni inyección de autenticación.

- **URL upstream** — Introduce la URL raíz de la plataforma (p. ej. `https://api.anthropic.com`), sin prefijo de versión
- **API Key** — Opcional en modo passthrough; el cliente puede llevar su propia autenticación

Ideal para escenarios donde la autenticación del lado del cliente (p. ej. suscripciones OAuth) debe preservarse.

## Múltiples endpoints

Cada plataforma puede tener múltiples endpoints, cada uno con su propia dirección y tipo de interfaz. AiDog selecciona automáticamente el endpoint coincidente según la petición del cliente.

Por ejemplo, la misma plataforma puede soportar tanto la interfaz de chat como la interfaz de asistente de codificación.

## Configuración de modelos

### Modelos por defecto

En los ajustes de la plataforma, especifica qué modelo usar por defecto para diferentes tipos de peticiones. AiDog empareja automáticamente el modelo más adecuado según el nombre de modelo solicitado.

### Obtener lista de modelos

Pulsa el botón «Obtener modelos» para recuperar automáticamente los modelos disponibles de la plataforma.

### Mapeo de modelos

Mapea los nombres de modelos solicitados a los nombres reales soportados por la plataforma. Consulta [Mapeo de modelos](/es/platforms/model-mapping).

## Configuración de consulta de saldo

Para plataformas que usan el sistema NewAPI, configura consultas de saldo separadas:

- **URL de consulta** — Separada de la URL de API
- **Token de consulta** — Obtenido del panel (diferente de la API Key)
- **User ID** — Opcional

Consulta [Saldo y Cuotas](/es/stats/balance).

## Presupuesto manual

Para plataformas sin consulta automática de saldo, establece límites de presupuesto manual:

- **Total** — Límite global de gasto
- **Ventana móvil** — Límite en una ventana temporal móvil
- **Ventana fija** — Límite que se reinicia según un calendario fijo
- **Diario** — Límite de gasto diario

Cuando se agota, AiDog deja de reenviar a esa plataforma y hace failover automáticamente. Consulta [Saldo y Cuotas](/es/stats/balance).

## Prueba de modelo

Tras guardar una plataforma, prueba su conectividad:

1. Pulsa el botón «Probar» en la tarjeta de la plataforma
2. Elige un modo de prueba: Rápido / Lote / Personalizado
3. Visualiza los resultados: tiempo de respuesta, número de tokens, éxito o fallo

Consulta [Prueba de modelos](/es/platforms/model-testing).

## Lista de plataformas

La página Plataformas muestra todas las tarjetas de plataforma añadidas, cada una mostrando:

- Nombre y logotipo de la plataforma
- Indicador de estado de salud (verde/amarillo/rojo)
- Saldo o utilización de cuota
- Número de peticiones de hoy y tasa de éxito

Soporta búsqueda por pinyin para localizar rápidamente una plataforma.

## Otras operaciones

- **Habilitar/Deshabilitar** — Pausar una plataforma sin eliminarla
- **Editar** — Pulsa una tarjeta de plataforma para ver detalles y modificar ajustes
- **Eliminar** — Pulsa el botón eliminar. Los logs y estadísticas relacionados se conservan
- **Arrastrar para reordenar** — Arrastra las tarjetas en la lista para ajustar el orden de visualización

## FAQ

### ¿Cómo rellenar la URL Base?

La URL Base es la ruta base completa incluyendo el prefijo de versión:

| Plataforma | URL Base |
|----------|----------|
| OpenAI | `https://api.openai.com/v1` |
| Anthropic | `https://api.anthropic.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| Qwen | `https://dashscope.aliyuncs.com/compatible-mode/v1` |

:::caution{title="Nota"}
No añadas `/chat/completions` a la URL Base. AiDog lo maneja automáticamente.
:::

### Seguridad de la API Key

Las claves API se almacenan localmente y nunca se suben a ningún servidor.
