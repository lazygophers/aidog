# Codex

Configura la integración de AiDog con OpenAI Codex CLI en la pestaña **«Ajustes» → «Codex»**.

## Edición de configuración

Edita los ajustes de Codex en `~/.codex/config.toml` directamente en AiDog:

- **Provider** — Proveedor de API y URL base
- **Modelo** — Nombre del modelo a usar
- **Otras opciones** — Todos los elementos de configuración soportados por Codex

Los cambios se escriben inmediatamente en el archivo de configuración — sin necesidad de localizar y editar archivos manualmente.

## Configuración rápida

Una configuración mínima solo requiere la dirección proxy y cualquier API key:

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

## Configuración de grupo

Codex soporta el cambio entre grupos mediante perfiles:

1. Pulsa el botón «Codex» en una tarjeta de grupo para copiar el comando de inicio
2. Pega en tu terminal:

```bash
AIDOG_KEY='<group-name>' codex -p '<group-name>' \
  --dangerously-bypass-approvals-and-sandbox -a never
```

AiDog enruta las peticiones al grupo correspondiente según la variable de entorno `AIDOG_KEY`.

## Notas

- Codex usa la API Responses (`/v1/responses`), que AiDog detecta y enruta automáticamente
- Asegúrate de que la plataforma correspondiente se haya añadido a AiDog
- Añade mapeo de modelos en ajustes de plataforma o grupo si es necesario
- El prefijo de ruta `/proxy/v1` en la dirección proxy no debe omitirse
