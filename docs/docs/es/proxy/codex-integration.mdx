# Integración con Codex

AiDog soporta nativamente OpenAI Codex CLI, Claude Code y otros asistentes de codificación de IA.

## OpenAI Codex

### Configuración

Establece lo siguiente en el archivo de configuración de Codex `~/.codex/config.toml`:

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:7860/v1"
api_key = "any-value"

[model]
name = "o3"
```

### Notas

- Codex usa la **Responses API** (`/v1/responses`). AiDog detecta y enruta automáticamente.
- Asegúrate de que la plataforma destino tenga el protocolo `openai_responses` activado.
- Agrega mapeos de modelos a nivel de plataforma o grupo según sea necesario.

## Claude Code

### Configuración

Establece las variables de entorno:

```bash
export ANTHROPIC_BASE_URL="http://127.0.0.1:7860"
export ANTHROPIC_API_KEY="any-value"
```

O configura la API base URL en los ajustes de Claude Code.

### Notas

- Claude Code usa el **protocolo nativo de Anthropic**
- Asegúrate de que la plataforma destino soporte el protocolo Anthropic o configura conversión de protocolo

## Otros asistentes

Cualquier herramienta que soporte direcciones API personalizadas puede usar AiDog como proxy:

1. Establece la API base URL a `http://127.0.0.1:7860`
2. Establece la clave API a cualquier valor
3. Asegúrate de que AiDog tenga una plataforma y protocolo correspondientes configurados
