# Integración Codex

AiDog soporta nativamente OpenAI Codex CLI, Claude Code y otros asistentes IA, con editores de configuración integrados.

## OpenAI Codex

### Configuración

Define lo siguiente en el archivo de config de Codex `~/.codex/config.toml`:

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

### Página de ajustes Codex

AiDog ofrece un editor integrado en **«Ajustes → Codex»**:

- Edita los ajustes de `~/.codex/config.toml` directamente
- Sin necesidad de localizar y editar archivos manualmente
- Los cambios surten efecto inmediatamente tras guardar

### Notas

- Codex usa la **Responses API** (`/v1/responses`). AiDog detecta y enruta automáticamente.
- Asegúrate de que la plataforma destino tiene un endpoint con protocolo `openai_responses`.
- Añade mapeos de modelo a nivel de plataforma o grupo si hace falta.

## Claude Code

### Export de configuración en un clic

En la pestaña **«Ajustes → Claude Code»** de AiDog, haz clic en «Exportar configuración». AiDog escribe la dirección del proxy en el archivo de config de Claude Code (`~/.claude.json`).

### Configuración manual

También puedes usar variables de entorno:

```bash
export ANTHROPIC_BASE_URL="http://127.0.0.1:9876"
export ANTHROPIC_API_KEY="any-value"
```

### Página de ajustes Claude Code

AiDog ofrece un editor dedicado en **«Ajustes → Claude Code»**:

- **Edición de configuración** — Edita ajustes de Claude Code (modelo, permisos, sandbox, plugins, hooks, StatusLine, etc.)
- **Importar de Claude Code** — Importación en un clic de la config existente
- **Exportar a Claude Code** — Escribe la dirección del proxy de AiDog en la config de Claude Code
- **Sincronizar ajustes de grupo** — Genera automáticamente archivos de configuración independientes por grupo

### Notas

- Claude Code usa el **protocolo nativo de Anthropic**
- Asegúrate de que la plataforma soporta el protocolo Anthropic o configura conversión
- AiDog proporciona una ruta dedicada `/claude/v1/messages` para Claude Code

## Otros asistentes

Cualquier herramienta que soporte direcciones de API personalizadas puede usar AiDog como proxy:

1. Establece la URL base de la API a `http://127.0.0.1:9876`
2. Establece la clave API a cualquier valor
3. Asegúrate de que AiDog tiene plataforma y protocolo coincidentes configurados
