# Enrutamiento avanzado

Domina las capacidades avanzadas de enrutamiento de AiDog para un control fino del tráfico.

## Agregación multiplataforma

Añade varias plataformas a un grupo con diferentes modos. Ver [Crear y gestionar](/es/groups/manage-groups).

## Failover

Al fallar una plataforma, AiDog prueba automáticamente la siguiente:

```
Petición → Plataforma A (429 Rate Limit)
        → Plataforma B (Éxito)
        → Devolver respuesta
```

Failover se activa por:

- HTTP 429 (Rate Limit)
- HTTP 5xx (Server Error)
- Timeout de conexión
- Errores de red

## Load Balancing

Distribuye peticiones por peso. Ejemplo:

| Plataforma | Peso | Tráfico esperado |
|----------|--------|------------------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## Alias de modelo

Crea alias vía mapeo de modelo:

| Alias | Mappea a | Propósito |
|-------|---------|---------|
| `fast` | `gpt-4o-mini` | Respuestas rápidas |
| `smart` | `claude-3-opus` | Alta calidad |
| `code` | `deepseek-coder` | Generación de código |

Los clientes usan alias; AiDog enruta a plataformas reales según la configuración del grupo.

## Enrutamiento encadenado

Combina mapeos de grupo y plataforma para transformación multicapa:

1. Cliente pide modelo `smart`
2. Grupo A mapea `smart` → `claude-3-opus`
3. Plataforma X mapea `claude-3-opus` → `claude-3-opus-20240229`
4. Enviado a Anthropic

## Timeout a nivel de grupo

Cada grupo puede definir timeout de petición y de conexión de forma independiente, anulando los globales. Útil cuando modelos más lentos necesitan más tiempo de espera.

Ver [Crear y gestionar](/es/groups/manage-groups).
