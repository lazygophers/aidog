# Ver logs

La página **Logs** registra cada petición de proxy que pasa por AiDog, facilitando diagnosticar problemas de enrutamiento, conversión de protocolo y respuestas upstream.

## Lista de logs

Cada entrada muestra:

| Campo | Descripción |
|------|------|
| Grupo | Nombre del grupo coincidente |
| Modelo | Modelo original solicitado por el cliente |
| Modelo real | Modelo enviado al upstream tras el mapeo |
| Protocolo | Protocolo entrante → protocolo upstream |
| Plataforma | Plataforma que procesó realmente la petición |
| Estado | Código de estado HTTP (éxito / fallo) |
| Duración | Duración total de la petición (milisegundos) |
| Tokens | Número de tokens de entrada / salida / caché |
| Hora | Momento de la petición |

La lista muestra 50 entradas por página con paginación.

## Filtrado

Usa los filtros superiores para localizar logs rápidamente:

- **Plataforma**: filtrar por plataforma específica
- **Grupo**: filtrar por grupo
- **Estado**: solo éxito / solo fallo
- **Rango de tiempo**: todo / última 1 hora / 6 horas / 24 horas / 7 días / 30 días
- **Modelo**: buscar por nombre de modelo original o real

## Detalles de la petición

Haz clic en cualquier entrada para expandir sus detalles y ver la cadena completa:

- **Petición de usuario**: URL, cabeceras y cuerpo originales
- **Petición upstream**: URL convertida, cabeceras y cuerpo enviados al upstream
- **Respuesta upstream**: código de estado, cabeceras y cuerpo devueltos por el upstream
- **Respuesta de usuario**: cabeceras y cuerpo finales devueltos al cliente

Los detalles permiten copiar con un clic, para pegar en un ticket de soporte o herramienta de depuración.

:::tip
El contenido de «Petición de usuario» y «Petición upstream» se controla mediante los ajustes de logs. Por defecto, el cuerpo no se registra. Consulta [Ajustes de logs](/es/logs/log-settings).
:::
