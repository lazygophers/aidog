# Reglas de enrutamiento

Las reglas de enrutamiento determinan cómo AiDog asocia las peticiones entrantes al grupo y plataforma adecuados.

## Flujo de coincidencia

```
Petición entrante → Detectar interfaz → Coincidir grupo → Seleccionar plataforma → Mapear modelo → Reenviar
```

## Coincidencia de grupo

Las peticiones se asignan a grupos por prioridad:

1. **Coincidencia de ruta** — La ruta de la petición coincide con el prefijo de ruta configurado del grupo
2. **Coincidencia de nombre de modelo** — El modelo está en la tabla de mapeo de un grupo
3. **Grupo por defecto** — Se usa sin coincidencia
4. **Primer grupo** — Se usa sin grupo por defecto

## Selección de plataforma

Dentro de un grupo, las plataformas se seleccionan por modo de enrutamiento:

### Modo Failover

Selecciona la primera plataforma disponible. Si falla, prueba la siguiente automáticamente. Si todas fallan, devuelve error.

### Modo Load Balance

Selecciona una plataforma aleatoriamente por peso. El failover también aplica al fallar.

## Failover

El failover automático se activa por:

- Limitación de tasa (429)
- Error de servidor (5xx)
- Timeout de conexión
- Error de red

## Cadena de mapeo de modelo

Orden de resolución del nombre de modelo:

1. **Mapeo a nivel de grupo** — Si el grupo define mapeos
2. **Mapeo a nivel de plataforma** — Si la plataforma define mapeos
3. **Valor original** — Usa el nombre de modelo original de la petición

```
Cliente pide: gpt-4o
  → Mapeo de grupo: gpt-4o → deepseek-chat
  → Mapeo de plataforma: (ninguno)
  → Enviado a plataforma: deepseek-chat
```
