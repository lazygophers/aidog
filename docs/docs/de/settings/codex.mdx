# Codex

Konfigurieren Sie die AiDog-Integration mit OpenAI Codex CLI im Tab **„Einstellungen" → „Codex"**.

## Konfigurationsbearbeitung

Codex-Einstellungen in `~/.codex/config.toml` direkt in AiDog bearbeiten:

- **Provider** — API-Anbieter und Base-URL
- **Modell** — Zu verwendender Modellname
- **Weitere Optionen** — Alle von Codex unterstützten Konfigurationseinträge

Änderungen werden sofort in die Config-Datei geschrieben — kein manuelles Suchen und Bearbeiten nötig.

## Schnellkonfiguration

Eine Minimalkonfiguration erfordert nur Proxy-Adresse und einen beliebigen API-Key:

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

## Gruppenkonfiguration

Codex unterstützt den Wechsel zwischen Gruppen über Profile:

1. Auf die „Codex"-Schaltfläche einer Gruppenkarte klicken, um den Startbefehl zu kopieren
2. Im Terminal einfügen:

```bash
AIDOG_KEY='<gruppen-name>' codex -p '<gruppen-name>' \
  --dangerously-bypass-approvals-and-sandbox -a never
```

AiDog leitet Anfragen basierend auf der Umgebungsvariablen `AIDOG_KEY` an die entsprechende Gruppe weiter.

## Hinweise

- Codex nutzt die Responses-API (`/v1/responses`), die AiDog automatisch erkennt und routet
- Stellen Sie sicher, dass die entsprechende Plattform in AiDog hinzugefügt wurde
- Falls nötig, Model-Mapping in Plattform- oder Gruppeneinstellungen hinzufügen
- Das Pfad-Präfix `/proxy/v1` in der Proxy-Adresse darf nicht weggelassen werden
