# Modellpreise

Alle Modellpreisdaten unter **„Einstellungen" → „Modellpreise"** verwalten. Preisdaten dienen der Kostenschätzung von Anfragen.

## Preissynchronisation

Ein-Klick-Sync der neuesten Preise aus Preisquellen:

- Auf „Jetzt synchronisieren" klicken, um neueste Preise abzurufen
- Sync-Ergebnis: hinzugefügt, aktualisiert, fehlgeschlagen
- „Auto-Sync" aktivieren, um in konfigurierten Intervallen abzurufen

Auto-Sync-Intervalle: 1 Stunde / 6 Stunden / 12 Stunden / 24 Stunden / 7 Tage

## Fallback-Preis

Für Modelle ohne Tabelleneintrag wird ein Fallback-Preis verwendet:

- **Eingabe-Fallback** — Standard $3.0 / Million Tokens
- **Ausgabe-Fallback** — Standard $3.0 / Million Tokens

Anpassbar im Sync-Abschnitt.

## Preistabelle

Die Tabelle zeigt diese Spalten:

| Spalte | Beschreibung |
|--------|-------------|
| Modell | Modellname |
| Quelle | LiteLLM (synchronisiert) / Manuell |
| Standardplattform | Standardplattform für dieses Modell |
| Eingabe $/M | Eingabe-Token-Preis (pro Million Tokens) |
| Ausgabe $/M | Ausgabe-Token-Preis |
| Cache-Lesen $/M | Cache-Token-Lesepreis |

Suche nach Modellname, Filter nach Quelle, anpassbare Seitengröße und Seitensprung.

## Preise verwalten

- Preiseintrag löschen (Lösch-Button am Zeilenende)
- Preisänderungen greifen sofort

## Kostenschätzung

Nach jeder Anfrage führt AiDog automatisch aus:

1. Token-Nutzung (Eingabe / Ausgabe / Cache) aus der Upstream-Antwort extrahieren
2. Einzelpreis in der Preistabelle suchen
3. Geschätzte Kosten berechnen

Preis-Fallback-Reihenfolge: Benutzerdefinierter Preis → Synchronisierter Preis → Fallback-Preis

:::tip{title="Hinweis"}
Geschätzte Kosten dienen nur als Referenz. Tatsätzliche Abrechnung basiert auf der Plattformabrechnung.
:::

Siehe [Preise & Abrechnung](/de/stats/pricing).
