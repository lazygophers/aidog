# Plattform hinzufügen

Plattformen sind das Kernkonzept in AiDog — jede repräsentiert einen AI-API-Service-Endpoint.

![AiDog — Plattform hinzufügen](/screenshots/add-platform.png)

## Schritte zum Hinzufügen

1. Zur **„Plattformen"**-Seite gehen
2. Auf die **„+"**-Schaltfläche oben rechts klicken
3. Plattform aus der Preset-Liste auswählen (oder manuell ausfüllen)
4. API-Key eingeben und Speichern klicken

:::tip{title="Auto-Gruppe"}
Nach dem Speichern einer Plattform erstellt AiDog automatisch eine Gruppe dafür, sodass sie sofort ohne manuelle Konfiguration genutzt werden kann.
:::

## Basiskonfiguration

| Feld | Beschreibung | Beispiel |
|-------|-------------|---------|
| Name | Anzeigename | `Mein OpenAI` |
| Base-URL | API-Basis-URL (mit Versionspräfix) | `https://api.openai.com/v1` |
| API-Key | Plattform-API-Schlüssel | `sk-...` |

### Plattform-Presets

AiDog enthält 50+ eingebaute Plattform-Presets (OpenAI, Anthropic, Gemini, DeepSeek, Qwen u. v. m.). Auswählen eines Presets füllt alle Konfigurationsfelder automatisch — nur API-Key eingeben.

Einige Presets sind mit „Coding Plan" markiert — sie konfigurieren Kontingentabfragen automatisch.

## Passthrough-Modus

Einige Plattformtypen unterstützen den „Passthrough"-Modus: AiDog leitet Client-Anfrage-Header und -Body unverändert weiter, ohne Protokollkonvertierung oder Authentifizierungs-Injection.

- **Upstream-URL** — Plattform-Root-URL eingeben (z. B. `https://api.anthropic.com`), ohne Versionspräfix
- **API-Key** — Im Passthrough-Modus optional; der Client kann seine eigene Authentifizierung mitbringen

Ideal für Szenarien, in denen clientseitige Authentifizierung (z. B. OAuth-Abos) erhalten bleiben muss.

## Mehrere Endpoints

Jede Plattform kann mehrere Endpoints haben, jeweils mit eigener Adresse und Schnittstellentyp. AiDog wählt automatisch den passenden Endpoint basierend auf der Client-Anfrage.

Beispielsweise kann dieselbe Plattform sowohl die Chat-Schnittstelle als auch die Coding-Assistant-Schnittstelle unterstützen.

## Modellkonfiguration

### Standardmodelle

In den Plattform-Einstellungen festlegen, welches Modell standardmäßig für verschiedene Anfragetypen verwendet wird. AiDog matcht automatisch das passendste Modell basierend auf dem angeforderten Modellnamen.

### Modellliste abrufen

Auf die Schaltfläche „Modelle abrufen" klicken, um verfügbare Modelle automatisch von der Plattform zu holen.

### Model-Mapping

Angeforderte Modellnamen auf die tatsächlichen, von der Plattform unterstützten Modellnamen mappen. Siehe [Model-Mapping](/de/platforms/model-mapping).

## Guthabenabfrage-Konfiguration

Für Plattformen mit dem NewAPI-System separate Guthabenabfragen konfigurieren:

- **Abfrage-URL** — Separat von der API-URL
- **Abfrage-Token** — Aus dem Dashboard (anders als API-Key)
- **User-ID** — Optional

Siehe [Guthaben & Kontingente](/de/stats/balance).

## Manuelles Budget

Für Plattformen ohne automatische Guthabenabfrage manuelle Budgetlimits setzen:

- **Gesamt** — Gesamte Ausgabengrenze
- **Rollierendes Fenster** — Ausgabengrenze im rollierenden Zeitfenster
- **Festes Fenster** — Ausgabengrenze mit festem Zurücksetzungszeitplan
- **Täglich** — Tägliche Ausgabengrenze

Wenn erschöpft, stoppt AiDog die Weiterleitung an diese Plattform und führt automatisch Failover durch. Siehe [Guthaben & Kontingente](/de/stats/balance).

## Modelltest

Nach dem Speichern einer Plattform die Konnektivität testen:

1. Auf die Schaltfläche „Testen" auf der Plattform-Karte klicken
2. Einen Testmodus wählen: Schnell / Batch / Benutzerdefiniert
3. Ergebnisse ansehen: Antwortzeit, Token-Anzahl, Erfolg oder Fehler

Siehe [Modelltest](/de/platforms/model-testing).

## Plattformliste

Die Plattformen-Seite zeigt alle hinzugefügten Plattform-Karten, jede zeigt:

- Plattformname und Logo
- Gesundheitsstatus-Indikator (grün/gelb/rot)
- Guthaben oder Kontingentnutzung
- Heutige Anfragezahl und Erfolgsquote

Unterstützt Pinyin-Suche, um eine Plattform schnell zu finden.

## Weitere Operationen

- **Aktivieren/Deaktivieren** — Plattform pausieren, ohne sie zu löschen
- **Bearbeiten** — Auf eine Plattform-Karte klicken, um Details zu sehen und Einstellungen zu ändern
- **Löschen** — Auf die Lösch-Schaltfläche klicken. Zugehörige Logs und Statistiken bleiben erhalten
- **Drag zum Sortieren** — Karten in der Liste ziehen, um die Anzeigereihenfolge anzupassen

## FAQ

### Wie wird die Base-URL ausgefüllt?

Die Base-URL ist der vollständige Basis-Pfad inklusive Versionspräfix:

| Plattform | Base-URL |
|----------|----------|
| OpenAI | `https://api.openai.com/v1` |
| Anthropic | `https://api.anthropic.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| Qwen | `https://dashscope.aliyuncs.com/compatible-mode/v1` |

:::caution{title="Hinweis"}
Kein `/chat/completions` an die Base-URL anhängen. AiDog übernimmt das automatisch.
:::

### API-Key-Sicherheit

API-Schlüssel werden lokal gespeichert und nie auf einen Server hochgeladen.
