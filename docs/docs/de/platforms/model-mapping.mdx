# Modellzuordnung

Die Modellzuordnung ermöglicht es, Modellnamen aus Client-Anfragen in die tatsächlichen, von der Plattform unterstützten Modellnamen zu übersetzen.

## Anwendungsfälle

- Client fragt `gpt-4o` an, aber Sie möchten zu DeepSeeks `deepseek-chat` routen
- Modellbenennung plattformübergreifend standardisieren
- A/B-Tests äquivalenter Modelle auf verschiedenen Plattformen

## Konfiguration

Fügen Sie Zuordnungsregeln im Abschnitt „Modellzuordnung" der Plattform hinzu:

| Angefragtes Modell | Zugeordnet zu |
|---------------|---------|
| `gpt-4o` | `deepseek-chat` |
| `claude-3-opus` | `claude-3-opus-20240229` |

### Zuordnungsregeln

1. **Exakte Übereinstimmung** — Ordnet nur bei exaktem Modellnamen zu
2. **Original beibehalten** — Nicht übereinstimmende Modellnamen bleiben unverändert
3. **Priorität** — Plattformebene zuerst, dann Gruppenebene

## Verwendung in Gruppen

Modellzuordnung kann auf zwei Ebenen konfiguriert werden:

- **Plattformebene** — Gilt für alle Anfragen dieser Plattform
- **Gruppenebene** — Gilt nur innerhalb der Routing-Regeln der Gruppe

Gruppenebene hat Vorrang vor Plattformebene. Siehe [Routing-Regeln](/groups/routing-rules).
