# Modell-Mapping

Das Modell-Mapping übersetzt Modellnamen aus Client-Anfragen in die von der Plattform tatsächlich unterstützten Namen.

## Anwendungsfälle

- Client fordert `gpt-4o`, soll aber zu DeepSeeks `deepseek-chat` geroutet werden
- Einheitliche Modellnamen über Plattformen hinweg
- Auswahl über Aliase vereinfachen

## Konfiguration

Fügen Sie Mapping-Regeln im Abschnitt „Modell-Mapping" der Plattform oder Gruppe hinzu:

| Angefordertes Modell | Mappt auf |
|---------------|---------|
| `gpt-4o` | `deepseek-chat` |
| `claude-3-opus` | `claude-3-opus-20240229` |

### Regeln

- **Exakte Übereinstimmung** — Mappt nur bei exakt passendem Namen
- **Original erhalten** — Nicht passende Namen bleiben unverändert
- **Gruppenpriorität** — Gruppen-Mappings haben Vorrang vor Plattform-Mappings

## Zweistufiges Mapping

Zwei Ebenen:

- **Plattformebene** — Gilt für alle Anfragen dieser Plattform
- **Gruppenebene** — Gilt nur innerhalb der Routing-Regeln der Gruppe

Siehe [Routing-Regeln](/de/groups/routing-rules).

## Alias-Beispiel

Erzeugen Sie kurze Aliase; Clients nutzen nur den Alias:

| Alias | Ziel | Zweck |
|-------|--------|---------|
| `fast` | `gpt-4o-mini` | Schnelle Antworten |
| `smart` | `claude-3-opus` | Hohe Qualität |
| `code` | `deepseek-coder` | Code-Erzeugung |
