# Modell-Test

AiDog bietet einen integrierten Modell-Test, um Plattform-Konnektivität und Modellverfügbarkeit ohne externe Tools zu prüfen.

## Verwendung

Klicken Sie auf der **„Plattformen"**-Seite auf «Test» einer Plattformkarte, um das Test-Panel zu öffnen.

## Testmodi

| Modus | Beschreibung |
|------|-------------|
| **Schnelltest** | Sendet eine kurze Anfrage mit dem Standardmodell der Plattform zur schnellen Konnektivitätsprüfung |
| **Einzelmodell** | Testet ein bestimmtes Modell |
| **Batch-Test** | Testet mehrere Modelle nacheinander |
| **Zufallstest** | Testet alle verfügbaren Modelle der Plattform eins nach dem anderen |
| **Custom-Test** | Modell wählen plus eigenem Prompt und Max-Token-Anzahl |

## Testergebnisse

Jedes Ergebnis zeigt:

- **Modellname** — Das getestete Modell
- **Status** — Erfolg oder Misserfolg
- **Antwortzeit** — Gesamtzeit von Senden bis vollständigem Empfang
- **Token-Verbrauch** — Eingabe-/Ausgabe-Token-Anzahlen
- **Antwortinhalt** — Vom Modell zurückgegebener Text (einklappbar)

## Anwendungsfälle

- **Nach Plattform-Hinzufügung** — API-Key und Base-URL korrekt?
- **Nach Endpoint-Änderung** — Neuer Endpoint funktioniert?
- **Nach Modell-Mapping** — Zielmodell verfügbar?
- **Fehlerbehebung** — Bei proxidierten Fehlern zuerst Plattform testen

## Hinweise

- Der Test nutzt API-Key und Endpoint der Plattform und verbraucht echte Tokens
- Mock-Plattform-Tests senden keine echten Anfragen
- Batch- und Zufallsmodi testen mehrere Modelle und dauern länger
