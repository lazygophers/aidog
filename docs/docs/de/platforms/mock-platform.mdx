# Mock-Plattform

Mock-Plattformen senden keine echten KI-Anfragen — sie geben vorkonfigurierte Antworten zurück. Ideal für Entwicklung, Debugging, Routing-Regel-Tests und Client-Integrationsprüfung.

## Mock-Plattform erstellen

1. Gehen Sie zur **„Plattformen"**-Seite
2. Klicken Sie auf **„+"**
3. Wählen Sie **„Mock"** aus dem Plattformtyp-Liste
4. Kein API-Key oder Base-URL nötig
5. Speichern

Nach dem Speichern erzeugt AiDog automatisch eine entsprechende Gruppe. Richten Sie den Client auf den Gruppenpfad, um den Mock zu nutzen.

## Antwortkonfiguration

Passen Sie die Antwortparameter an:

| Parameter | Beschreibung | Standard |
|-----------|-------------|---------|
| Antworttext | Simulierter Antworttext | `Hello from mock` |
| Statuscode | HTTP-Statuscode | `200` |
| Antwortverzögerung | Simulierte Verzögerung in ms | `0` |
| Eingabe-Tokens | Simulierte Eingabe-Token-Anzahl | `10` |
| Ausgabe-Tokens | Simulierte Ausgabe-Token-Anzahl | `20` |
| Cache-Tokens | Simulierte Cache-Tokens | `0` |
| Finish-Reason | Simulierter Finish-Grund (z. B. stop) | `stop` |
| Streaming | Als Stream zurückgeben (Anfrage folgen oder erzwingen) | Anfrage folgen |

## Fehler-Simulation

Unterstützt die Simulation verschiedener Fehlerszenarien:

| Fehlermodus | Beschreibung |
|------------|-------------|
| Kein Fehler | Normale Antwort |
| HTTP-Fehler | Festgelegter HTTP-Fehlerstatus |
| Rate Limit (429) | Simuliert Ratenlimitierung |
| Timeout | Simuliert Anfrage-Timeout |

Nützlich, um Client-Fehlerbehandlung oder AiDogs Failover zu testen.

## Anwendungsfälle

- **Entwicklung** — Client-Integration ohne echte API-Keys testen
- **Routing-Tests** — Gruppen-Matching und Mapping-Regeln verifizieren
- **Failover-Test** — Fehler simulieren, um automatisches Umschalten zu beobachten
- **Performance-Test** — Verzögerungen setzen, um Verhalten bei langsamen Antworten zu testen
