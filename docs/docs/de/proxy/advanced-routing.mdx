# Erweitertes Routing

Meistern Sie AiDogs erweiterte Routing-Fähigkeiten für feingranulare Verkehrskontrolle.

## Multi-Plattform-Aggregation

Mehrere Plattformen mit verschiedenen Routing-Modi einer Gruppe hinzufügen. Siehe [Erstellen & verwalten](/de/groups/manage-groups).

## Failover

Bei Plattformfehler versucht AiDog automatisch die nächste in der Gruppe:

```
Anfrage → Plattform A (429 Rate Limit)
        → Plattform B (Erfolg)
        → Antwort zurück
```

Failover wird ausgelöst durch:

- HTTP 429 (Rate Limit)
- HTTP 5xx (Server Error)
- Verbindungs-Timeout
- Netzwerkfehler

## Load Balancing

Verteilt Anfragen gewichtet über Plattformen. Beispiel:

| Plattform | Gewicht | Erwarteter Traffic |
|----------|--------|------------------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## Modell-Aliase

Aliase via Modell-Mapping:

| Alias | Mappt auf | Zweck |
|-------|---------|---------|
| `fast` | `gpt-4o-mini` | Schnelle Antworten |
| `smart` | `claude-3-opus` | Hohe Qualität |
| `code` | `deepseek-coder` | Code-Erzeugung |

Clients nutzen Aliase; AiDog routet zu echten Plattformen gemäß Gruppenkonfiguration.

## Verkettetes Routing

Kombinieren Sie Gruppen- und Plattform-Mappings für mehrschichtige Transformation:

1. Client fordert Modell `smart`
2. Gruppe A mappt `smart` → `claude-3-opus`
3. Plattform X mappt `claude-3-opus` → `claude-3-opus-20240229`
4. An Anthropic gesendet

## Timeout auf Gruppenebene

Jede Gruppe kann Anfrage- und Verbindungs-Timeout unabhängig setzen und globale Defaults überschreiben. Nützlich bei langsameren Modellen, die längere Wartezeiten brauchen.

Siehe [Erstellen & verwalten](/de/groups/manage-groups).
