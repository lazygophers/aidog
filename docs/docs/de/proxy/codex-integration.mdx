# Codex-Integration

AiDog unterstützt nativ OpenAI Codex CLI, Claude Code und andere KI-Coding-Assistenten mit integrierten Konfigurationseditoren.

## OpenAI Codex

### Konfiguration

In Codex' Konfigurationsdatei `~/.codex/config.toml` setzen:

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

### Codex-Einstellungsseite

AiDog bietet einen integrierten Codex-Konfigurationseditor unter **„Einstellungen → Codex"**:

- `~/.codex/config.toml` direkt bearbeiten
- Kein manuelles Suchen/Bearbeiten nötig
- Änderungen greifen sofort nach Speichern

### Hinweise

- Codex verwendet die **Responses API** (`/v1/responses`). AiDog erkennt und routet automatisch.
- Sicherstellen, dass die Zielplattform einen Endpoint mit `openai_responses`-Protokoll hat.
- Modell-Mappings auf Plattform- oder Gruppenebene bei Bedarf hinzufügen.

## Claude Code

### Ein-Klick-Konfigurationsexport

In AiDogs **„Einstellungen → Claude Code"**-Tab auf «Konfiguration exportieren» klicken. AiDog schreibt die Proxy-Adresse automatisch in Claude Codes Konfigurationsdatei (`~/.claude.json`).

### Manuelle Konfiguration

Alternativ Umgebungsvariablen setzen:

```bash
export ANTHROPIC_BASE_URL="http://127.0.0.1:9876"
export ANTHROPIC_API_KEY="any-value"
```

### Claude-Code-Einstellungsseite

AiDog bietet einen dedizierten Editor unter **„Einstellungen → Claude Code"**:

- **Konfigurationsbearbeitung** — Claude-Code-Einstellungen bearbeiten (Modell, Berechtigungen, Sandbox, Plugins, Hooks, StatusLine usw.)
- **Aus Claude Code importieren** — Ein-Klick-Import der bestehenden Claude-Code-Konfiguration
- **Nach Claude Code exportieren** — AiDogs Proxy-Adresse in Claude-Code-Config schreiben
- **Gruppeneinstellungen synchronisieren** — Automatisch unabhängige Konfigurationsdateien pro Gruppe erzeugen, einfacher Wechsel

### Hinweise

- Claude Code verwendet das **native Anthropic-Protokoll**
- Sicherstellen, dass die Zielplattform das Anthropic-Protokoll unterstützt oder Konvertierung konfigurieren
- AiDog bietet einen dedizierten Pfad `/claude/v1/messages` für Claude Code

## Andere Assistenten

Jedes Tool mit benutzerdefinierbaren API-Adressen kann AiDog als Proxy nutzen:

1. API-Basis-URL auf `http://127.0.0.1:9876` setzen
2. API-Key auf beliebigen Wert setzen
3. Sicherstellen, dass AiDog passende Plattform und Protokoll konfiguriert hat
