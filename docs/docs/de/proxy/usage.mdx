# Proxy-Nutzung

AiDog betreibt einen lokalen HTTP-Proxy-Server, der KI-API-Anfragen empfängt und weiterleitet.

## Proxy-Adresse

Standardadresse:

```
http://127.0.0.1:9876
```

Port in **„Einstellungen → Proxy"** änderbar.

## Auto-Start

Aktivieren Sie „Proxy automatisch starten". Wenn aktiv, startet der Proxy-Dienst beim AiDog-Launch automatisch.

## Grundlegende Nutzung

Richten Sie jeden KI-Client mit benutzerdefinierbarer API-Adresse auf AiDog:

| Client-Typ | Konfiguration |
|-------------|---------------|
| OpenAI SDK | `base_url` auf Proxy-Adresse setzen |
| curl | Proxy-Adresse direkt anfragen |
| Web-Apps | API-Endpoint auf Proxy setzen |
| Coding-Assistenten | API-Basis-URL setzen |

### Python (OpenAI SDK)

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:9876/proxy/v1",
    api_key="any-value"  # AiDog nutzt Ihren echten konfigurierten Key
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello!"}]
)
```

### Node.js

```javascript
import OpenAI from 'openai';

const client = new OpenAI({
  baseURL: 'http://127.0.0.1:9876/proxy/v1',
  apiKey: 'any-value',
});

const response = await client.chat.completions.create({
  model: 'gpt-4o',
  messages: [{ role: 'user', content: 'Hello!' }],
});
```

### curl

```bash
curl http://127.0.0.1:9876/proxy/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer any-value" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

## Streaming

AiDog unterstützt SSE-Streaming (Server-Sent Events) vollständig. Client-Streaming-Anfragen werden transparent an die Upstream-Plattform weitergeleitet.

## Timeout

Proxy-Anfragen haben Timeout-Schutz:

- Standard-Timeout: 300 Sekunden
- Global oder pro Gruppe konfigurierbar
- Gibt bei Ablauf 504 Gateway Timeout zurück
