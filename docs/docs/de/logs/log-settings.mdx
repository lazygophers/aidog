# Log-Einstellungen

Auf der **Einstellungen**-Seite steuern Sie Granularität und Aufbewahrung von Logs — eine Balance zwischen Observability und Datenschutz / Speicherplatz.

## Aufzeichnungsstufen

Logging hat drei Stufen von Schaltern, die schrittweise greifen:

| Stufe | Beschreibung | Standard |
|------|------|------|
| Hauptschalter | Wenn aus, werden gar keine Proxy-Logs aufgezeichnet | An |
| Original-Nutzeranfrage aufzeichnen | Zeichnet Original-Anfrage-Header und -Body des Clients auf | Aus |
| Upstream-Anfrage aufzeichnen | Zeichnet konvertierte Anfrage-Header und -Body für den Upstream auf | Aus |

:::warning
Der Anfrage-Body kann sensible Inhalte enthalten (Prompts, Keys usw.). Stellen Sie sicher, dass Ihre lokale Umgebung vertrauenswürdig ist, bevor Sie „Original-Nutzeranfrage aufzeichnen" oder „Upstream-Anfrage aufzeichnen" aktivieren.
:::

## Aufbewahrung

Logs werden nach drei unabhängigen Aufbewahrungsfristen bereinigt:

| Typ | Standard | Bereinigungsmethode |
|------|------|----------|
| Original-Nutzeranfrage | 7 Tage | Leert nur das Body-Feld, behält die Log-Zeile |
| Upstream-Anfrage | 7 Tage | Leert nur das Body-Feld, behält die Log-Zeile |
| Log-Einträge | 90 Tage | Löscht den gesamten Eintrag |

Anders gesagt: Anfrage-Bodies älter als 7 Tage werden geleert (Statistiken bleiben erhalten), Logs älter als 90 Tage werden ganz gelöscht.

## Manuelles Bereinigen

Unter **Einstellungen → Datenverwaltung** können Sie jederzeit manuell „Abgelaufene Logs bereinigen" auslösen, was die obigen Aufbewahrungsregeln sofort anwendet.

## Datenspeicherung

Alle Logs werden in der lokalen Datenbank `~/.aidog/aidog.db` gespeichert und nie auf einen Server hochgeladen.
