# Logs ansehen

Die **Logs**-Seite zeichnet jede Proxy-Anfrage auf, die AiDog durchläuft, und erleichtert die Diagnose von Routing-, Protokollkonvertierungs- und Upstream-Antwortproblemen.

## Log-Liste

Jeder Log-Eintrag zeigt:

| Feld | Beschreibung |
|------|------|
| Gruppe | Name der gematchten Gruppe |
| Modell | Ursprünglich vom Client angefordertes Modell |
| Tatsächliches Modell | Nach Model-Mapping an den Upstream gesendetes Modell |
| Protokoll | Eingehendes Protokoll → Upstream-Protokoll |
| Plattform | Plattform, die die Anfrage tatsächlich verarbeitet hat |
| Status | HTTP-Statuscode (Erfolg / Fehler) |
| Dauer | Gesamtanfragedauer (Millisekunden) |
| Token | Anzahl Eingabe- / Ausgabe- / Cache-Tokens |
| Zeit | Zeitpunkt der Anfrage |

Die Liste zeigt 50 Einträge pro Seite mit Paginierung.

## Filterung

Nutzen Sie die Filter oben, um Logs schnell zu finden:

- **Plattform**: nach spezifischer Plattform filtern
- **Gruppe**: nach Gruppe filtern
- **Status**: nur Erfolg / nur Fehler
- **Zeitraum**: alle / letzte 1 Stunde / 6 Stunden / 24 Stunden / 7 Tage / 30 Tage
- **Modell**: Suche nach Original- oder tatsächlichem Modellnamen

## Anfrage-Details

Klicken Sie einen Log-Eintrag, um Details aufzuklappen und die vollständige Anfragekette zu sehen:

- **Nutzeranfrage**: Original-Anfrage-URL, Header und Body
- **Upstream-Anfrage**: konvertierte URL, Header und Body an den Upstream
- **Upstream-Antwort**: Statuscode, Header und Body vom Upstream
- **Nutzerantwort**: finale Header und Body an den Client

Details unterstützen Ein-Klick-Kopieren zum Einfügen in Support-Tickets oder Debug-Tools.

:::tip
Der Inhalt von „Nutzeranfrage" und „Upstream-Anfrage" wird durch die Log-Einstellungen gesteuert. Standardmäßig wird der Body nicht aufgezeichnet. Siehe [Log-Einstellungen](/de/logs/log-settings).
:::
