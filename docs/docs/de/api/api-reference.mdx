# API-Referenz

AiDog stellt eine lokale HTTP-API zur Abfrage von Gruppeninformationen bereit. Alle Endpunkte sind **ausschließlich localhost** (`127.0.0.1`) und erfordern keinen externen Netzwerkzugriff.

## POST /api/group-info

Fragt geschätzte Gruppeninformationen ab (Guthaben, Coding-Plan-Auslastung, Anfragestatistiken usw.). Nur **Einzelplattform-Gruppen** liefern anwendbare Daten; Mehrplattform- oder plattformlose Gruppen geben `applicable: false` zurück.

### Authentifizierung

```
Authorization: Bearer <group_name>
```

Der `group_name` ist der in AiDog konfigurierte Gruppenname. Er entspricht dem Wert der von Claude Code verwendeten Umgebungsvariablen `ANTHROPIC_AUTH_TOKEN`.

### Anfrage

- **Methode**: POST
- **URL**: `http://127.0.0.1:<port>/api/group-info`
- **Header**: `Authorization: Bearer <group_name>`

### Antwort (200 OK)

#### Einzelplattform-Gruppe:

```json
{
  "applicable": true,
  "balance": 48.20,
  "spent": 1.23,
  "coding_plan": [
    {
      "name": "5h",
      "utilization": 23,
      "pace": "fast",
      "reset_at": null
    },
    {
      "name": "7d",
      "utilization": 41,
      "pace": "normal",
      "reset_at": null
    }
  ],
  "requests": 42,
  "success_rate": 95.2,
  "cache_rate": 13.3,
  "total_tokens": 123456,
  "currency": "",
  "balance_days_remaining": 5.2
}
```

#### Mehrplattform- / plattformlose Gruppe:

```json
{
  "applicable": false,
  "balance": 0.0,
  "spent": 0.0,
  "coding_plan": [],
  "requests": 0,
  "success_rate": 0.0,
  "cache_rate": 0.0,
  "total_tokens": 0,
  "currency": "",
  "balance_days_remaining": null
}
```

### Antwortfelder

| Feld | Typ | Beschreibung |
|-------|------|-------------|
| `applicable` | boolean | Ob es sich um eine Einzelplattform-Gruppe mit verfügbaren Daten handelt |
| `balance` | number | Geschätztes verbleibendes Guthaben |
| `spent` | number | Kumulierte geschätzte Ausgaben |
| `coding_plan` | array | Coding-Plan-Stufen (Claude usw.); leer bei guthabenbasierten Plattformen |
| `coding_plan[].name` | string | Stufenname (5h, 7d usw.) |
| `coding_plan[].utilization` | number | Auslastungsprozentsatz (0-100) |
| `coding_plan[].pace` | string | Verbrauchsrate: `fast`, `normal` oder `busy` |
| `coding_plan[].reset_at` | number\|null | Zurücksetzungs-Zeitstempel (Unix-Sekunden) |
| `requests` | number | Gesamtanzahl Anfragen |
| `success_rate` | number | Erfolgsquote (0-100) |
| `cache_rate` | number | Cache-Trefferrate (0-100) |
| `total_tokens` | number | Insgesamt verbrauchte Tokens |
| `currency` | string | Währungseinheit |
| `balance_days_remaining` | number\|null | Geschätzte verbleibende Guthabentage (null falls nicht berechenbar) |

### Fehlerantworten

| Status | Bedingung |
|--------|-----------|
| 401 | Fehlender oder ungültiger `Authorization`-Header |
| 200 | Gruppe nicht gefunden → `applicable: false` |

### Beispiel

```bash
curl -X POST \\
  -H "Authorization: Bearer my-group" \\
  http://127.0.0.1:9876/api/group-info
```
