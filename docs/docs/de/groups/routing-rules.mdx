# Routing-Regeln

Routing-Regeln bestimmen, wie AiDog eingehende Anfragen der passenden Gruppe und Plattform zuordnet.

## Matching-Fluss

```
Eingehende Anfrage → Schnittstelle erkennen → Gruppe matchen → Plattform wählen → Modell mappen → Weiterleiten
```

## Gruppen-Matching

Anfragen werden nach Priorität Gruppen zugeordnet:

1. **Pfad-Match** — Anfragepfad passt zum konfigurierten Pfad-Präfix der Gruppe
2. **Modellnamen-Match** — Modellname steht in der Mapping-Tabelle einer Gruppe
3. **Standardgruppe** — Ohne Match verwendet
4. **Erste Gruppe** — Ohne Standardgruppe verwendet

## Plattformauswahl

Innerhalb einer Gruppe erfolgt die Auswahl nach Routing-Modus:

### Failover-Modus

Wählt die erste verfügbare Plattform. Bei Fehler automatisch die nächste. Wenn alle fehlschlagen, Fehler zurück.

### Load-Balance-Modus

Wählt gewichtet zufällig eine Plattform. Failover gilt auch bei Fehler.

## Failover

Automatisches Failover wird ausgelöst durch:

- Ratenlimitierung (429)
- Serverfehler (5xx)
- Verbindungs-Timeout
- Netzwerkfehler

## Modell-Mapping-Kette

Auflösungsreihenfolge des Modellnamens:

1. **Gruppen-Mapping** — Falls die Gruppe Mappings definiert
2. **Plattform-Mapping** — Falls die Plattform Mappings definiert
3. **Originalwert** — Verwendet den Originalnamen der Anfrage

```
Client fordert: gpt-4o
  → Gruppen-Mapping: gpt-4o → deepseek-chat
  → Plattform-Mapping: (keins)
  → An Plattform gesendet: deepseek-chat
```
