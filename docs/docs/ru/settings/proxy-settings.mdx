# Системные настройки

Управление прокси-сервисом и конфигурацией приложения во вкладке **«Настройки» → «Системные настройки»**.

![AiDog — Системные настройки](/screenshots/settings.png)

## Прокси-сервис

Статус-бар сверху:

- Индикатор состояния (зелёный = работает)
- Поле порта (не редактируется во время работы)
- Кнопка старт/стоп

| Настройка | По умолчанию | Описание |
|---------|---------|-------------|
| Порт прокси | `9876` | Порт прослушивания HTTP-прокси |
| Автозапуск | Вкл | Автозапуск прокси при старте AiDog |

Изменения порта вступают в силу после перезапуска прокси.

## Тайм-ауты

| Настройка | По умолчанию | Описание |
|---------|---------|-------------|
| Тайм-аут запроса | `300s` | Макс. ожидание ответа upstream |
| Тайм-аут подключения | `10s` | Макс. ожидание установки соединения |

Системные значения по умолчанию; уровни группы и модели могут переопределять.

## Настройки журналирования

См. [Настройки журналирования](/ru/logs/log-settings).

| Настройка | По умолчанию | Описание |
|---------|---------|-------------|
| Главный переключатель | Вкл | Выкл для остановки журналирования |
| Журнал запросов пользователя | Выкл | Запись исходных заголовков и тела |
| Журнал upstream-запросов | Выкл | Запись преобразованных заголовков и тела |
| Хранение запросов пользователя | 7 дней | Очистка тела после срока, статистика сохраняется |
| Хранение upstream-запросов | 7 дней | Очистка тела после срока, статистика сохраняется |
| Хранение журнала | 90 дней | Удаление записей целиком |

## Журнал приложения

Консольные логи выводятся всегда; ниже — только файлы.

| Настройка | По умолчанию | Описание |
|---------|---------|-------------|
| Файловое журналирование | Вкл | Запись логов в файл |
| Уровень логов | `INFO` | TRACE / DEBUG / INFO / WARN / ERROR |
| Хранение | `3 часа` | Автоочистка после срока |
