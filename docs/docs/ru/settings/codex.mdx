# Codex

Интеграция AiDog с OpenAI Codex CLI во вкладке **«Настройки» → «Codex»**.

## Редактирование конфигурации

Редактируйте `~/.codex/config.toml` прямо в AiDog:

- **Provider** — API-провайдер и base URL
- **Model** — имя модели
- **Прочее** — все поддерживаемые Codex элементы конфигурации

Изменения пишутся в файл сразу.

## Быстрая конфигурация

Минимум — адрес прокси и любой API-ключ:

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

## Конфигурация группы

Codex поддерживает переключение групп через профили:

1. Нажмите «Codex» на карточке группы для копирования команды
2. Вставьте в терминал:

```bash
AIDOG_KEY='<group-name>' codex -p '<group-name>' \
  --dangerously-bypass-approvals-and-sandbox -a never
```

AiDog маршрутизирует запросы в соответствующую группу по переменной `AIDOG_KEY`.

## Примечания

- Codex использует Responses API (`/v1/responses`), AiDog определяет и маршрутизирует автоматически
- Убедитесь, что соответствующая платформа добавлена в AiDog
- Добавьте маппинг моделей при необходимости
- Префикс пути `/proxy/v1` в адресе прокси обязателен
