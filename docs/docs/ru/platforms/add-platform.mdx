# Добавление платформы

Платформы — ключевое понятие AiDog: каждая представляет эндпоинт AI API.

![AiDog — Добавление платформы](/screenshots/add-platform.png)

## Шаги добавления

1. Перейдите на страницу **«Платформы»**
2. Нажмите кнопку **«+»** в правом верхнем углу
3. Выберите платформу из списка пресетов (или заполните вручную)
4. Введите API Key и нажмите «Сохранить»

:::tip{title="Автогруппа"}
После сохранения платформы AiDog автоматически создаёт для неё группу, чтобы вы могли сразу начать использование без ручной настройки.
:::

## Базовая конфигурация

| Поле | Описание | Пример |
|-------|-------------|---------|
| Имя | Отображаемое имя | `My OpenAI` |
| Base URL | Базовый URL API (с префиксом версии) | `https://api.openai.com/v1` |
| API Key | Ключ API платформы | `sk-...` |

### Пресеты платформ

AiDog включает 50+ встроенных пресетов (OpenAI, Anthropic, Gemini, DeepSeek, Qwen и др.). Выбор пресета автоматически заполняет все поля — достаточно ввести API Key.

Некоторые пресеты помечены «Coding Plan» — они автоматически настраивают запросы квоты.

## Режим Passthrough

Некоторые типы платформ поддерживают режим «passthrough»: AiDog передаёт заголовки и тело клиентского запроса как есть, без преобразования протокола или инъекции аутентификации.

- **Upstream URL** — введите корневой URL платформы (например `https://api.anthropic.com`), без префикса версии
- **API Key** — опционален в режиме passthrough; клиент может передавать свою аутентификацию

Идеально для сценариев, где нужно сохранить клиентскую аутентификацию (например OAuth-подписки).

## Несколько эндпоинтов

Каждая платформа может иметь несколько эндпоинтов, каждый со своим адресом и типом интерфейса. AiDog автоматически выбирает подходящий эндпоинт по клиентскому запросу.

Например, одна платформа может поддерживать и чат-интерфейс, и интерфейс ассистента кода.

## Конфигурация моделей

### Модели по умолчанию

В настройках платформы укажите, какую модель использовать по умолчанию для разных типов запросов. AiDog автоматически подбирает наиболее подходящую модель по запрошенному имени.

### Получение списка моделей

Нажмите «Fetch Models», чтобы автоматически получить доступные модели платформы.

### Маппинг моделей

Сопоставьте запрошенные имена моделей с фактическими именами, поддерживаемыми платформой. См. [Маппинг моделей](/ru/platforms/model-mapping).

## Конфигурация запроса баланса

Для платформ системы NewAPI настройте отдельные запросы баланса:

- **URL запроса** — отдельно от URL API
- **Токен запроса** — получается из панели (отличается от API Key)
- **User ID** — опционально

См. [Баланс и квоты](/ru/stats/balance).

## Ручной бюджет

Для платформ без автоматического запроса баланса задайте лимиты ручного бюджета:

- **Total** — общий лимит расходов
- **Rolling Window** — лимит в скользящем окне
- **Fixed Window** — лимит, сбрасываемый по фиксированному расписанию
- **Daily** — дневной лимит

При исчерпании AiDog прекращает перенаправление на эту платформу и автоматически выполняет отработку отказа. См. [Баланс и квоты](/ru/stats/balance).

## Тестирование моделей

После сохранения платформы проверьте связность:

1. Нажмите «Test» на карточке платформы
2. Выберите режим: Quick / Batch / Custom
3. Просмотрите результаты: время ответа, количество токенов, успех/провал

См. [Тестирование моделей](/ru/platforms/model-testing).

## Список платформ

Страница «Платформы» показывает все добавленные карточки, каждая отображает:

- Имя и логотип платформы
- Индикатор здоровья (зелёный/жёлтый/красный)
- Использование баланса или квоты
- Количество запросов и успешность за сегодня

Поддерживает пиньинь-поиск для быстрого поиска платформы.

## Другие операции

- **Включить/Отключить** — приостановить платформу без удаления
- **Редактировать** — щёлкните карточку для просмотра деталей и изменения настроек
- **Удалить** — нажмите кнопку удаления. Связанные журналы и статистика сохраняются
- **Перетаскивание** — перетаскивайте карточки для изменения порядка

## FAQ

### Как заполнить Base URL?

Base URL — полный базовый путь, включая префикс версии:

| Платформа | Base URL |
|----------|----------|
| OpenAI | `https://api.openai.com/v1` |
| Anthropic | `https://api.anthropic.com/v1` |
| DeepSeek | `https://api.deepseek.com/v1` |
| Qwen | `https://dashscope.aliyuncs.com/compatible-mode/v1` |

:::caution{title="Важно"}
Не добавляйте `/chat/completions` к Base URL. AiDog обрабатывает это автоматически.
:::

### Безопасность API Key

API-ключи хранятся локально и никогда не загружаются на сервер.
