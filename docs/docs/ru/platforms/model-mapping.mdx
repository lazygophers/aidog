# Маппинг моделей

Маппинг моделей позволяет переводить имена моделей из клиентских запросов в фактические имена, поддерживаемые платформой.

## Сценарии использования

- Клиент запрашивает `gpt-4o`, но нужно направить на `deepseek-chat` от DeepSeek
- Стандартизировать именование моделей между платформами
- Упростить выбор модели псевдонимами

## Конфигурация

Добавьте правила маппинга в разделе «Model Mapping» платформы или группы:

| Запрашиваемая модель | Сопоставляется с |
|---------------|---------|
| `gpt-4o` | `deepseek-chat` |
| `claude-3-opus` | `claude-3-opus-20240229` |

### Правила

- **Точное совпадение** — маппинг только при точном совпадении имени
- **Сохранение оригинала** — не сопоставленные имена проходят без изменений
- **Приоритет группы** — маппинг уровня группы имеет приоритет над маппингом уровня платформы

## Двухуровневый маппинг

Маппинг моделей можно настроить на двух уровнях:

- **Уровень платформы** — применяется ко всем запросам на этой платформе
- **Уровень группы** — применяется только в правилах маршрутизации этой группы

См. [Правила маршрутизации](/ru/groups/routing-rules).

## Пример псевдонимов

Используйте маппинг для создания коротких псевдонимов. Клиенту достаточно использовать псевдоним:

| Псевдоним | Цель | Назначение |
|-------|--------|---------|
| `fast` | `gpt-4o-mini` | Быстрые ответы |
| `smart` | `claude-3-opus` | Высокое качество |
| `code` | `deepseek-coder` | Генерация кода |
