# Тестирование моделей

AiDog имеет встроенную функцию тестирования моделей для проверки связности платформы и доступности моделей без внешних инструментов.

## Как использовать

На странице **«Платформы»** нажмите «Test» на карточке платформы, чтобы открыть панель тестирования.

## Режимы тестирования

| Режим | Описание |
|------|-------------|
| **Quick Test** | Короткий запрос на модели по умолчанию для быстрой проверки связности |
| **Single Model** | Тест конкретной модели |
| **Batch Test** | Последовательный тест нескольких моделей |
| **Random Test** | Построчный тест всех доступных моделей платформы |
| **Custom Test** | Выбор модели и ввод своего промпта и лимита токенов |

## Результаты тестирования

Каждый результат показывает:

- **Имя модели** — модель, использованная для теста
- **Статус** — успех или провал
- **Время ответа** — общее время от отправки запроса до получения полного ответа
- **Использование токенов** — количество входных и выходных токенов
- **Содержимое ответа** — текст, возвращённый моделью (сворачиваемый)

## Сценарии

- **После добавления платформы** — проверить корректность API Key и Base URL
- **После смены эндпоинтов** — подтвердить работу нового эндпоинта
- **После маппинга моделей** — убедиться, что целевая модель доступна
- **Диагностика** — при сбоях проксированных запросов сначала протестируйте соответствующую платформу

## Примечания

- Тестирование использует настроенный API Key и эндпоинт платформы, расходуя реальные токены
- Тесты mock-платформ не отправляют реальных запросов
- Пакетный и случайный режимы проверяют несколько моделей последовательно и занимают больше времени
