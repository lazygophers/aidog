# Платформа-заглушка (Mock)

Mock-платформы не отправляют реальных AI-запросов — они возвращают заранее настроенные ответы. Идеально для разработки, отладки, тестирования правил маршрутизации и проверки клиентских интеграций.

## Создание mock-платформы

1. Перейдите на страницу **«Платформы»**
2. Нажмите кнопку **«+»**
3. Выберите **«Mock»** в списке типов платформ
4. API Key и Base URL не требуются
5. Нажмите «Сохранить»

После сохранения AiDog автоматически создаёт соответствующую группу. Укажите клиенту путь этой группы для использования mock.

## Конфигурация ответа

Mock-платформы позволяют настраивать следующие параметры ответа:

| Параметр | Описание | По умолчанию |
|-----------|-------------|---------|
| Текст ответа | Симулируемый текст ответа | `Hello from mock` |
| Код состояния | HTTP-код состояния | `200` |
| Задержка ответа | Симулируемая задержка в мс | `0` |
| Входные токены | Симулируемое число входных токенов | `10` |
| Выходные токены | Симулируемое число выходных токенов | `20` |
| Токены кэша | Симулируемые токены попадания в кэш | `0` |
| Причина завершения | Симулируемая причина (например stop) | `stop` |
| Streaming | Возвращать ли потоком (по запросу / принудительно вкл/выкл) | По запросу |

## Симуляция ошибок

Mock-платформы поддерживают симуляцию различных ошибок:

| Режим ошибки | Описание |
|------------|-------------|
| Без ошибки | Нормальный ответ |
| HTTP-ошибка | Указанный HTTP-код ошибки |
| Rate Limit (429) | Симуляция ограничения частоты |
| Timeout | Симуляция тайм-аута |

Используйте для тестирования обработки ошибок клиентом или поведения отработки отказа AiDog.

## Сценарии

- **Разработка** — тестирование клиентской интеграции без реальных API-ключей
- **Тесты маршрутизации** — проверка сопоставления групп и правил маппинга моделей
- **Тесты отработки отказа** — симуляция ошибок для наблюдения автоматического переключения платформ
- **Нагрузочное тестирование** — установка задержек для проверки поведения клиента при медленных ответах
