# Планирование и автоматический выключатель

AiDog предоставляет **умное планирование** и **платформенный автоматический выключатель** для нескольких платформ в группе, накладывая учет здоровья и автоматическую защиту поверх маршрутизации failover / load-balance.

## Умное планирование

Планировщик (`SchedulerState`) поддерживает метрики здоровья платформ в реальном времени:

| Метрика | Описание |
| --- | --- |
| EMA задержки | Экспоненциальная скользящая средняя задержки (α = 0.3, ~последние 6 выборок) |
| In-flight | Текущее число запросов в обработке (inflight) |
| Подряд неудач | Счётчик отказов автоматического выключателя |
| Состояние выключателя | Текущее состояние трёхстороннего автомата |

При выборе платформы метрики здоровья объединяются для сортировки — предпочтение низко-задержочным, с малым in-flight, здоровым платформам.

## Трёхсторонний автомат выключателя

Каждая платформа независимо поддерживает состояние выключателя (`BreakerState`):

```
Closed ──отказы достигают порога──▶ Open ──тайм-аут──▶ HalfOpen ──зонд успешен──▶ Closed
                                                         └─зонд неудачен──▶ Open
```

| Состояние | Поле | Поведение |
| --- | --- | --- |
| **Closed** | `fails` | Нормальное пропускание; записывает подряд идущие отказы |
| **Open** | `until_ms` | Выключатель разомкнут; отфильтровывается из кандидатов (`Admission = Reject`) |
| **HalfOpen** | `probes` | Полуоткрытый зонд; ограниченное пропускание (`Admission = Probe`) |

### Пороговая конфигурация (`BreakerThresholds`)

| Параметр | Описание |
| --- | --- |
| `failure_threshold` | Подряд идущих отказов для срабатывания выключателя |
| `open_secs` | Длительность разомкнутого состояния (сек) |
| `half_open_max` | Макс. число зондов в полуоткрытом состоянии |

## Допуск

При выборе платформ-кандидатов планировщик возвращает трёхсторонний результат допуска:

- **Allow** — нормальное пропускание
- **Reject** — выключатель платформы Open; удаляется из кандидатов
- **Probe** — полуоткрытое пропускание (ограниченный зонд)

## Развязка с auto_disabled

Выключатель (`BreakerState`) и авто-отключение платформы (`auto_disabled`, по ошибке аутентификации) фильтруются как **объединение**:

- Платформа с выключателем Open → временно удалена (восстановимо)
- Платформа auto_disabled → постоянно отключена (требует ручного/auth-восстановления)
- Любое срабатывание → не маршрутизируется

Два механизма развязаны: автоматический выключатель — временную защита с учётом здоровья; auto_disabled — постоянное состояние ошибки аутентификации.

## Привязка сессии (Sticky)

Планировщик имеет встроенную таблицу привязки сессий (`StickyTable`):

- Запросы с одним ключом сессии привязываются к одной платформе (сохраняя контекст)
- In-memory LRU + вытеснение по TTL
- TTL: 30 минут бездействия
- Лимит емкости: 4096 записей

## Точки конфигурации

- **Уровень платформы**: настройте пороги выключателя на странице редактирования платформы
- **Уровень группы**: выберите стратегию планирования на странице редактирования группы
- **Глобальные значения по умолчанию**: настройте пороги по умолчанию на странице настроек
