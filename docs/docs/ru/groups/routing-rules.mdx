# Правила маршрутизации

Правила маршрутизации определяют, как AiDog сопоставляет входящие запросы с нужной группой и платформой.

## Поток сопоставления

```
Входящий запрос → Определение интерфейса → Сопоставление группы → Выбор платформы → Маппинг модели → Перенаправление
```

## Сопоставление группы

Запросы сопоставляются с группами по приоритету:

1. **Совпадение пути** — путь запроса совпадает с настроенным префиксом пути группы
2. **Совпадение имени модели** — имя модели запроса есть в таблице маппинга группы
3. **Группа по умолчанию** — используется при отсутствии совпадения
4. **Первая группа** — используется, если нет группы по умолчанию

## Выбор платформы

Внутри группы платформы выбираются по режиму маршрутизации:

### Режим Failover

Выбирается первая доступная платформа. При сбое автоматически пробуется следующая. При отказе всех возвращается ошибка.

### Режим Load Balance

Случайный выбор платформы по весу. При сбое также применяется отработка отказа.

## Отработка отказа

Автоматическая отработка отказа срабатывает при:
- Ограничении частоты (429)
- Ошибке сервера (5xx)
- Тайм-ауте подключения
- Сетевой ошибке

## Цепочка маппинга моделей

Порядок разрешения имени модели:

1. **Маппинг уровня группы** — если группа определяет маппинг моделей
2. **Маппинг уровня платформы** — если платформа определяет маппинг
3. **Оригинальное значение** — использование оригинального имени модели из запроса

```
Клиент запрашивает: gpt-4o
  → Маппинг группы: gpt-4o → deepseek-chat
  → Маппинг платформы: (нет)
  → Отправлено платформе: deepseek-chat
```
