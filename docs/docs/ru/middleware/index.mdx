# Движок правил промежуточного ПО

AiDog поставляется с настраиваемым движком правил промежуточного ПО для запросов/ответов, выполняющим правила на двух этапах цепочки проксирования — **входящем** (вход запроса) и **исходящем** (возврат ответа): нормализация, переопределение, маскирование, инъекция, фильтрация чувствительных слов, обнаружение ошибок и автоматический выключатель. Все правила настраиваются через UI — без кода.

## Архитектура

| Возможность | Описание |
| --- | --- |
| Хранение в одной таблице | Все правила в таблице SQLite `middleware_rule` |
| Синглон движка | `MiddlewareEngine` компилирует правила в regex при запуске, сопоставляет по приоритету на запрос |
| Двунаправленное монтирование | Входящее (перезапись тела запроса) + исходящее (перезапись тела ответа / SSE-потока) |
| Трёхуровневое переопределение | `global` → `group` → `platform`; более поздние области переопределяют более ранние одноимённые правила |
| Потоковая обработка по чанкам | SSE-ответы обрабатываются по чанкам без буферизации |
| Встроенные пресеты | Маскирование API-ключа / email / телефона + правила классификации ошибок по умолчанию |

## Типы правил

Движок определяет 8 `RuleType`, сгруппированных по этапу выполнения:

| Тип | Этап | Назначение |
| --- | --- | --- |
| `request_filter` | Входящий | Фильтрация полей запроса (например allow/deny список моделей) |
| `sensitive_word` | Входящий | Блокировка чувствительных слов (паттерн — само слово) |
| `redaction` | Входящий | Маскирование значений полей |
| `content_filter` | Входящий | Фильтрация содержимого |
| `dynamic_injection` | Входящий | Динамическая инъекция (system / header / body) |
| `response_override` | Исходящий | Переопределение тела ответа |
| `rectifier` | Исходящий | Выпрямитель (SSE / JSON / кодировка / фикс отсутствующих полей) |
| `error_rule` | Исходящий | Классификация ошибок (повтор / выключатель / переопределение кода состояния) |

## Поля правил

Поля каждого правила (таблица `middleware_rule`):

| Поле | Описание |
| --- | --- |
| `name` / `description` | Имя и описание |
| `rule_type` | Тип правила |
| `scope` | Область: `global` / `group` / `platform` |
| `scope_ref` | Ссылка на область (имя группы/платформы; пусто для global) |
| `match_type` | Метод сопоставления (`contains` / `regex`) |
| `pattern` | Паттерн сопоставления |
| `action` | Действие при совпадении (`warn` / `block` / `mask` / `inject`) |
| `config` | JSON-конфиг расширения |
| `priority` | Приоритет (выше — раньше) |
| `enabled` | Переключатель включения |
| `is_builtin` | Встроенный пресет (нельзя удалить, можно отключить) |
