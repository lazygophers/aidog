# Конфигурация и область

## Трёхуровневое переопределение

Правила промежуточного ПО наслаиваются по области (`scope`); при выполнении **более поздние области переопределяют более ранние**:

```
global (глобальные по умолчанию) → group (уровень группы) → platform (уровень платформы)
```

| scope | scope_ref | Применяется к |
| --- | --- | --- |
| `global` | пусто | Все запросы |
| `group` | имя группы | Запросы этой группы |
| `platform` | имя платформы | Запросы, перенаправляемые этой платформе |

В пределах одного `rule_type` более конкретная область переопределяет более общую. Например: глобально запретить модель, но разрешить на конкретной платформе → правило платформы переопределяет глобальное.

## UI

Точки конфигурации промежуточного ПО:

1. **Глобальные по умолчанию**: страница настроек → вкладка Middleware (`AppSettings`)
2. **Уровень группы**: встроена в страницу редактирования группы
3. **Уровень платформы**: встроена в страницу редактирования платформы

Каждая позволяет просматривать / добавлять / редактировать / отключать / удалять правила; область автоматически привязывается к текущему контексту.

## Редактор правил

Каждое правило настраивает:

- **Тип** (`rule_type`): один из 8
- **Сопоставление** (`match_type` + `pattern`): contains / regex и т.д.
- **Действие** (`action`): warn / block / mask / inject и т.д.
- **Конфиг расширения** (`config`): JSON, зависит от типа (`replacement`, точка инъекции, стратегия классификации ошибок)
- **Приоритет** (`priority`): выше — раньше
- **Включено** (`enabled`)

## Производительность и безопасность

- regex правил компилируются при запуске и кэшируются; нулевая стоимость компиляции на запрос
- Лимит размера DFA ≤ 1 МиБ (`REGEX_DFA_SIZE_LIMIT`), предотвращение ReDoS
- Лимит размера исходника regex ≤ 1 МиБ (`REGEX_SIZE_LIMIT`)
