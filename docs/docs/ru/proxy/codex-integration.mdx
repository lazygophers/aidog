# Интеграция Codex

AiDog нативно поддерживает OpenAI Codex CLI, Claude Code и другие AI-ассистенты кода со встроенными редакторами конфигурации.

## OpenAI Codex

### Конфигурация

Укажите в файле конфигурации Codex `~/.codex/config.toml`:

```toml
[provider]
name = "openai"
base_url = "http://127.0.0.1:9876/proxy/v1"
api_key = "any-value"

[model]
name = "o3"
```

### Страница настроек Codex

AiDog предоставляет встроенный редактор конфигурации Codex в **«Настройки» → «Codex»**:

- Редактируйте настройки `~/.codex/config.toml` напрямую
- Не нужно вручную искать и править файлы конфигурации
- Изменения вступают в силу сразу после сохранения

### Примечания

- Codex использует **Responses API** (`/v1/responses`). AiDog определяет и маршрутизирует автоматически.
- Убедитесь, что целевая платформа имеет эндпоинт с протоколом `openai_responses`.
- Добавьте маппинг моделей на уровне платформы или группы при необходимости.

## Claude Code

### Экспорт конфигурации в один клик

На вкладке **«Настройки» → «Claude Code»** нажмите «Экспортировать конфигурацию». AiDog автоматически записывает адрес прокси в файл конфигурации Claude Code (`~/.claude.json`).

### Ручная конфигурация

Можно также задать переменные окружения:

```bash
export ANTHROPIC_BASE_URL="http://127.0.0.1:9876"
export ANTHROPIC_API_KEY="any-value"
```

### Страница настроек Claude Code

AiDog предоставляет специальный редактор настроек в **«Настройки» → «Claude Code»**:

- **Редактирование конфигурации** — настройки Claude Code (модель, разрешения, sandbox, плагины, hooks, StatusLine и др.)
- **Импорт из Claude Code** — импорт существующей конфигурации в один клик
- **Экспорт в Claude Code** — запись адреса прокси AiDog в конфигурацию Claude Code
- **Синхронизация настроек группы** — автоматическая генерация независимых файлов конфигурации для каждой группы

### Примечания

- Claude Code использует **нативный протокол Anthropic**
- Убедитесь, что целевая платформа поддерживает протокол Anthropic или настройте преобразование протокола
- AiDog предоставляет специальный путь `/claude/v1/messages` для Claude Code

## Другие ассистенты

Любой инструмент, поддерживающий пользовательские API-адреса, может использовать AiDog как прокси:

1. Установите базовый URL API на `http://127.0.0.1:9876`
2. Установите API-ключ на любое значение
3. Убедитесь, что в AiDog настроена соответствующая платформа и протокол
