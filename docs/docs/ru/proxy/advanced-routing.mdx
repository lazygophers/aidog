# Расширенная маршрутизация

Освойте расширенные возможности маршрутизации AiDog для тонкого управления трафиком.

## Мультиплатформенная агрегация

Добавьте несколько платформ в группу с разными режимами маршрутизации. См. [Создание и управление](/ru/groups/manage-groups).

## Отработка отказа

При сбое запроса к платформе AiDog автоматически пробует следующую платформу в группе:

```
Запрос → Платформа A (429 Rate Limit)
        → Платформа B (Успех)
        → Возврат ответа
```

Отработка отказа срабатывает при:
- HTTP 429 (Rate Limit)
- HTTP 5xx (Ошибка сервера)
- Тайм-ауте подключения
- Сетевых ошибках

## Балансировка нагрузки

Распределение запросов по нескольким платформам по весу. Пример:

| Платформа | Вес | Ожидаемый трафик |
|----------|--------|------------------|
| OpenAI | 70 | ~70% |
| DeepSeek | 30 | ~30% |

## Псевдонимы моделей

Создавайте псевдонимы через маппинг моделей:

| Псевдоним | Сопоставляется с | Назначение |
|-------|---------|---------|
| `fast` | `gpt-4o-mini` | Быстрые ответы |
| `smart` | `claude-3-opus` | Высокое качество |
| `code` | `deepseek-coder` | Генерация кода |

Клиенты используют псевдонимы; AiDog направляет на реальные платформы по конфигурации группы.

## Цепочечная маршрутизация

Комбинируйте маппинги группы и платформы для многоуровневого преобразования:

1. Клиент запрашивает модель `smart`
2. Группа A маппит `smart` → `claude-3-opus`
3. Платформа X маппит `claude-3-opus` → `claude-3-opus-20240229`
4. Отправлено в Anthropic

## Тайм-аут уровня группы

Каждая группа может независимо задавать тайм-аут запроса и подключения, переопределяя глобальные значения. Полезно, когда более медленным моделям нужно больше времени.

См. [Создание и управление](/ru/groups/manage-groups).
