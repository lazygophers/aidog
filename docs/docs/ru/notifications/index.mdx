# Системные уведомления

AiDog имеет встроенный модуль уведомлений, оповещающий о ключевых событиях ассистентов кода вроде Claude Code / OpenAI Codex (завершение задачи, ожидание ввода, ошибка) через четыре канала — **TTS-озвучивание / всплывающее окно / звук / входящие** — чтобы вы оставались в курсе статуса прокси даже вдали от экрана.

## Как это работает

1. Ассистент кода вызывает локальный эндпоинт `POST /api/notify` через скрипт hook при событии
2. AiDog диспетчеризирует по типу (`NotifType`) и настроенным каналам (`Channels`)
3. TTS озвучивает, всплывает системное уведомление, играет звук, запись пишется во входящие

## Типы уведомлений

Четыре `NotifType`:

| Тип | Литерал | Триггер |
| --- | --- | --- |
| TaskComplete | `task_complete` | Задача завершена |
| WaitingInput | `waiting_input` | Ожидание ввода пользователя |
| Error | `error` | Произошла ошибка |
| Custom | `custom` | Пользовательский |

Каждый тип настраивается независимо: озвучивать ли, какие каналы.

## Каналы и формы

Форма диспетчеризации (`NotifForm`) определяет включённые каналы:

| Форма | Каналы |
| --- | --- |
| Full | TTS + popup + звук + входящие |
| PopupOnly | popup + входящие |
| InboxOnly | входящие |
| SoundOnly | звук |

> Всплывающие уведомления сохраняются во входящих для последующего просмотра.

## TTS-бэкенды

Три `TtsBackend`:

| Бэкенд | Описание |
| --- | --- |
| CrossPlatform | Кроссплатформенный (по умолчанию) |
| MacSay | Нативный macOS `say` |
| WebSpeech | Web Speech API браузера |
