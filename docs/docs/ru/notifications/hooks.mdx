# Интеграция hook

Уведомления запускаются **hook'ами** ассистента кода. AiDog генерирует скрипты hook и в один клик инъецирует их в конфигурацию Claude Code / Codex; при событиях скрипт вызывает локальный эндпоинт `POST /api/notify`.

## Поддерживаемые клиенты

`HookClient`:

| Клиент | Расположение конфигурации |
| --- | --- |
| Claude Code | поле `hooks` файла `settings.json` |
| Codex | конфигурация Codex (поле notify) |

## Инъекция в один клик

Настройки → Уведомления → Интеграция hook:

- **Генерация скриптов** (`generate_hook_scripts`) — генерирует скрипты hook по типам, записывает в локальные пути (`ScriptPaths`)
- **Инъекция** (`inject_hooks`) — записывает пути скриптов в event hook'и Claude Code `settings.json` или notify-конфиг Codex
- **Удаление** (`remove_hooks`) — очищает инъецированные hook'и (только записи AiDog, по `is_aidog_command`)

Ручное редактирование конфигурации не требуется.

## Поток работы

```
Событие ассистента кода (завершение / ожидание ввода / ошибка)
   ↓ срабатывает hook
скрипт hook → POST /api/notify
   ↓
Диспетчеризация AiDog (по NotifType + Channels)
   ↓
TTS / popup / звук / входящие
```

## Ручной вызов эндпоинта

Можно вызвать эндпоинт напрямую без hook для теста:

```bash
curl http://127.0.0.1:9876/api/notify \
  -H "Content-Type: application/json" \
  -d '{"type": "task_complete", "title": "Done", "body": "Task ended"}'
```

> Эндпоинт `POST /api/notify` только localhost, тот же порт, что прокси (по умолчанию 9876).

## Удаление инъекции

При отключении уведомлений используйте «Удалить hook» для очистки записей AiDog из конфигурации Claude Code / Codex.
