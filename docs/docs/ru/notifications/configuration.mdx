# Конфигурация уведомлений

Настройки уведомлений (`NotificationSettings`) в Настройки → Уведомления. Управляют главным переключателем, TTS, каналами по типам.

## Глобальные переключатели

| Поле | По умолчанию | Описание |
| --- | --- | --- |
| `enabled` | true | Главный переключатель. Если выкл — вся диспетчеризация пропускается |
| `tts_enabled` | true | Главный TTS. AND с по-типовым `tts` (оба вкл для озвучивания) |
| `tts_backend` | CrossPlatform | TTS-бэкенд (CrossPlatform / MacSay / WebSpeech) |

## Конфигурация по типам (per_type)

`per_type` — карта `NotifType → TypeSetting`; каждый тип независимо:

| Поле | Описание |
| --- | --- |
| `tts` | Озвучивать ли этот тип (AND с глобальным `tts_enabled`) |
| `form` | Форма диспетчеризации (Full / PopupOnly / InboxOnly / SoundOnly) |

Отсутствующие ключи трактуются как все каналы вкл + Full.

## Переменные шаблона

Шаблоны заголовка/тела поддерживают плейсхолдеры, подставляемые при диспетчеризации (`substitute_vars`):

- `{project}` — имя проекта
- `{status}` — статус
- другие контекстные переменные

Пример: шаблон `{project} · {status}` → заполняется фактическими значениями.

## Тестовое уведомление

На странице настроек есть кнопка «Тест» (`notification_test`), диспетчеризирующая тестовое уведомление по текущей конфигурации.
