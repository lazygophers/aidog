# Просмотр журналов

Страница **Журналы** записывает каждый прокси-запрос, проходящий через AiDog, облегчая диагностику маршрутизации, преобразования протоколов и ответов upstream.

## Список журналов

Каждая запись показывает:

| Поле | Описание |
|------|------|
| Группа | Имя сопоставленной группы |
| Модель | Исходная модель, запрошенная клиентом |
| Фактическая модель | Модель, отправленная upstream после маппинга |
| Протокол | Входящий протокол → upstream-протокол |
| Платформа | Платформа, обработавшая запрос |
| Статус | HTTP-код (успех / сбой) |
| Длительность | Общая длительность (мс) |
| Токены | Входные / выходные / кэш |
| Время | Момент запроса |

Список показывает по 50 записей на страницу с пагинацией.

## Фильтрация

Используйте фильтры сверху для быстрого поиска:

- **Платформа**: по конкретной платформе
- **Группа**: по группе
- **Статус**: только успех / только сбой
- **Временной диапазон**: всё / последний час / 6 часов / 24 часа / 7 дней / 30 дней
- **Модель**: поиск по исходному или фактическому имени модели

## Детали запроса

Щёлкните любую запись для раскрытия деталей и просмотра полной цепочки запроса:

- **Запрос пользователя**: исходный URL, заголовки и тело запроса
- **Upstream-запрос**: преобразованный URL, заголовки и тело, отправленные upstream
- **Upstream-ответ**: код состояния, заголовки и тело ответа upstream
- **Ответ пользователю**: финальные заголовки и тело, возвращённые клиенту

Детали поддерживают копирование в один клик.

:::tip
Содержимое «Запрос пользователя» и «Upstream-запрос» управляется настройками журналирования. По умолчанию тело запроса не записывается. См. [Настройки журналирования](/ru/logs/log-settings).
:::
