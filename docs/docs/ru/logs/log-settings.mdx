# Настройки журналирования

На странице **Настройки** можно управлять детализацией и сроком хранения журналов, балансируя наблюдаемость и приватность / расход диска.

## Уровни записи

Журналирование имеет три уровня переключателей, действующих прогрессивно:

| Уровень | Описание | По умолчанию |
|------|------|------|
| Главный переключатель | Если выкл — никакие прокси-журналы не записываются | Вкл |
| Запись исходного запроса пользователя | Записывает исходные заголовки и тело запроса клиента | Выкл |
| Запись upstream-запроса | Записывает преобразованные заголовки и тело запроса upstream | Выкл |

:::warning
Тело запроса может содержать чувствительные данные (промпты, ключи и т.д.). Убедитесь, что локальное окружение доверено, прежде чем включать запись.
:::

## Хранение

Журналы очищаются по трём независимым периодам хранения:

| Тип | По умолчанию | Метод очистки |
|------|------|----------|
| Исходный запрос пользователя | 7 дней | Очищает только поле тела, сохраняет строку |
| Upstream-запрос | 7 дней | Очищает только поле тела, сохраняет строку |
| Записи журнала | 90 дней | Удаляет всю запись |

Иначе говоря, тела запросов старше 7 дней очищаются (статистика сохраняется), а журналы старше 90 дней удаляются целиком.

## Ручная очистка

В **Настройки → Управление данными** можно вручную запустить «Очистить устаревшие журналы» в любой момент.

## Хранение данных

Все журналы хранятся в локальной базе `~/.aidog/aidog.db` и никогда не загружаются на сервер.
