# MCP 页面全屏适配 (mcp-fullscreen-layout)

## 目标
MCP 管理页 (`src/pages/Mcp.tsx`) 当前用固定宽度，未适配全屏/宽屏场景，宽屏下内容挤在左侧大量留白。改为响应式自适应宽度，铺满可用区域（与其它页面一致的布局规范）。

## 背景 / 问题
- 用户反馈：MCP 页面没有适配全屏的场景，使用了固定宽度。
- 现象：窗口最大化/全屏时，MCP 页主体内容仍是固定窄宽度，右侧大片空白。

## 需求拆解 (前端，单页)
- 定位 `Mcp.tsx` 中固定宽度来源（`maxWidth` / `width: NNNpx` / 固定 px 容器 / 居中 wrapper）。
- 改为自适应：容器宽度跟随视口（如 `width:100%` + 合理 `max-width` 或直接铺满 padding 内边距），与同项目其它管理页（Platforms / Skills / Settings）的宽度策略对齐 —— **先看这些页怎么做，照搬同一模式，禁自创新方案**（见 code-reuse-rules）。
- 保留必要的可读性约束（如超宽屏下正文不至于一行过长，可设上限），但默认应充分利用宽屏空间，消除大片留白。
- 列表/卡片/表单等子区域随容器自适应，不出现横向溢出或错位。

## 复用 (Explore 须先做)
- 对比 `src/pages/Platforms.tsx` / `Skills.tsx` / `Settings.tsx` 顶层容器宽度写法，取一致策略。
- Liquid Glass UI 风格不变。

## 验收标准
- 窗口最大化/全屏时 MCP 页主体铺满可用宽度，无大片右侧留白。
- 普通窗口尺寸下布局正常，无横向滚动条/溢出/错位。
- 与其它管理页宽度观感一致。
- `yarn build` (tsc && vite build) 通过。
- 若涉及文案改动则 `node scripts/check-i18n.mjs` 绿（本任务预计无新增文案）。

## 非目标
- 不重构 MCP 功能逻辑，仅布局宽度适配。
- 不改后端、不改其它页面。

## 单一交付
纯前端单页布局修复，单 worktree，单一可验收交付。
